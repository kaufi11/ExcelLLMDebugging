Findings
- One clear formula error: cell I21 contains =H21*38. It should be =H21*C21.
  - Effect: I21 yields 38 instead of the correct weighted score 3 (H21=1 * C21=3). That overstates Oracle’s weighted total by 35 points.
- Minor consistency issue: some weighted-score cells were entered as hard values instead of formulas (for example R8 and R9 show correct numbers but not a formula). Hard-coded values are risk points for future errors.

Why it’s wrong
- The pattern for each vendor’s weighted score is: WeightedScore = (VendorScore) * (CategoryWeight). For the Oracle column that is I = H * C. Using 38 instead of C21 breaks that pattern and produces the wrong subtotal.

Fix (quick)
1. Select I21 and replace the formula with:
   =H21*C21
2. Press Enter. Excel will recalc I44 (the Oracle WS sum) and any dependent totals.
3. (Optional but recommended) For safety, copy the formula pattern down the Oracle weighted-score column: select I8, then drag/fill I8:I43 so each row uses Hrow*Crow.
