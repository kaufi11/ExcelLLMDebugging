I checked the workbook and found a few real errors that will produce wrong totals and percentages — plus some sloppy formulas that are harmless but should be cleaned up. Quick summary, cause, and exact fixes:

1) Big error — wrong operator in FTEE (row for NCC)
- Problem: FTEE Calculation!F14 has F14 = +E14 * 600 (multiplying hours by 600).
- Why it’s wrong: every other non-credit row uses E/600 to convert non‑credit hours to FTEE. Multiplying produces a huge FTEE and inflates the college and total FTEE and all related % values.
- Fix: replace F14 with =E14/600

2) Several stray “+1” values artificially add 1 to data/totals
- Occurrences:
  - Revenues by Source!C11 = 141577402+1
  - General State Financial Aid!B21 = 3684419+1
  - FTEE Calculation!E27 = SUM(E11:E26)+1
- Why it’s wrong: adding +1 increments the raw value and every dependent total/percentage by one unit — a wrong hack (probably meant to adjust rounding).
- Fix: remove the +1 (use the true value or the SUM without +1). If you need to compensate for rounding, use ROUND() where appropriate, not +1.

3) Incorrect/overbroad SUM ranges (includes blank rows)
- Examples:
  - Revenues by Source: B15 = SUM(B9:B14) and C15 = SUM(C9:C14) — there are only rows 9–13 of data.
  - Expenditures by Function: B19 = SUM(B9:B18) and C19 = SUM(C9:C18) — data ends at row 17.
  - Expenditures by Category: B15/C15 include B9:B14/C9:C14 though data ends at row 13.
  - FTEE Calculation totals use ranges to row 26 but data ends at row 25.
- Why it’s a problem: summing extra blank rows is usually harmless, but it’s sloppy and can hide future errors if extra rows contain stray values.
- Fix: restrict sums to the actual data ranges:
  - Revenues: B15 =SUM(B9:B13) and C15 =SUM(C9:C13)
  - Expenditures by Function: B19 =SUM(B9:B17) and C19 =SUM(C9:C17)
  - Expenditures by Category: B15 =SUM(B9:B13) and C15 =SUM(C9:C13)
  - FTEE: use B27 =SUM(B11:B25), C27 =SUM(C11:C25), E27 =SUM(E11:E25), F27 =SUM(F11:F25), H27 =SUM(H11:H25), I27 =SUM(I11:I25)

4) Cosmetic / inconsistent formulas
- Many cells use a leading + (e.g. =+C11/H11) — harmless but unnecessary. Not an error, but remove the + for clarity.
- Revenues C9 uses =9432730+111410104 — better to use =SUM(9432730,111410104) or replace with the single resulting number for clarity.

5) Rounding adjustments — use proper function
- If the +1 was intended to force totals to match after rounding, use ROUND() or ROUND(SUM(...),0) instead of adding 1. Example:
  - =ROUND(SUM(E11:E25),0)
