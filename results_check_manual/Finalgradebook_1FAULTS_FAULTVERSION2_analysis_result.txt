1) Systematic wrong AVERAGE references (Sheet3)
- Problem: Cells P4:P18 use the wrong L-column row in the AVERAGE formula — they reference L40, L41, … L54 instead of L4, L5, … L18. Example:
  - P4: =AVERAGE(D4,F4,H4,J4,L40)  (should be L4)
  - P5: =AVERAGE(D5,F5,H5,J5,L41)  (should be L5)
  - ... continuing the pattern through P18 which points at L54.
- Impact: Each P-row averages only 4 test-percentage columns (D,F,H,J) because the L reference points to empty cells, so the test-average (Sheet3 P) is incorrect (and typically larger than it should be).
- Fix: Change each Pn to use its corresponding Ln. Example corrected formulas:
  - P4: =AVERAGE(D4,F4,H4,J4,L4)
  - P5: =AVERAGE(D5,F5,H5,J5,L5)
  - ...
  - P18: =AVERAGE(D18,F18,H18,J18,L18)

5) Inconsistent units in final grade calculation (Sheet1 I3:I17)

  - Tests/Midterm/Final/Participation (columns D,E,F,G) are stored as fractions 0–1 (Sheet3 averages) and are multiplied by 25 or 12.5 (not by 0.25/0.125).
- This is not a numeric error (the math can be made to work), but it is inconsistent and fragile/ confusing.
- Recommendation: pick one consistent convention:

    - Better: keep D..G as 0–1 and use weights 0.25/0.125 consistently: = (C3/100*0.125*100?) — simpler: convert everything to percentages (0–100) then apply decimal weights.
  - I recommend converting columns D:E:F:G to percentages 0–100 (multiply current fractional values by 100) or change the I formulas to consistently use decimals (e.g. multiply homework by 0.125 and test fraction by 0.25). Example consistent formula (if D..G are fractions 0–1 and you want final grade as 0–100):
