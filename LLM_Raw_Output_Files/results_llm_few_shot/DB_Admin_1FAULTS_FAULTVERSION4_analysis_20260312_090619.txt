Why these fixes work
- The Ap sheets display two blocks: Proficiency (should map Rate!G11..G15 to Rate!E11..E15) and Experience (should map Rate!G16..G20 to Rate!E16..E20). Some rows were pointing to the wrong Rate!G (B19) or the L formulas referenced the wrong E row (L32/L33). Correcting the references aligns each description with its correct rating cell so the IF multipliers compute the intended scores.
- Fixing the IF operator in Ap3 L20 makes the formula test equality properly.

Suggestion to prevent this class of error
- Use INDEX/MATCH (or VLOOKUP with appropriate layout) so the L formulas find the Rate!E value by matching the text in column B instead of hard-coding E11/E12/etc. Example (replace an IF chain that multiplies by a factor):
  =IF(K16=2, INDEX(Rate!$E:$E, MATCH(B16, Rate!$G:$G, 0)) * 0.33,
     IF(K16=3, INDEX(Rate!$E:$E, MATCH(B16, Rate!$G:$G, 0)) * 0.66,
     IF(K16=4, INDEX(Rate!$E:$E, MATCH(B16, Rate!$G:$G, 0)) * 1, 0)))
- That way you only rely on the description to find the correct E value; moving rows on Rate won't break the Ap sheets.
