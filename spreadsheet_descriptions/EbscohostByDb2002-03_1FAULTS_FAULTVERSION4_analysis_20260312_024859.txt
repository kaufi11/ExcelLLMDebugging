Spreadsheet overview — structure and key contents (concise)

General layout (consistent across all sheets)
- One sheet per campus: HawCC, HonCC, KapCC, KauCC, LeeCC, MauCC, UHH, UHM, UHWO, UHCWH, WinCC.
- Title block: A1 = "University of Hawaii", A2 = "Ebscohost Usage by Database, 2002-03 (July-June)"; A4 = campus name.
- Column headers: row 7 = "Database Name" (A7), "Searches" (B7), "Articles Viewed" (C7); row 8 subheaders under Articles Viewed: C8 = "Full-text", D8 = "PDF", E8 = "Total".
- Data rows: databases listed in A10:A28 (Academic Search Premier down to Religion and Philosophy Collection).
- Metrics: B10:B28 = Searches; C10:C28 = Full-text article views; D10:D28 = PDF downloads; E10:E28 = per-database Total (Full-text + PDF).
- Totals row: row 29 shows campus sums. B29 = total Searches, C29 = total Full-text, D29 = total PDF, E29 = total articles (C29 + D29).

Formulas and calculation notes
- Per-database totals E10:E28 use E# = SUM(C#:D#) (adds Full-text + PDF).
- Campus-level total in E29 uses E29 = SUM(C29:D29) (adds the totals from C29 and D29); equivalently E29 represents overall Full-text + PDF for that campus.
- Searches (column B) are not included in the E-column totals (E sums only C + D).
- Consistent data range: databases in rows 10–28; total row 29.
