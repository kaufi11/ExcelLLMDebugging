High-level summary (concise)

Workbook purpose
- A scoring/qualification tool for a job requisition (DATABASE ADMINISTRATOR). Rate sheet defines the competency items and weights; Ap1–Ap3 capture three applicants’ proficiency/experience selections and compute weighted scores; Summary aggregates applicants’ scores.

Sheets and contents

1) Rate (master)
- Header: C3 = Requisition #: "M34933"; C4 = Job Title: "DATABASE ADMINISTRATOR"; C5 = Department: "VICE PROVOST FOR RESEARCH".
- Core table: competency text in G11:G20 (e.g., "Computer and/or network security…", "Data integrity…", "Develop, program…", etc.).
- Rating scale / weights: E11:E20 set to 100 (user data). E22 = SUM(E11:E20) (total).
- Yes/No markers in B/C for each competency (X marks indicate requirement).
- Legend/key for proficiency (None/Low/Medium/High) and experience (None/<1/1–3/3–5/>5).

2) Ap1 (applicant 1)
- Pulls header info from Rate: D3 = Rate!C3, D4 = Rate!C4, D5 = Rate!C5.
- Questions: B16:B20 and B29:B33 pull competency text from Rate (Rate!G11:G20).
- Applicant selections: K16:K20 (proficiency numeric codes), K29:K33 (experience numeric codes).
- Scoring formulas (column L):
  - Proficiency L16:L20 use nested IFs mapping codes (2→0.33, 3→0.66, 4→1) multiplied by corresponding Rate!E## (e.g., L16 = IF(K16=2,Rate!E11*0.33,...)).
  - Experience L29:L33 use nested IFs mapping codes (2→0.25, 3→0.5, 4→0.75, 5→1) multiplied by corresponding Rate!E##.
- Note: many L formulas are cross-sheet references to Rate!E values to compute weighted points.

3) Ap2 (applicant 2)
- Same layout/logic as Ap1: headers pulled from Rate, B16:B20 and B29:B33 pull Rate!G11:G20, K columns set applicant choices, L columns compute weighted scores with the same IF-multipliers as Ap1.

4) Ap3 (applicant 3)
- Same structure as Ap1/Ap2: pulls headers, shows questions, K-coded answers, L computed scores.

5) Summary
- Column headers 1..10 correspond to the 10 question slots; L column is total.
- Rows for Ap1/Ap2/Ap3: cells B8:K10 pull each applicant’s L16:L33 scores (e.g., B8 = 'Ap1'!L16, …, K10 = 'Ap3'!L33).
- Totals: L8:L10 = SUM of each applicant’s row (SUM(Bx:Kx)).

Key formulas and logic
- Aggregation: Rate!E22 = SUM(E11:E20).
- Proficiency scoring: nested IFs with multipliers .33/.66/1 × Rate!E(cell) (proficiency codes 2,3,4).
- Experience scoring: nested IFs with multipliers .25/.5/.75/1 × Rate!E(cell) (experience codes 2–5).
- Cross-sheet links: Ap sheets reference Rate!G11:G20 (question text) and Rate!E11:E20 (weights); Summary pulls computed points from Ap sheets.

Important observations / issues to review (concise)
- Several apparent copy/paste reference errors in Ap sheets:
  - Many L formulas that should use Rate!E16, E19, E20 etc. instead reference Rate!E14, E17 or E18. Example: Ap1 L19 references Rate!E14 but B19 pulls Rate!G16 (should use Rate!E16).
  - Ap1 L32 references Rate!E17 instead of Rate!E19; L33 references Rate!E18 instead of Rate!E20. Similar mismatches appear in Ap2 and Ap3.
- Ap3 L20 is hard-coded as =5 (instead of computing via the IF formula like other proficiency cells).
- These mismatches will produce incorrect weighted scores for affected competencies and therefore wrong totals in Summary. Recommend auditing L16:L20 and L29:L33 on each Ap sheet to ensure each score uses the matching Rate!E cell for the question text pulled from Rate!G##.
