1) ID column (column A) showing negative numbers instead of the IDs
- Problem: many entries in column A are written like 356-78-4365 (with hyphens) but Excel interprets that as a numeric expression (356 - 78 - 4365) and calculates a negative number.
- Why it’s wrong: IDs with hyphens must be text, not expressions.
- Fix: convert the column to Text and re-enter IDs as text. Easiest:
  - Select column A → Format Cells → Text → OK, then retype each ID (or edit each and put a leading apostrophe before the ID: '356-78-4365).  
  - Or use a formula if you still have the original string in another cell: =TEXT("356-78-4365","@") or ="356-78-4365".
- Result: IDs will display correctly and will not be used accidentally in calculations.

2) Grade IF formulas reference different O38 cells (inconsistent cross-sheet references)
- Problem: most grade formulas are like =IF(O6>O$38*0.9,... ) (good — uses the sheet’s own O38). But some rows/sheets reference other sheets (e.g. 'F Noon'!O$38 or 'M Noon'!O$38) or even mix sheets in single formulas.
- Why it’s wrong: grades should be compared to a consistent “total possible” value. If different sheets point to different O38 cells, the grade boundaries differ and produce inconsistent/incorrect grades.
- Fix:
  - Decide one source for the “Total possible points” (for example O38 on each sheet or a single summary sheet cell). Then make every grade formula use the same absolute reference, e.g. either same-sheet absolute:
    =IF(O6>=O$38*0.9,"A",IF(O6>=O$38*0.8,"B",IF(O6>=O$38*0.7,"C",IF(O6>=O$38*0.6,"D","F"))))
    or (better) create a named range (e.g. MaxPoints) on the master sheet and use:
    =IF($O6>=MaxPoints*0.9,"A",...).
  - Also use >= instead of > if you want the boundary score (exact 90%, 80% etc) to be included in the higher grade.
- Result: all sheets will use the same thresholds and grades will be consistent.

3) SUMs and totals may be wrong because some score cells are stored as text or missing
- Problem: you use SUM(Bx:Nx) for row totals, but there are many cells that appear non-numeric (text like "3 & 4", stray characters, or blank cells that should be zero). SUM silently ignores text and that can hide data-entry errors.
- Why it’s wrong: numeric cells stored as text are not included in SUM; totals/averages become wrong.
- Fix checklist:
  - Use Excel’s Error Checking (green triangles) or test a sample with =ISNUMBER(B6) to find text-but-number cells.
  - Convert text numbers to numbers: select the range → Text to Columns → Finish; or copy a blank cell, Paste Special → Multiply by 1; or use VALUE() to coerce strings.
  - Remove stray characters (e.g., remove ampersands or text in score cells). Keep text like "1&2 on Sam" only in header rows, not in data rows.
  - Optionally make the total formula robust: =SUMPRODUCT(--(B6:N6)) will coerce number-strings to numbers for summation, but it’s better to correct the data.
- Result: row totals O6 etc will reflect the true numeric sum.
