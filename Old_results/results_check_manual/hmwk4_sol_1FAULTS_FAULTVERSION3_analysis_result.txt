1) S2 ("Latitude") — wrong conversion (seconds vs degrees)
- Current formula: =F2+G2*60+H2*3600
- Problem: that produces total arc-seconds (170,481) not degrees. The rest of the workbook (e.g. the HA/altitude formulas) use $S$2 multiplied by $U$1 (pi/180) expecting degrees, not seconds.
- Correct formula: =F2+G2/60+H2/3600
- Impact: this is a major error — it affects every calculation that uses $S$2 (e.g. the V (airmass) formulas).

2) Many V (airmass) formulas multiply the seconds/minutes term by 3600 instead of dividing
- Examples of incorrect formulas (representative):
  - V55 uses (ABS(F55)+G55/60+H55*3600)
  - V65 uses (ABS(F65)+G65/60+H65*3600)
  - V75, V85, V95, V105, V115, V125, V135, V145, V155, V165 (and likely additional rows in those 10-row blocks) show the same pattern.
- Problem: H is the arcsecond component and should be divided by 3600, not multiplied. Multiplying by 3600 produces a huge angle and makes the trigonometric functions meaningless.
- Correct pattern: (ABS(Fn)+Gn/60+Hn/3600)
- Note: fix all Vnn formulas that currently have Hn*3600 → Hn/3600.

3) Wrong denominator in AA (std-dev-of-sky) formulas — mis-typed cell used
- Pattern / problem: In many "AA" cells (the per-block sky STDEV adjustment) the formula ends with /Nxx/Nyy where the second term is a cell in column N (a row near the end of the block) instead of dividing by the corresponding O column cell (the per-measurement gain/other factor).
- Examples:
  - AA15: =STDEV(P15:P24)/N15/N24  → second "/N24" is wrong (should be /O15)
  - AA25: =STDEV(P25:P34)/N25/N34  → should be /N25/O25
  - AA35: =STDEV(P35:P44)/N35/N44  → should be /N35/O35

- Correct pattern: =STDEV(Pn:Pn+9)/Nn/On (i.e. divide STDEV by the same denominators used for the corresponding average in Z and the X-block formulas)
- Note: AA5 was correct (AA5 = STDEV(P5:P14)/N5/O5). The error starts in the block beginning at row 15 and repeats every block.
