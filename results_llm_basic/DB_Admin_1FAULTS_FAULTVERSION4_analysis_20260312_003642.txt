Why this fixes it
- Each L cell is supposed to multiply the correct Rate!E value for the matching question (E11 → L16, E12 → L17, …, E20 → L33). The wrong formulas were pointing to E11..E18 incorrectly; fixing the references aligns the correct weight value with the correct question and K-key selection. Fixing the IF in Ap3 L20 restores the intended "=" comparisons.

Recommendation to avoid recurrence
- Use a single consistent formula pattern and then copy it down (relative references) rather than manually editing individual cell references. Example robust pattern for the proficiency block (starting L16) using OFFSET (so you only write it once and copy down):
  - Proficiency (L16 and down): =IF(K16=2,OFFSET(Rate!$E$11,ROW()-16,0)*0.33,IF(K16=3,OFFSET(Rate!$E$11,ROW()-16,0)*0.66,IF(K16=4,OFFSET(Rate!$E$11,ROW()-16,0)*1,0)))
