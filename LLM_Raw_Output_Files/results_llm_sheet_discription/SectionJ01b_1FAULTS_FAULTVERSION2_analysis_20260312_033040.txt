3) Hard-coded / non-referenced formulas
- Problem: Example in T&M N37 is written as =920+(920*0.023) (literal 920). Hard-coded numbers break automatic updates and produce inconsistencies.
- Fix: Replace hard-coded expressions with references to the prior option cell:
  - e.g. N37 → =K37*1.02 (or =K37*1.023 only if 2.3% is intentionally required)
- Why: Using cell references keeps formulas dynamic and consistent when base/previous values change.

4) Fragile cumulative formulas — risk of future copy errors
- Problem: Current pattern chains previous column =Prev + Prev*0.02 across many columns; that worked but led to the inconsistent 0.023 and a few manual deviations.
- Better solution (safer & easier to audit): compute each option directly from the Base Pd with a power formula so each option is self-contained:

  - D (Opt2) = B * 1.02^2
  - E (Opt3) = B * 1.02^3
  - … I (Opt7) = B * 1.02^7
  - Example for row 14: C14 = $B14*1.02, D14 = $B14*1.02^2, etc.
- Why: This prevents propagation-of-error and makes it obvious that every option is +2% compounded from base.
