I checked the workbook and found the main formula errors that will give wrong totals/NPV. Short fixes and explanations below.

1) Faulty cells: AA17 (Environmental clean up in 2020) and AA29 (Scrap value in 2020)
- Error type: Wrong-column reference / year mismatch (mechanical)
- Explanation: Both formulas use G5 and G6 (year 2000 cells) instead of the 2020 column values (AA5 and AA6). That makes the 2020 cleanup and scrap values reference costs in year 2000 instead of year 2020.
- Fix:
  - AA17 → =0.1*(AA5+AA6)  (or =0.1*AA5 + 0.1*AA6)
  - AA29 → =0.02*(AA5+AA6)
- Why: When you calculate a cost that applies in year 2020 you must reference the cost cells in the same year column (AA). Using G5/G6 pulls the wrong-year amounts.

2) Faulty cell: R19 (Discount factor row)
- Error type: Formula typo → breaks discount factor chain (mechanical / logical)
- Explanation: All discount factors are being propagated by multiplying the previous cell by (1/1.12). R19 is written as =Q19/(1/1.12) which is equivalent to Q19*1.12 (the opposite), so from R19 on the DF series is wrong and every present value after year 10 will be incorrect.
- Fix: change R19 to multiply like the others:
  - R19 → =Q19*(1/1.12)
- Why: Discount factor for year n = previous year * (1/(1+rate)). Using division by (1/1.12) is the inverse and inflates rather than discounts.

- Fix: After correcting R19 and AA17/AA29, recalc the sheet. Verify G20:AA20 and G32:AA32 (present value of costs/benefits) now use correct discount factors and year references.
