Summary — main problems
- Many cells in the daily data area (rows 9–39, columns D:M and some in E) contain wrong formulas that either:
  - Sum a single unrelated cell (e.g. E9 =SUM(E42)), or
  - Chain across columns (F9 =SUM(E9), G9 =SUM(F9) …), causing wrong repeated values, or
  - Use the wrong function on a single cell (J19 =COUNT(I19)).
- The totals row (row 40) is broken from column F onward: F40 =SUM(E40), G40 =SUM(F40) etc. That simply propagates E40 across all totals instead of summing each column.
- Result: per‑day totals and column totals are incorrect.

What to do (quick fixes)
1. Clear wrong formulas in the data area
   - Select D9:M39 (and any individual bad cells like E9, J19) and Clear Contents. These should be raw data entries (or correct formulas), not formulas that reference other cells in the same row/column incorrectly.

2. Create per‑day totals in column C
   - In C9 enter: =SUM(D9:M9)
   - Fill/Ctrl+D down C9:C39. This gives each day's Total #.

3. Fix column totals in row 40
   - Replace the current formulas from D40 to M40 with proper column sums:
     - D40 =SUM(D9:D39)
     - E40 =SUM(E9:E39)
     - F40 =SUM(F9:F39)
     - G40 =SUM(G9:G39)
     - H40 =SUM(H9:H39)
     - I40 =SUM(I9:I39)
     - J40 =SUM(J9:J39)
     - K40 =SUM(K9:K39)
     - L40 =SUM(L9:L39)
     - M40 =SUM(M9:M39)

4. Re-enter your daily data
   - Type the actual daily counts into D9:M39 (or paste them). The C column and row 40 totals will update automatically.

Why this works (short)
- The bad formulas were either referencing the wrong single cell or cascading references, so they produced wrong replicated values. Clearing them and using SUM across the correct vertical ranges ensures each column total sums the actual daily values. Adding C9 =SUM(D9:M9) gives the correct per‑row total so C40 (already =SUM(C9:C39)) becomes correct.
