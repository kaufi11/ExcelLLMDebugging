Summary of MODEL07 worksheet (MODEL 7: Push‑Down & Consolidation)

Overall layout
- Single worksheet that contains two working areas on the same sheet:
  - Push‑down worksheet: begins at row 49 (“WORKSHEET TO REFLECT THE PUSH‑DOWN BASIS OF ACCOUNTING”).
  - Consolidation worksheet: begins at row 100 (“CONSOLIDATION WORKSHEET AS OF …”).
- Header/instructions at top (rows 1–46) with cautions, where to load problem data, and where to use macros.
- Problem data stored at the right side: K105:U132 (source values for problems 7‑1 through 7‑7). Row 98–101 identifies problem numbers/keys.

Key sections and contents
- Excess cost / push‑down analysis (rows ~15–22): area for analyzing investment account excess cost elements (inventory, land, bldg & eq, accumulated depreciation, patent, goodwill). B22 shows total formula =SUM(C22:I22).
- Push‑down entries (rows ~24–46): debit/credit entry lines to implement push‑down accounting; C39 =SUM(C27:C38) and D39 =SUM(D27:D38) compute totals. Macro Q is intended to post entries into the push‑down area (rows beginning 49).
- Push‑down balance sheet area (rows 52–65): standard balance sheet lines (Cash, A/R, Notes receivable, Inventory, Land, Bldg & Eq., Accum. Depr., Patent, Goodwill). New basis columns use formulas like F54 = (C54 + D54 − E54). Totals: C65 =SUM(C54:C64), etc.
- Analysis of parent’s investment — book value element (rows ~78–85): shows subsidiary equity breakdown and B85 = SUM(C85:F85).
- Basic elimination entry (rows 88–96): populates elimination entry lines using results of the analysis (e.g., C90 = $C$85, C91 = $D$85, C92 = $E$85, C93 = $F$85, D94 = $B$85). C95/D95 sum debits/credits. Macro P posts this entry to the Consolidation worksheet.

Consolidation worksheet structure (rows 100+)
- Columned consolidation layout keyed to multiple problems (columns K:U hold subsidiary or parent columns for different problem setups).
- Balance sheet rows duplicate the push‑down balance sheet items for each problem column (Cash K105:U105, A/R K106:U106, Inventory K108:U108, Land K111:U111, Bldg & Eq K112:U112, Accum. Depr. K113:U113, Patent K114:U114, Goodwill K117:U117).
- Totals and proofs:
  - Row 118 sums assets per column (e.g., K118 = SUM(K105:K117)).
  - Row 132 sums liabilities & equity per column (e.g., K132 = SUM(K120:K131)).
  - Proof rows compare debits/credits: D134 = (D118 + D132), E134 = (E118 + E132); earlier proof for push‑down D76 = (D65 + D74), E76 = (E65 + E74).
- Account formulas in consolidation: many cells use the pattern Fxx = (Bxx + Cxx + Dxx − Exx) or similar to compute combined/new balances; push‑down/new basis formulas mirror that pattern in the push‑down section.

Problem data and sample values
- K105:U132 is the problem data grid used to load unadjusted balances. It contains numeric examples for cash, receivables, inventory, land, bldg & eq., accumulated depreciation (often negative), patent, goodwill, payables, long‑term debt, and equity accounts for various test problems (e.g., K105 = 55,000; K112 = 723,000; K113 = -123,000; K120 = 300,000; K122 = 700,000; K124 = 300,000, etc.).

Macros (keyboard shortcuts)
- A–G: load UNADJUSTED problem data into the PUSH‑DOWN worksheet (A loads 7‑1, B loads 7‑2, … G loads 7‑7). Some (D–G) also load into the CONSOLIDATION worksheet.
- P: post the basic elimination entry to the CONSOLIDATION worksheet.
- Q: post the unamortized excess‑cost reclassification entry to the PUSH‑DOWN worksheet.
- U, W, X: erase/clear specified columns (U clears columns C:D:E in push‑down; W clears column B in consolidation; X clears D:E in consolidation).

Important formulas to note (representative)
- Push‑down new basis: Fxx = Cxx + Dxx − Exx (applies to Cash through Goodwill rows).
- Push‑down entry totals: C39 = SUM(C27:C38), D39 = SUM(D27:D38).
- Basic elimination: uses absolute references to the analysis totals ($B$85:$F$85) to populate C90:C94 and D94.
- Consolidation asset totals: K118 = SUM(K105:K117); similar for other problem columns.
- Proofs: row 76 and row 134 sum the balance sheet and Liab & Equity sides to check equality.

Usage flow (high level)
1. Load problem data using macro A–G (loads K105:U132 into the working columns).
2. Complete analyses (excess cost, investment book value).
3. Use macro Q to post push‑down reclassifications (rows beginning 49).
4. Use macro P to post the basic elimination entry to the consolidation area (rows beginning 100).
5. Check proofs (rows 76 and 134) for balanced debits/credits.
