Why these fixes
- The 463 in E12 is a human typo: the squared error must compare observed (C12) vs predicted (D12). Using 463 makes the error nonsense and will mislead the solver.  
- The SUM range must include every squared-error row. Summing the wrong range gives an incorrect SSE so the optimizer will fit to the wrong target.  
- Using ABS(B12) ensures the model uses |h|^n (the correct van Genuchten form) and avoids problems if some pressure values are positive or if Excel treats unary minus oddly.
