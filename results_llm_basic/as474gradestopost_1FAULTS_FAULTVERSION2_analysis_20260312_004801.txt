Findings (short)
- One clear formula error: cell F27 is =AVERAGE(F7:F25) but all other column averages use rows 7–26. F27 omits the student in row 26, so the Class average for Ex 5 is wrong.
- Minor layout/label issue: the note "(includes quizes)" appears off one cell (listed as I6). It should be next to the column header "Avg" (I5) so readers understand I5 includes quizzes.

Why it matters
- Omitting row 26 in F27 makes the Ex 5 class average too high/low depending on that student’s score — summary statistics must include the same student set.
- Misplaced labels confuse readers about what the Avg column contains.

Fixes (easy)
1. Correct the bad average:
   - Select F27 and change the formula to:
     =AVERAGE(F7:F26)
   - Press Enter. That includes the missing student.

2. Make ranges future-proof (optional but recommended):
   - Convert the data range to an Excel Table (select the data rows/columns → Insert → Table). Then use structured references; column averages will automatically include added rows.
   - Or use entire-column average but avoid header: =AVERAGE(F7:F1000) (or F7:F) with care.

3. Fix the label:
   - Move "(includes quizzes)" so it is adjacent to the I5 header (e.g., put it in I6 or as a comment on I5) or edit I5 to read "Avg (includes quizzes)".

Quick explanation of the solution
- Changing F27 to =AVERAGE(F7:F26) fixes the omitted-row error so all per-column averages use the same student set.
- Converting to a Table makes Excel update averages automatically when you add/remove students, preventing this kind of range-omission error in the future.
- Putting the note next to the header removes confusion about what the Avg column contains.
