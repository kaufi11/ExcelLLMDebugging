High-level summary
- Workbook contains 2 sheets: "Profs" (primary data) and "URLs" (collection of web links).
- "Profs" is a flat contact/list table with one header row (row 1) and data rows starting at row 2 (records through row 63).
- "URLs" lists site names and corresponding URLs in column A (sparse rows; labels and links alternate).

"Profs" sheet — structure and contents
- Columns (A:O) and their meanings:
  - A Number (record ID)
  - B Name
  - C Position
  - D University / Affiliation
  - E Email address
  - F Suggested by
  - G Personal or Related Web Page
  - H Address (street)
  - I (occasionally used for secondary address lines)
  - J City
  - K S/P (state/province)
  - L CNTY (country; sometimes used for state)
  - M Zip
  - N Phone
  - O FAX
- Data rows: row 2 = first record (Roy D. Yates). Data continues through row 63 (last listed: Ratish J. Punnoose etc.). Overall about 62 data rows (rows 2–63).

Key formulas and patterns
- Sequential numbering in column A: most rows use formulas like A3 = A2 + 1, A4 = A3 + 1, etc., so column A auto-increments from the initial value in A2.
- Exception: A61 uses a nonstandard formula =A60 + 685, creating a large jump in numbering at that row and therefore misaligns subsequent auto-incremented numbers.
- Many web pages are in column G. Some secondary address details use column I.
- Some email cells contain "?" (missing). Several rows have repeated/duplicate names or emails (e.g., Joseph A. Paradiso / Joe Paradiso use same email).

Notable data points / anomalies
- First record: A2 = 1, B2 = Roy D. Yates, E2 = ryates@winlab.rutgers.edu.
- Row with detailed address and phone: row 28 (Andrew T. Campbell) has full web, street, city (New York), state NY, country USA, zip, phone, fax.

- Phones/faxes populated for some rows (e.g., rows 28, 30, 33, 50, 55, 56, 57, 59, 61, 62, 63), otherwise often blank.

Overall observations / recommendations (brief)
- Column A numbering relies on chained formulas — fine except for A61 which introduces a large jump; fix if unintended.
- Some rows have incomplete contact info (emails, phones, address fields); consider standardizing address components into consistent columns (use I only for intended second address line).
- Consider removing duplicate entries or consolidating repeated names/emails.
