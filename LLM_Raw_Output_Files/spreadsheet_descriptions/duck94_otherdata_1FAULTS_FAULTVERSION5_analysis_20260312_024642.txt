Overview
- Workbook organizes the DUCK94 nearshore experiment inventory. Each experiment (August, October, Other) has its own sheet/tab with a header (experiment no., PI code, month, title, PI/Co-PI) followed by a standardized instrument table.
- Common table columns across sheets: ID, From, To (dates), Description, Sampling Rate, Record Length, Sampling Interval, FRF Coordinates (Cross‑Shore, Longshore), Water Depth (m), Height Above Bottom (m), Gauge Depth (m), Data Type, Format, Additional Information.

Sheets (concise)

1) Intro
- Title and short program description.
- Summary table (rows 7–11) listing experiments with Experiment #/PI code, Experiment Title, Investigators, and Time period (e.g., 2 (RHBJune/Sept), 21 (FRF), 29 (DOZRJune/Sept)).

2) RHBJune
- Header: EXPERIMENT NO: 2, PI CODE RHB, MONTH: June.
- PI: Richard H. Bennett; Co‑PI: Casey Church, Glen R. Anderson.
- Instrument table lists piezometers (IDs PA0, PA1, PD0–PD5).
- Typical sampling: 10 Hz; record lengths 4–6 hrs; location SIS / Pier; Type: Sediment pore pressure/Pressure; Format: Digital.
- Note: question text in D19–D20: “Are there 5 or 6 sensors collecting Pore Pressure data ??”

3) RHBSept
- Same structure as RHBJune but MONTH: September; date ranges From=12 To=23.
- Same instrument set (PA0, PA1, PD0–PD5), same sampling 10 Hz and metadata.
- Same question about 5 or 6 sensors in D18–D19.

4) FRF
- Header: EXPERIMENT NO: 21, PI CODE: FRF, label “FRF MEASUREMENTS PROGRAM.”
- Long list of sensors (IDs 111, 121, … 933, etc.) including Senso‑Metrics & Paroscientific pressure gauges, MMI current meters, Baylor wave gauge, 6 km waverider, wind sensors, rain gauge, atmospheric pressure, air temperature, sound pressure.
- Typical sampling: 2 Hz; record length: 10240 sec; sampling interval: every 3 hrs.
- Key numeric columns: FRF Coordinates (F/G), Water Depth (H), Bottom (J), and a computed column I with formulas like =SUM(H5-J5) (repeated per row) — this is effectively H − J; some rows use the same pattern but I20 contains =SUM(H20-251) (likely an erroneous reference).
- Notable values: Water depths H range (e.g., 7.9, 8, 17); Bottom values J ~7.4–7.66 for many pressure gauges; some coordinate values negative (F23 = -927.3) or N/A.
- Format column L indicates Digital for time series; some video entries (SVHS / Super 8).

5) DOZRJune
- Header: EXPERIMENT NO: 29, PI CODE DOZR, MONTH: June. PI: Brad Werner; Co‑PI: Steve Elgar.
- Instrument table lists Setra capacitance pressure gauges p01–p08 and various video/video time‑series entries.
- Typical sampling: 2 Hz for pressure gauges; J (Water Depth) and K (Above Bottom) provided (e.g., J9 0.0, K9 0.0 for p01; other gauges K = 0.2, 0.4, 0.6).
- FRF coordinates in columns H/I (e.g., H9=100, I9=900).
- To date entries: many rows show C = 1994‑07‑04.

6) DOZRSept
- Same layout as DOZRJune but MONTH: September; From=5 To=19.
- p01–p08 same sensor specs and coordinates; video entries repeated.

Formulas and anomalies
- FRF sheet uses I = SUM(H − J) (e.g., =SUM(H5-J5)) to compute a difference (height offset). SUM wrapper is unnecessary; H − J alone suffices.
- I20 in FRF uses =SUM(H20-251) which is inconsistent (likely intended =SUM(H20-J20) or =H20−J20).
- Several N/A or missing values in coordinate/depth columns (expected for some sensors like wind/wind direction).
- RHB sheets include an unresolved question about number of pore pressure sensors.

Key constants/typical values
- RHB pore pressure sampling: 10 Hz, 4–6 hrs.
- FRF instrument sampling: 2 Hz, 10240 sec record, every 3 hrs.
- DOZR pressure gauges: 2 Hz, continuous/continuous sampling intervals.

Layout notes
- Each sheet repeats the same metadata header and instrument table structure for consistency.
- Columns are consistent across sheets though some headers shift by rows (e.g., header rows at row 6–9 depending on sheet).
- Additional info column used for array names, tripod locations, or video format.
