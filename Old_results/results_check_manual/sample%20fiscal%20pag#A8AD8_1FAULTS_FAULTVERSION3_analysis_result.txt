1) Incorrect use of COUNT instead of SUM — likely a real error
- Sheet: 1856 — Cell: H25
  - Current formula: =COUNT(H9:H24)
  - Issue: This returns the number of numeric cells (16) instead of totaling the revenue values in H9:H24.
  - Effect: Any downstream totals that use '1856'!H25 (for example Summary and other rollups) will show the count (16) rather than the true revenue total for that period/column. I can see Summary (and fund-level totals) displaying 16 in that position.
  - Fix: Change to =SUM(H9:H24) (or otherwise supply the correct aggregation). Verify dependent totals after change.

2) Potentially fragile 3D SUM ranges depend on sheet order
- Examples: Summary uses formulas like =SUM('1003':'1856'!D6:D6), =SUM('1003':'1856'!D28:D28), etc.
  - Note: 3D ranges ('FirstSheet':'LastSheet'!Range) include all sheets between those two sheets in workbook tab order. If the workbook's tab order changes or additional sheets are inserted between 1003 and 1856, these SUMs will include/exclude unexpected sheets.
  - Recommendation: Confirm workbook sheet order is intentional and stable; if not, replace with explicit sheet references or named ranges.

- Example: Sheet 1856 H25 uses COUNT (see #1). Many other fund sheets have totals calculated with SUM; 1856 is inconsistent.
