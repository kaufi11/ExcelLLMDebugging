Summary of the problem
- The main error is that many cells in column H (the “account-range” column such as "1000-1099", "1200-1299", etc.) are incorrect — they don't match the account number in column A. This looks like a copy/paste or row-shift mistake so the ranges are offset by 100 or 200 in many places.
- Result: account-range lookups and grouping are wrong (bad for reports, mapping, validation).

Why this is happening
- The ranges in column H were likely created by copying blocks and are offset from the rows with the actual account numbers (column A). Some H cells are hard-coded (or pasted) instead of being derived from the account number, so they no longer match when rows were added/edited.

How to fix (two options)

1) Quick manual fix
- For a short list, edit each wrong H cell and replace with the correct hundred-range that matches the A cell (e.g., for A22 = 1110 set H22 = "1100-1199").

2) Best / automatic fix (recommended)
- Replace the hard-coded values in column H with a formula that derives the hundred-range from the account number in column A. This avoids future mismatches.
- If column A contains a plain numeric account (e.g., 1010, 1115, 2410), use this formula in H (adjust row numbers as needed) and fill down:

  =IFERROR(TEXT(FLOOR(A22,100),"0") & "-" & TEXT(FLOOR(A22,100)+99,"0"),"")

  (Put this in H22 and fill down.)
- If column A sometimes contains text like "4010-***" use this safer formula (handles numeric or "####-***"):

  =IFERROR(
     TEXT(FLOOR(VALUE(LEFT(A22,4)),100),"0000") & "-" & TEXT(FLOOR(VALUE(LEFT(A22,4)),100)+99,"0000"),
     ""
  )

  - Paste into H22 and fill down. This extracts the left 4 digits, coerces to a number, computes the hundred block, and prints as "####-####".

Steps to implement the automatic fix
1. Insert the safer formula into the first H cell of the block (e.g., H16 or H22) replacing any hard-coded value.
2. Fill (drag) the formula down through the entire H column area used.
3. Scan a few rows to verify ranges now match the A column.
4. If desired, copy column H and Paste Special → Values to convert formulas back to static text.
