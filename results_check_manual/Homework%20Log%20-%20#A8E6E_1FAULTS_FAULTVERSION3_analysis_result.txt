4) Hard-coded denominators and explicit IF chains — maintainability risks (recommendations)
- Many cells use formulas like:
  - Kn = (IF(Ln="x",1)+IF(Mn="x",1)+IF(Nn="x",1))/3
  - Qn = (IF(Rn="x",1)+...+IF(Xn="x",1))/7
  - AAn = (IF(ABn="x",1)+...+IF(AKn="x",1))/10
  - An = (IF(AOn="x",1)+...+IF(AWn="x",1))/9
- These work, but are fragile if columns are added/removed or you change the mark used other than "x".
- Suggest simpler, less error-prone formulas using COUNTIF and explicit ranges, e.g.:
  - Kn: =COUNTIF(Ln:Nn,"x")/COUNTA(Ln:Nn)  (or /3 if the numerator should always be out of 3)
  - Qn: =COUNTIF(Rn:Xn,"x")/7
  - AAn: =COUNTIF(ABn:AKn,"x")/10
  - An: =COUNTIF(AOn:AWn,"x")/9
- Using COUNTIF(range,"x") is easier to read and less likely to produce mismatches when columns change.
