Summary of the errors I found
1) Wrong sign in B36:
- Current formula: =B33-7
- Should be: =B33+7
Why it’s wrong: =B33-7 gives a date one week earlier (duplicates an earlier class), so the spring-break rows (B36:B38) and the classes after the break end up with wrong/duplicated dates.

2) Minor issues (not calculation errors but worth fixing):
- Many formulas have an unnecessary leading plus, e.g. "=+B16+2". This works but is untidy.
- A7 (http://ce.ecn.purdue.edu/~puenvhyd) does not match the other row‑7 entries (they use "~puevnhyd"). That looks like a typo in the URL.
- Row‑6 has the same long URL repeated across many columns — probably accidental copy/paste.

How to fix (simple direct fixes)
1) Fix B36:
- Replace =B33-7 with =B33+7.
- After that B37 (=B36+2) and B38 (=B37+2) will be correct and B39 (=B36+7) will move to the correct post‑break Monday.

2) Clean up formulas / typos:
- Remove the unnecessary leading pluses (optional): change "=+B16+2" to "=B16+2".
- Correct the URL typo in A7 (and anywhere else it’s wrong) so all row‑7 links are consistent.
- If the repeated row‑6 URLs are unintended, clear the extra cells and keep one cell with the link.

Better, more robust formula (optional)
If you want Excel to automatically handle Mon/Wed/Fri pattern (so you don’t need manual +7 references), use this in the cell below any date (example assumes dates are Mon/Wed/Fri and you want the next class date):
- Put this in B13 and fill down:
  =IF(WEEKDAY(B12,2)=5, B12+3, B12+2)
Explanation: WEEKDAY(...,2) returns 1=Mon ... 5=Fri. If the previous date was Friday, add 3 days to get Monday; otherwise add 2 days.

That fixes the B36 error and prevents similar mistakes going forward.
