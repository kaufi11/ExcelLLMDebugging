2) Wrong average range for Pract.3 (section 2)

3) Inconsistent grade cutoffs (>= vs >) and odd thresholds
- Problem: K6:K25 use >=89.9% etc., while K36:K55 and K26/K56 use >89.9% etc. This inconsistency can misclassify students whose percent equals the boundary (for example exactly 89.9%). Also 89.9% is an unusual boundary — maybe you intended 90%.
- Fix: Pick one consistent rule and apply to all grade cells. Example (recommended, tidy and clear):
  =IF(J6>=0.90,"A",IF(J6>=0.80,"B",IF(J6>=0.70,"C",IF(J6>=0.60,"D","F"))))
  Copy that formula down for all K cells (K6:K55) and use the same pattern for K26 and K56. If you really want 89.9% as the cutoff use >=89.9% consistently.

4) Class % may be calculated without Attendance/Participation
- Problem: J6 (and J36, etc.) = I6/400 uses only the pract. total (D:G). If you want Class % to include Att./Partcip. (H), the formula is wrong.
- Fix (if class % should include attendance): change J6 to =(I6+H6)/(I3+H3) or explicitly =(I6+H6)/450 and copy down. Also change J26/J56 similarly to use group means: =(I26+H26)/450.

5) (Minor) Leading zeros in SS#s
- Problem: values like 0915 will lose the leading zero unless the cell is text-formatted.
- Fix: Format the SS# column as Text or enter as '0915 (leading apostrophe) to preserve zeros.
