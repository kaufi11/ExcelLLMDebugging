1) Wrong sign for load-reactance (Imag ZL) — cell B26
- Problem: B26 = B6*1000/B21*SIN(B22*PI()/180) produces a negative imaginary Z for a lagging load. That is incorrect: for a lagging load the impedance angle θ = V∠0 − I∠(negative) = −B22 (positive), so the imaginary part should be sin(−B22) = −sin(B22).
- Fix: change B26 to either
  = -B6*1000/B21*SIN(B22*PI()/180)
  or
  = B6*1000/B21*SIN(-B22*PI()/180)
- Why: this yields the correct positive reactive (inductive) sign for a lagging load.

3) (Not an error but a clarity / reliability tip) ATAN2 argument order and conversion to degrees
- Issue: Excel's ATAN2 argument order and the *180/PI() conversion can be confusing. The sheet uses ATAN2(real, imag)*180/PI() consistently — this matches the sheet's convention and appears to be working. However it's easy to get wrong or misread later.
- Recommendation: replace usages like ATAN2(G,H)*180/PI() with DEGREES(ATAN2(G,H)) for readability, or if you prefer the usual math order (atan2(imag,real)) use DEGREES(ATAN2(H,G)). Example:
  =DEGREES(ATAN2(G,H))  (same as your current formula)
  OR to use the conventional atan2(imag,real):
  =DEGREES(ATAN2(H,G))
- Why: DEGREES() makes intent clearer and prevents unit-conversion mistakes; being explicit about the argument order avoids confusion later.
