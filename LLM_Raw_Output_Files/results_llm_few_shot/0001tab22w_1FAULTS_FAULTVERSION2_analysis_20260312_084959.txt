1) Wrong operator (typing) in AC22:
   - Cell AC22 has =SUM(100-(S22-U22-W22-Y22-AA22)) — it uses minus signs instead of plus signs. This gives a wrong "Other" percent.

2) Misplaced/extra data in some rows (data alignment error):
   - Several rows have stray numbers in the columns between the header columns (e.g. F18, H18, F19, H19, F20, H20, Z22, etc.). Those extra entries shift or duplicate values and will make the category percentages and "Other" calculations wrong.

3) Fragile formulas and text values:
   - Many "Other" cells use =SUM(100-(...)). SUM around a single expression is harmless but confusing. More importantly, if any referenced cell contains text like "n/appl" the formula returns #VALUE.
   - Totals that add cells will also break if non-numeric text is in any referenced cell.
