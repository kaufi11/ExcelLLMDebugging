Summary of workbook structure and contents (concise)

Sheets
- Intro
  - Title/description of DUCK94 experiment (A1–A4). Explains data are grouped: August, October, Other.
  - Table header in row 6: "Experiment #, PI Code" (A6), "Experiment Title" (B6), "Investigators" (C6), "Time period" (D6).
  - Rows A7–D11 list experiments: 2 (RHB) — Bennett/Church (June, Sept), 21 (FRF) — FRF (June–November), 29 (DOZR) — Werner/Elgar (June, Sept).

- RHBJune
  - Header block: EXPERIMENT NO (C1=2), PI CODE (E1=RHB), MONTH (G1=June), title text in A3–A4, PI/Co-PI names in row 6.
  - Main table starts row 8 with columns: Gauge ID (A), From (B), To (C), Description (D), Rate (E), Length (F), Interval (G), FRF Cross Shore (H), FRF Longshore (I), Water Depth (J), Height above bottom (K), Gauge Depth (L), Data Type (M), Format (N), Additional Information (O).
  - Sample rows A10–A17 list piezometer sensors (PA0, PA1, PD0..PD5), sampling 10Hz, record length 4–6 hrs, FRF location "SIS Pier", Data = Sed./Pressure, Format = Digital.
  - Two question cells present asking whether there are 5 or 6 sensors collecting pore pressure (D19:D20).

- RHBSept
  - Same structure as RHBJune with MONTH = September and similar table of PA/PD sensors (rows A9–A16), rate 10Hz, 4–6 hrs, Data = Time Series, Format = Digital.
  - Same question about number of sensors in D18:D19.

- FRF
  - Header: EXPERIMENT NO = 21, PI CODE = FRF, LONG (E1 text).
  - Table header row 3 with similar columns: ID, Description, Rate, Length, Interval, Cross Shore (F), Longshore (G), Depth (H), Bottom (I), Gauge Depth (J), Type (K), Format (L), Info (M).
  - Many sensor entries (rows 5–30): sensor IDs (111,121,131,...,511,519,529,601...), sensor types include Senso‑Metrics Pressure Gauge, Paroscientific, MMI Current Meters, Baylor Wave Gauge, Rain Gauge, Atmospheric Pressure, etc.
  - Key values: sampling rates typically 2 Hz, record length ~10240 sec, interval "every 3 hrs." Coordinates in columns F/G and depths in H/J. Data type mostly "Time Series", Format mostly "Digital".
  - Column I contains formulas intended to compute H - J (height above bottom). Examples: I5..I19 show =SUM(H5-J5) etc. One anomalous formula at I20 =SUM(H20-85) (inconsistent reference) and some rows have "N/A" for coordinates/depths.

- DOZRJune
  - Header: EXPERIMENT NO = 29, PI CODE = DOZR, MONTH = June; title and PI/Co‑PI rows present.
  - Table similar to others with sensor IDs p01..p08 (rows A9–A16): Setra capacitance pressure gauges, 2 Hz, continuous, FRF coords (H) = 100–150, Longshore ~880–900, small water depths J and heights K/L given.
  - Additional video entries (rows B17–B20): FRF b/w video, Sony Handycam; formats SVHS, Super 8, some listed as TimeSeries/Digital.

- DOZRSept
  - Same layout as DOZRJune but date ranges From/To are 5–19, same sensors p01..p08 with similar coords and properties; includes video entries similarly.

Key formulas and calculations
- FRF sheet column I appears to compute height above bottom as (Depth H) - (Bottom J) but uses =SUM(H#-J#) instead of a direct subtraction. This yields the difference but is an odd use of SUM.
- I20 uses =SUM(H20-85) which is inconsistent (subtracts literal 85 rather than J20).
- One FRF cell I14..I19 etc are consistent with =SUM(H#-J#). Several rows have N/A values in coordinate/depth columns.

Common columns (present across experiment tables)
- ID, From, To (dates), Description, Sampling Rate, Record Length, Sampling Interval, FRF Coordinates (Cross‑Shore, Longshore), Water Depth, Height Above Bottom, Gauge Depth, Data Type, Format, Additional Info.

Data types & formats
- Most sensors: Time Series, Format = Digital.
- Video records: Video / SVHS Tape or Super 8 (analog).
- Sampling rates: RHB piezometers 10 Hz; FRF and DOZR pressure/wave sensors commonly 2 Hz; some video 30 Hz.

Notable values and issues
- Experiment numbers: 2 = RHB (pore pressure), 21 = FRF (measurements program), 29 = DOZR (swash zone).
- Repeated question in RHB sheets: "Are there 5 or 6 sensors collecting Pore Pressure data ??" (suggests metadata uncertainty).
- Inconsistent/odd formulas in FRF column I (use of SUM for subtraction; one literal 85).
- Several cells contain "?" for sampling interval or record length, and many N/A coordinates—metadata incomplete in places.

Overall layout
- Each experiment sheet has a small header block (experiment no, PI code, month, PI names), followed by a consistent tabular register of instruments with columns for timing, location (FRF cross‑shore/longshore), instruments specs (rate, length, type), data type/format, and notes. Intro sheet summarizes experiments and links to per‑experiment tabs.
