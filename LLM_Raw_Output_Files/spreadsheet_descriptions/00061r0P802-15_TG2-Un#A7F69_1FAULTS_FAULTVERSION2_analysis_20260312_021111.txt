Summary of workbook

Sheets
- Profs: main contact directory (columns A–O).  
- URLs: single-column list of resource names and web links.

Profs (structure & content)
- Header row (row 1) labels: A=Number, B=Name, C=Position, D=University, E=Email address, F=Suggested by, G=Personal or Related Web Page, H=Address, (I has no header but is used occasionally for a second address line), J=City, K=S/P (state/province), L=CNTY (country), M=Zip, N=Phone, O=FAX.
- Rows 2–~63 are individual entries (academics, students, researchers) with name, affiliation and contact details. Example key entries: A2 = 1 / B2 Roy D. Yates (Rutgers) with email ryates@winlab.rutgers.edu; row 28 Andrew T. Campbell with full mailing address, phone and fax; row 34 Henning Schulzrinne (Columbia); several MIT and Columbia contacts; many entries have "Suggested by" populated (frequently Ian Gifford or Kevin Marquess).
- Column A numbering: mostly auto-increment formulas (A3 = A2+1, A4 = A3+1, …). One anomaly: A37 uses =A36-1 (a decrement), which will break the expected sequential numbering. Aside from that, numbering is formula-based rather than typed values.
- Column G contains personal/web pages for many people (e.g., G28, G29, G30, G35, G40, G41, etc.). Columns H–M are used for postal address components; I is used inconsistently as a secondary address column though it lacks a header.
- Data quality notes: some email fields are unknown/question-marked ("?"); some duplicates/inconsistencies (e.g., Joe Paradiso appears more than once with the same email); spelling inconsistencies (e.g., "Virgina" vs "Virginia"); phone/fax are sparsely populated.
- No other calculated fields or macros appear—sheet is a flat contact list with light use of address subfields.

URLs (structure & content)
- Single column (A) with descriptive labels and links on separate rows. Items include major project and group URLs such as WPI, MIT LCS, MIT Media Lab, WIND, Congestion Manager, Columbia CellularIP, IETF Mobile IP, CMU Monarch, MosquitoNet, MEDIA (KTH), INRIA hierarchical MIPv6, NUS Mobile IP, etc.
- Layout is alternating label/URL in many cases (label on one row, link on the next).

Overall layout & purpose
- A flat, tabular directory of professionals and students in networking/wireless with contact details and related web links. Column A uses simple +1 formulas for record IDs (one formula anomaly). Some address fields span multiple columns (H–M) and column I is used without a header. The URLs sheet is a companion list of project/institution links. Data contains some missing values and a few inconsistencies but no complex formulas beyond the sequential numbering.
