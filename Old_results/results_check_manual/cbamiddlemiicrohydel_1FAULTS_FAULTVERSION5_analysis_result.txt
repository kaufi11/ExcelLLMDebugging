Detailed findings, with recommended fixes

1) Discount-factor inconsistency (critical)
- Cells involved:
  - Costs: row 19 (G19:AA19). Formulas mostly use successive multiplication by (1/1.12) except I19:
    - H19 = =G19*(1/1.12) — correct (0.892857142857143)
    - I19 = =H19*(249/249.2492) — this is anomalous (gives ~0.891964). It does NOT apply the (1/1.12) discount and breaks the chain.
    - J19 onward resume multiplying previous cell by (1/1.12).
  - Benefits: row 31 (G31:AA31) uses a clean geometric discount: H31 = G31*(1/1.12), I31 = H31*(1/1.12), etc. (this is consistent).

- Why it's a problem: Costs and Benefits should use the same discount timing convention. The wrong I19 keeps the year-2002 costs almost undiscouted relative to the correct 1/(1.12^2) factor, so PV(costs) is inflated for that year. That affects:
  - I20 (=I18*I19) and the total present value of costs C20 (=SUM(G20:AA20) and C36/G36).
  - B/C ratio and NPV (C38/C39 etc) are therefore distorted.

- Suggested correction:
  - Replace I19 formula with =H19*(1/1.12) (or better use relative references: for each cell from H19 onward use =previous_cell/(1+$rate$) or calculate via =1/(1+$rate$)^(n) with a single rate cell).
  - Alternatively make all DF rows compute using a single cell for rate and POWER function (e.g., for column corresponding to year t: =1/(1+$B$rate)^(t - baseYear)) so costs and benefits can both reference same DF.

- Approximate effect if corrected:
  - Current I19 ≈ 0.891964; correct 1/(1.12^2) ≈ 0.7971938776. Using the correct factor reduces the present value of year-2002 costs I20 from 186,682 → ~166,892 (approx -19,790). Thus total PV of costs (C20) would fall by ~19.8k and NPV would increase by similar amount. (This is a quick approximation — recompute after you fix the DF row.)

The labels say "cost of environmental clean up in year 2020" and "scrap value (2% of total cost of the project)". If "total cost of the project" or the base for scrap/cleanup is meant to be the full initial construction cost (sum of contributions) you should ensure the formula references the intended cells:
