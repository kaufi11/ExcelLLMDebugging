Overview
- Workbook contains three identical class sections: sheets SecA, SecB, SecC.
- Purpose: gradebook for CS 110 (Introduction to Computers and Applications) with per-student scores, assignment columns, tests, extras and automatic totals.

Top area (metadata & controls)
- Row 1: metadata labels (Year, Semester, School, Subject, Course, Section, TA, Instructor, Coordinator).
- Row 2: values: Year=2004, Semester=Spring, School=01, Subject=198, Course=110; Coordinator in I2 = "Goelz".
- A4: course title "CS 110 - INTRODUCTION TO COMPUTERS AND APPLICATIONS".
- A5 / B5: "Grades as of" and B5 has =NOW() to show timestamp.
- A8/A9: keyboard reminders for Export (Shift-Ctrl-X) and Print (Shift-Ctrl-P).
- A10/B10: Upload target (B10 = https://lists.fas.rutgers.edu/cgi-bin/gradebook/gb.pl).

Grade weighting / legend (row 9–10)
- Column headers at row 9: Total Points (I9), Assignment Points (J9), Test 1 (K9), Test 2 (L9), Final Exam (M9), Late, Free Late, TA Extra Credit (P9), Software Extra Credit (Q9), then individual assignment/tool columns R9:AE9 (WWW, Email, HomePage, Excel, PowerPoint, Database, Paint, Basic 2..Basic 9 etc.).
- Row 10 contains weight/scale values:
  - I10 = SUM(J10:M10) (total of assignment + test + final weights)
  - J10 = SUM(R10:AF10) (sums assignment weights across R:AF — note AF is outside the R:AE range used elsewhere)
  - K10 = 100, L10 = 100, M10 = 150
  - P10 = 10, Q10 = 10 (extra credit values)
  - R10:AE10 contains assignment weights: 10,10,10,15,15,15,10,5,5,10,10,10,10,15 (per the listing).

Student table layout (starts row 13 header; rows 14+ student rows)
- Row 13 columns: Last Name (A13), First Name (B13), SSN (C13), CID (D13), NetID (E13), Comments (F13), Final Grade (G13), Attendance (H13), Total Points (I13), Assignment Points (J13), Test 1 (K13), Test 2 (L13), Final Exam (M13), Late/Free Late (N/O), P/Q extras, R:AE individual assignment scores.
- Per-student formulas (rows 14 onward):
  - J(row) = SUM(R:AE) — sums the student's assignment columns (e.g., J14 = SUM(R14:AE14)).
  - I(row) = MIN(500, SUM(J: M, P:Q)) — caps the student's total at 500 and sums Assignment points (J), Test1-2 and Final (K:M), and extra credits (P:Q). Example: I14 = MIN(500, SUM(J14:M14, P14:Q14)).
- This pattern repeats for rows 14 through at least 50 (SecC includes row 50); most J rows use SUM(Rx:AE x). Exception noted: J19 is listed as =2 (fixed value) rather than a SUM formula.

Notes / potential issues to watch
- J10 uses SUM(R10:AF10) while all student assignment sums use R:AE and assignment weights are defined through AE10. AF may be empty or a mismatch — verify intended range.
- J19 = 2 is anomalous relative to the other J-row formulas — check if intentional (e.g., a special-case assignment or data entry).
- Total cap enforced by MIN(500, …) in I-column for all student rows limits maximum recorded total to 500.

Summary
- Three nearly identical section sheets with metadata, a weight/legend row, an upload URL, per-assignment weight values, and a student roster (name/IDs) where assignment sums (J), tests/final (K:M) and extras (P:Q) feed a capped Total Points (I) calculated per student. Key fixed weights: Test1=100, Test2=100, Final=150; several assignment weights in R:AE and extras P/Q = 10 each.
