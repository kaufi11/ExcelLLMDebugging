- Section identifiers: C5 = Sec. 8006, C35 = Sec. 8004.
- Each section contains 20 student rows (20 rows of data per block).

Formulas (representative)
- Per-student total: I6:I55 = =SUM(Dn:Gn)
- Class percent: Jn = =In/400 (block 1 uses I3; block 2 uses I33)
- Letter grade (block 1 example): =IF(Jn>=89.9%,"A",IF(Jn>=79.9%,"B",IF(Jn>=69.9%,"C",IF(Jn>=59.9%,"D","F"))))

Notable inconsistencies / anomalies to review
- Grade-threshold consistency: Block 1 grade formula uses >= thresholds (>=89.9% etc.); block 2 uses > thresholds (>= vs > inconsistency).
- MEAN formulas: F56 uses =AVERAGE(F37:F55) (starts at row 37) instead of F36:F55 — likely a typo.
- Some student rows have missing scores (e.g., G55 appears blank) — SUM will treat blanks as zero; verify intended data.
- Duplicate headers and professor cells are placed in different columns (Professor appears in J1 for block 1 and K31 for block 2) — minor layout inconsistency.
- Totals and class % exclude attendance (I is explicitly “Total (w/o A/P)”), so attendance is not factored into the Class % as currently calculated.

Summary
- Two 20-student sections with identical grading layout: four practicals (400 points total), attendance recorded separately (50 points). Per-student totals and class percentages are calculated from the practicals only; letter grades assigned by nested IFs. Cleanups recommended: unify IF operators, correct average range typo, and verify any missing cell entries or intended inclusion of attendance in percentages.
