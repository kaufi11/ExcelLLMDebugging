Summary of workbook structure and contents

Sheets present
- Bulk compost (data rows A2:I22 shown)
- Treatments (data rows A2:I22 shown)
- Sheet3 (empty / not described)

Common layout (both main sheets)
- Column A: Day (integer day number)
- Column B: hour (decimal hour of day)
- Column C: start — formula converting Day+hour into a continuous "days from" timestamp:
  - Bulk compost: =A# + (B#/24) - 1.48  (applied in C4:C22 except a typo at C19)
  - Treatments:   =A# + (B#/24) - 1.71  (applied in C3:C22)
  - Purpose: create a fractional-day time variable for plotting/analysis
- Remaining columns: temperature measurements
  - Bulk compost: D = Compost, E = Ambient
  - Treatments: D = Group I, E = Group II, F = Group III, G = Group IV, H = Group V, I = Ambient

Key values / ranges
- Bulk compost:
  - Compost temps (col D): range ≈ 68.8 to 103.5 (peak 103.5 at row 6)
  - Ambient (col E): range ≈ 70.0 to 73.6
- Treatments:
  - Group I–V (cols D–H): typical values ~70–77; Group I peaks ~77.2; groups vary by row
  - Ambient (col I): range ≈ 70.0 to 74.7 (max 74.7 at row 15)

Notable issues / suggestions
- Typo in Bulk compost C19: formula uses =A19+(B19/783)-1.48 — should almost certainly be B19/24.
- The two sheets use different subtraction offsets in the start formula (-1.48 vs -1.71) — confirm these offsets are intentional (they shift the origin of the fractional-day timestamp).
- Several days have multiple time points (duplicate Day values with different hours) — suitable for time-series plotting.

Overall purpose
- Time-series temperature logging: both sheets convert day+hour to a continuous time variable and record compost/treatment temperatures plus ambient for comparison across time and treatment groups.
