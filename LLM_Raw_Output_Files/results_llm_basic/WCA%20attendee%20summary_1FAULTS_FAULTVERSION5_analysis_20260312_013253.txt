Findings
- Error: Cell B20 uses =COUNT(B12:B19). That returns 8 (the number of rows with numeric entries) but the intended total of "would like to attend" is 66.
- Cause: COUNT counts numeric cells; it does not add their values. You need SUM to add up the counts in those rows.

Quick fix
- Replace the formula in B20 with:
  =SUM(B12:B19)

Why this fixes it
- SUM adds the values (0+4+1+8+14+12+15+12 = 66). COUNT only returns how many numeric cells are present (8), so it produced the wrong total.

Extra checks / recommendations
- Verify the B12:B19 cells are numeric (not text). If any look like numbers but SUM returns an unexpected result, convert text-numbers to real numbers (select column → Data → Text to Columns the default → Finish, or multiply by 1).
- The other totals you showed (C20 =SUM(C12:C19), D20 =SUM(D12:D19), and E30/F30/G30 =SUM(...)) are correct — no change needed there.
- If you need totals that update correctly when rows are filtered, use =SUBTOTAL(9,B12:B19) instead of SUM.
