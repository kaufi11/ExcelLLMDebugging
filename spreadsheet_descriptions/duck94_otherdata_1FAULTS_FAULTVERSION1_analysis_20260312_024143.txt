Overview
- Workbook organizes the DUCK94 nearshore experiment inventory. Each experiment/group has its own sheet (Intro, RHBJune, RHBSept, FRF, DOZRJune, DOZRSept).
- Layout is consistent: top rows hold experiment metadata (Experiment No., PI code, month, title, PI/Co-PI), then a tabular instrument inventory with repeated column headers (Gauge ID, Date/From/To, Description, Sampling Rate, Record Length, Interval, FRF Coordinates, water depth/height/above-bottom, data type, format, notes).

Sheet-by-sheet summary

1) Intro
- Rows A1–A4: title and project description.
- A6:D6 define the table headers for the summary index.
- Rows A7–D11: high-level index of experiments showing Experiment # & PI code, Title, Investigators, and Time period (e.g., 2 = RHB, 21 = FRF, 29 = DOZR).

2) RHBJune (Experiment 2 — June)
- Purpose/PI metadata in rows 1–6.
- Instrument table (starts row 8): columns include ID (A), From/To dates (B–C), Description (D), Sampling rate (E; e.g., 10Hz), Record length (F; "4–6 hrs."), Interval (G), FRF coordinates Cross-Shore (H) / Longshore (I), Depth/Above bottom (J–K–L), Type (M; e.g., Sed.Pore Press./Pressure), Format (N).
- Repeated instrument rows for PA0, PA1, PD0–PD5 (piezometers). Many fields marked "?" or "Varies".
- Note text questions in D19–D20: "Are there 5 or 6 sensors collecting Pore Pressure data ??"

3) RHBSept (Experiment 2 — September)
- Same structure as RHBJune with similar instrument list (PA0/PA1/PD0–PD5).
- Sampling rate 10Hz; record length 4–6 hrs.; Data type listed as "Time Series" and Format = Digital.
- Same sensor-count question appears (D18–D19).

4) FRF (Experiment 21 — FRF Measurements Program)
- Metadata rows 1–2 show EXPERIMENT NO = 21 and PI CODE = FRF, LONG.
- Instrument table header slightly different but same core fields: ID, Description, Rate (C), Length (D; many = 10240 sec), Interval (E; e.g., every 3 hrs.), FRF Coordinates F/G (cross-shore, longshore), Depth (H), Bottom (J), Height off (I), Type/Format/Info.
- Common entries: many Senso-Metrics and Paroscientific pressure gauges (IDs 111,121,...,251 etc.), typical sampling = 2 Hz, record length = 10240 sec, Type = Time Series, Format = Digital.
- Key coordinate values: F & G values present for each sensor (e.g., F5=914.45, G5=825.72).
- Column I contains formula placeholders like "FORMULA =SUM(H5-J5)" (intended to compute H - J); one cell (I20) contains "=COUNT(H20-J20)" — inconsistent formula usage and some appear to be text rather than active formulas.
- J column (bottom) and H column (depth) used in these formulas; many rows have J filled (e.g., J5=7.504).
- Some sensors have N/A in depth/height fields (e.g., wind, rain, air temp).

5) DOZRJune (Experiment 29 — June)
- Metadata at top, PI Brad Werner, Co-PI Steve Elgar.
- Instrument table lists pressure gauges p01–p08 (Setra capacitance) with Sampling = 2Hz, Interval = Continuous, FRF coordinates cross-shore H and longshore I (e.g., H9=100, I9=900), water depths/above-bottom in J–L (typically 0, 0.2, 0.4, 0.6).
- Additional video entries (FRF b/w video, Sony Handycam) with 30Hz and format notes (SVHS Tape, Super 8).
- Several date/timestamps present (e.g., To column C for p01 = 1994-07-04 00:00:00).

6) DOZRSept (Experiment 29 — September)
- Same structure and instruments as DOZRJune but From/To ranges are numeric (e.g., From = 5, To = 19).
- p01–p08 repeated with same sampling, locations and water levels; video entries repeated.

Key formulas, values, and anomalies
- Repeated intended calculation in FRF sheet: I = H - J (shown as "FORMULA =SUM(H#-J#)"); likely intended to compute height off bottom or similar. Many of these appear as text labels rather than executed formulas.
- One anomalous formula in FRF I20 uses =COUNT(H20-J20) which is likely incorrect.
- Typical sampling rates: pressure gauges = 2 Hz (FRF/DOZR), piezometers = 10 Hz (RHB).
- Typical record lengths: 10240 sec for FRF instruments; 4–6 hrs. for RHB piezometers.
- Instrument types: Time Series, Video, Pressure (Sed. Pore Press., Paroscientific, Senso-Metrics), Current Meters.
- Coordinate system: FRF coordinates (columns F / G) appear to be cross-shore and longshore meter positions for each sensor.

Overall structure and consistency
- One tab per experiment, consistent header + instrument table format.
- Columns are consistent across sheets with minor name variations; fields capture ID, timing, description, sampling specs, spatial coordinates, depth/height details, data type/format, and notes.
- Some cells contain placeholders, "?", or textual "FORMULA" labels — review needed to convert those to working formulas and to standardize calculations (e.g., replace textual "FORMULA =SUM(H-J)" with actual =H#-J#).
