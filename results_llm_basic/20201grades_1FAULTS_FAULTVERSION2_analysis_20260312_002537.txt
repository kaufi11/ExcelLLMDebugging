1) Wrong SUM range on one student total (excludes a column)

2) SUMs returning errors because some score cells contain errors/text (e.g. "#", "x", "X", "+ X")
- Locations (examples): BR6 = "#", BR7 = "x  #", BL15 = "+ X", AZ13 = "x", etc. Any column in BN:BR (or other numeric blocks) containing an error symbol will make formulas like BS6 =SUM(BN6:BR6) evaluate to an error.
- Problem: a literal error value (like # or an Excel error) inside a SUM range causes the whole SUM to error. Text like "x" is ignored by SUM, but real error values are not.
- Fix options (pick one):
  a) Replace error markers with 0 or blank (preferred if those represent missing/0 points). This is the cleanest fix.
  b) If you must keep markers, change the SUM formulas to ignore errors, e.g. replace
     =SUM(BN6:BR6)
     with
     =SUM(IFERROR(BN6:BR6,0))
     (In older Excel you may need to enter as an array formula with Ctrl+Shift+Enter; in modern Excel IFERROR on ranges works.)
  c) Or use AGGREGATE/aggregate pattern or clean data with a helper column: =SUMPRODUCT(N(BN6:BR6)) with N converts text to 0 but will still fail on error values; so IFERROR is safer.
- Why: SUM won't return a number if any cell in the range holds an error; wrapping with IFERROR ensures errors count as 0.
