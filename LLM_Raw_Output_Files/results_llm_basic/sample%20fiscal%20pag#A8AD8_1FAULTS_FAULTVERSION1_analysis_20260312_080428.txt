Summary of the error
- On sheet 1198, cell H25 (Total Revenues for 2003-04) currently contains =COUNT(H9:H24). That returns the count of numeric cells (e.g. 16), not the sum of revenue amounts. Because H25 is used in later carry‑forward and resource formulas, this mis‑formula produces wrong totals throughout the Summary.

Why it's wrong (short)
- COUNT returns how many numeric cells exist. You want the numeric total of those cells — SUM.

Fix (exact steps)
1. Open sheet "1198".
2. Select cell H25.
3. Replace the formula =COUNT(H9:H24) with =SUM(H9:H24).
4. Press Enter and recalculate (F9 or File → Calculate).

What will change / why it fixes things
- H25 will now contain the actual total revenue for 2003-04 instead of the number of entries. Any downstream formulas that use H25 (carry forwards, totals, ending balance) will recalculate correctly.

Extra checks (recommended, quick)
- Search the workbook for any other totals that use COUNT where SUM should be used (Find for "COUNT(").
- Verify 3‑D SUM formulas (like =SUM('1003':'1856'!D6:D6)) include the intended sheets in the workbook sheet order.
- If totals still look odd after the fix, recalc and check intermediate totals (each fund's D25/H25 etc) to isolate other similar typos.
