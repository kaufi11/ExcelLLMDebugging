1) Wrong formula / typo in FRF sheet (biggest concrete error)
- Problem: Column I is supposed to be H − J (height off / difference), most cells use =SUM(Hx-Jx) (odd but works). Row 20 is wrong: I20 contains =SUM(H20-251) — that subtracts the number 251 instead of the gauge depth in J20 (typo).
- Why it’s wrong: it produces incorrect values for that row and is inconsistent with other rows.
- Fix: Replace those formulas with a simple subtraction referencing the correct cells. Example:
  - I5 -> =H5-J5 (not =SUM(H5-J5))
  - I20 -> =H20-J20
- How to fix quickly:
  - Select column I, use Find & Replace to remove the SUM wrapper (Find: =SUM(   Replace: = ), and then remove the trailing ) if needed; then manually fix any remaining bad references (I20).

2) Ambiguous / inconsistent count of pore‑pressure sensors in RHB sheets
- Problem: The sheet asks “Are there 5 or 6 sensors…?” but the table lists PA0, PA1 and PD0–PD5. Depending on what the question intends (count only PD sensors, or count all piezometers/pore-pressure transducers), the answer differs.
- Clarification & fix:
  - If counting PD# only: PD0–PD5 = 6 sensors.
  - If counting all pore‑pressure instruments (PA and PD): PA0 + PA1 + PD0–PD5 = 8 sensors.
  - Action: decide which group the question targets and update the cell text to the exact count (e.g., “PD sensors = 6; total pore‑pressure sensors = 8”). Better: add a small column “Sensor Type / Category” and a boolean “Collecting pore pressure?” so counts can be computed by a COUNTIF.
- How to implement: add column “Type (PorePressure=Yes/No)” and use =COUNTIF(range,"Yes") to get exact numbers.

3) Date / month inconsistency in DOZRJune (minor data quality issue)
- Problem: DOZRJune sheet header says MONTH = June but some To dates are 1994‑07‑04 (July 4). Also some date values are mixed formats (day numbers vs full dates).
- Fix:
