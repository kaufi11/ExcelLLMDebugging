Overview — workbook contains four sheets tracking student scores and final grades:

1) Sheet: Homework
- Layout: Column A = Pseudonym (rows 4:42). Columns B:K = HW1..HW10 scores. Column L = Total per student (formula: mostly =SUM(B#:K#)). Column N = Paper score (separate points column). Column P = status/notes (HC, P, NG, x).
- Key formulas: L4:L42 use SUM across B:K (e.g., L4 =SUM(B4:K4)). Class averages: L44 =AVERAGE(L4:L42) and N44 =AVERAGE(N4:N42).
- Special values/flags: HC = Honor Code violation, P = pending HC review, NG = not graded, x = not received. Text flags in numeric columns can make some totals non-standard (SUM ignores text). A few anomalies (e.g., L19 shows "FORMULA =2" instead of SUM).
- Representative values: many totals in N column show paper scores (range ~120–200); Homework totals L vary per student.

2) Sheet: Mid-term
- Layout: Column A = Pseudonym. Columns B:K = question/item columns (B sometimes used for an extra item). Row 5 lists total possible points per item. Column L = student Total (=SUM(C#:K#) or similar). Columns N and O = subtotals (N = SUM(C:G) for first part, O = SUM(H:K) for second part).
- Key formulas: per-student totals use SUM across question columns; N# and O# compute Q1/Q2 subtotals. Row 46 (Average) uses AVERAGE for each question and totals (e.g., C46 =AVERAGE(C6:C44), L46 =AVERAGE(L6:L44), N46/O46 likewise).
- Notes: consistent use of SUM for row totals and two subtotal ranges; per-question maximums in row 5 (summing to L5 total possible = SUM(C5:K5) = 50 for each subtotal column N5/O5).

3) Sheet: Final
- Layout: Column A = Pseudonym. Columns B:G = Q1..Q6. Column I = TOTAL per student (formula =SUM(B#:G#) in most rows).
- Key formulas: I4:I42 =SUM(B#:G#). Row 44 holds question averages (B44..G44 =AVERAGE of B4:B42 etc; I44 =AVERAGE(I4:I42)).
- Notes: one cell (I7) references SUM(B7:H7) though H is not part of defined questions — occasional inconsistent ranges appear.

4) Sheet: Final Grades
- Layout: Column A = Pseudonym. B = Papers (from Homework N), C = Mid-term (subtotal column N or O depending on mapping), D = Term (term total column D from Homework sheet?), E = Final (Final sheet TOTAL). Column F = Total final points (formula F# =SUM(B#:E#)). Column G = Note (HC, P, etc.). Columns I–K contain grade labels and numeric grade-range cutoffs (K45:K83 = cutoff scores).
- Key formulas: F# uses =SUM(B#:E#) to compute overall score. Top of sheet shows revision note and grade-range mapping (I/J/K area).
- Special handling: Some students marked HC or P are flagged in Note column; some grade components are "HC" or "P" text which will affect numeric sums.

General remarks / data-quality issues
- Primary formulas used: SUM for row totals and subtotals; AVERAGE for per-question/class averages.
- Several cells contain text flags (HC, P, NG, x) in columns otherwise numeric; SUM ignores text, which can mask missing/flagged scores. A few rows have non-standard formulas or values (e.g., L19 =2).
- Grade cutoffs / ranges are stored as numeric list in Final Grades (column K) and letter-grade labels in column I/J for reference.
