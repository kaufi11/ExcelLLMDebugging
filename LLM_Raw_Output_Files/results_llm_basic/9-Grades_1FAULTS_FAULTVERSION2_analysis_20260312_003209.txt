Key errors (with examples)
1) Inconsistent SUM ranges in Totals
- Example: University Total row B31 =SUM(B9:B29) but D31 =SUM(D9:D30) (and E31 =SUM(E9:E30), etc.).
- Effect: some columns include an extra row or omit a row, so column totals do not add to 1 (or are inconsistent across columns).

2) Wrong function used instead of SUM
- Example: O31 =COUNT(O9:O30). COUNT counts non‑empty cells, not sums percentages.
- Effect: O31 shows number of entries instead of the percent total.

3) Hard‑coded total instead of formula
- Example: M31 is set to 1 (entered manually) while adjacent totals are formulas.
- Effect: M31 may hide errors in the underlying data and is not dynamically correct when inputs change.

4) Blank cells or missing zeros and placeholder values
- Several grade cells are blank (or have weird placeholders like “N/A” or single “4” in some GPA cells). Blank cells are treated as zero by SUM but blanks make it hard to spot missing data; “N/A” text or other non‑numeric placeholders can break sums or averages.
- Effect: totals may be less than 1 or averages wrong if non‑numeric placeholders are present.

5) Division formulas that are fragile / inconsistent references
- Example on Sheet1: D1 =+C1/+C24 (repeated for many rows). This uses a relative reference for the denominator and an unnecessary plus sign; if C24 changes position or rows are inserted the link can break.
- Effect: percentages calculated from these may be wrong if the referenced cell moves or if C24 is not what you expect.
