Other issues, inconsistencies and recommended fixes
- Sheet "Q1" (all D4..G4, D5..G5, …)
  - The discount formulas in column D use $D$1 as the exponent. In the sheet header D1 is blank (only E1=1, F1=2, G1=3 are populated). Relying on a blank cell being treated as 0 in exponent works in Excel (blank coerces to 0) but is fragile and not transparent.
  - Recommendation: explicitly set D1 = 0 (Period 0) or replace references to $D$1 with an explicit 0 if you intend period 0. This makes the meaning explicit and avoids hidden behavior if someone clears the cell.
- Sheet "Q1" (discount-rate references)
  - Discount formulas use $J4 (for example) rather than $J$4; the column is fixed but the row is relative (so when copied down they reference the J value on the same row). This appears intentional, but it's a subtle mixed-anchoring style that can cause confusion if rows are moved or formulas copied incorrectly.
  - Recommendation: if the rate in column J is intended to be fixed per row (which it is), keep it as $J4 (relative row) — but document the structure or use $J$4 in each row explicitly to avoid accidental copying mistakes.
- Sheet "Q1 a-b, graph" vs sheet "Q1"
  - There are two tables that calculate the same NPV curve; they use the same cash flows but different row orderings/indices. Values match mathematically but alignment of rates/rows differs between sheets. Not an error per se, but be aware when comparing rows between sheets.
- Sheet "Q1 c-d, additional info"
  - Cell G26: =IRR(B2:E2,22%) returns #N/A (this is shown in the sheet). This is not a spreadsheet error — IRR with that guess does not converge; Excel returns #N/A. If you want to show a meaningful result, use a different guess or use a root-finder on the polynomial.
- Sheet "Q2" and "Q2 WITH DEPRECIATION" — use of blank period cells
  - Several discount formulas use exponents like ^C6 where C6 is blank. As with Q1, Excel treats blank as zero. This works but is fragile/unclear. Recommendation: populate the period header cells C6..G6 explicitly (0,1,2,3) to make discount exponents explicit.
- Sheet "Q2" (tax loss carry-forward logic)
  - The carry-forward logic uses IF() tests and SUM ranges that include the carry-forward cell itself (e.g. E46 = IF(D47=0,SUM(D45:D47),0)). This works because the IF branch prevents circular evaluation in the current data, but it is conditional and could become circular if numbers change (e.g., if D45 >= 0 the formulas take different branches). Recommendation: simplify logic or document assumptions (that D45 < 0 for early years) to avoid accidental circular references later.
