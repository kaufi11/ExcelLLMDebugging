1) Hard‑coded BK261 (should be formula)
- Where: Sheet4, cell BK261 (row for "Creating High Performance Project Teams").
- Problem: BK261 contains the value 12 instead of the usual formula that counts participants for that class. All other BK rows use BKn = SUM(Fn:BGn). Hard‑coding breaks row and grand totals if you change participant entries.
- Fix: Replace the 12 with the formula:

- Why: BK is "Number of Participants Per Class" and must total the per‑company participant counts in columns F:BG for that row. Using the SUM keeps it dynamic.

- Problem: T121 contains the literal 5 rather than the pattern formula used in other "Total" rows (e.g. Tn = SUM(Trow*$E$row) or similar). This gives an incorrect contribution into the per‑company totals.

  =T120*$E$120

- Why: All "Total" cells in that block should compute per‑company cost for that class using the number-of-participants in column T for the class row times the class RISE cost (E column). A literal 5 will not update when participants change.
