Detailed issues and suggested fixes

1) Income sheet — Revenue total incorrectly computed (HIGH)
- Cells: Income!E12, F12, G12, … P12
- Current formula pattern: e.g. E12 = =SUM(E10:E10) (same for other months)
- Problem: This sums only the Online Lemonade Sales row (E10) and ignores Street Sales (E11). Total revenue is therefore understated by the Street Sales amount.
- Why it matters: Gross profit, net profit, cash flow and the balance sheet are all downstream of this total, so a core number is wrong.
- Suggested fix: Change each total to sum both revenue rows, e.g.
  - E12 = =SUM(E10:E11)
  - F12 = =SUM(F10:F11)
  - etc.

2) Balance Sheet — Liabilities row contains incorrect value that propagates (HIGH)
- Cell: Balance Sheet!F14 (and downstream G14…)
- Current: F14 = 6 (formula: "=6"), then G14 = =F14 + Cashflow!G9 which yields odd negatives
- Problem: F14 is a stray hard-coded "6" — looks incorrect; E14 was 900,000 but F14 should carry-forward or be computed consistently.
- Why it matters: Total liabilities and Equity+Liabilities become inconsistent and produce wrong balance-sheet totals in many years.
- Suggested fix: Replace F14 with consistent carry/roll-forward formula. Likely intended:
  - F14 = =E14 + Cashflow!F9
  (or if E14 already included cashflow for that period, set F14 = E14). Review design intent and correct.

3) Income sheet — Sugar price formulas use lemon inflation cell (MEDIUM)
- Cells: Income!F8, G8, H8 … P8
- Current formula: e.g. F8 = =E8*(1+$D$7)
- Problem: they use $D$7 (the lemon-price monthly inflation) instead of $D$8 (the sugar-price monthly inflation). Right now D7 and D8 happen to be the same numeric value, so results match, but it’s logically incorrect and will break if the two rates differ.
- Suggested fix: change the growth reference to $D$8:
  - F8 = =E8*(1+$D$8), etc.

4) Depreciation sheet — Depreciation "Original Value" reference wrong (MEDIUM)
- Cell: Depreciation!C5
- Current formula: ="='Balance Sheet'!C8" (this points to the label "Original Value" on Balance Sheet rather than the numeric value)
- Problem: C8 on Balance Sheet is a header label; the actual numeric original value is in Balance Sheet!D8.
- Suggested fix: change Depreciation!C5 to reference the value:
  - Depreciation!C5 = ='Balance Sheet'!D8

5) Balance Sheet — Lemon Inventory references missing starting cell (MEDIUM)
- Cells: Balance Sheet!C7 row: F7 = =E7, G7 = =F7, etc.
- Problem: E7 (the initial value) is not defined on the sheet. F7 references E7 which is blank -> Excel treats as 0, so every inventory cell is 0. Probably you intended to populate E7 or link it to an inventory calculation (e.g. Income!E6 or a derived inventory figure).
- Suggested fix: Put the correct starting inventory value in Balance Sheet!E7 or change F7 to link to the correct source (for example =Income!E6*cost-per-lemon or similar).

6) Depreciation sheet — Total Depreciation accumulation breaks after column K (LOW→MEDIUM)

8) Income sheet — Miscellaneous inconsistent formulas and patterns (LOW)
- Many places use redundant or odd-range sums such as =SUM(E10:E10) or chained references offset by varying columns (e.g. Advertising row: I25 = =F25, J25 = =G25, K25 = =H25, etc.). These are not necessarily errors (they return correct numbers), but are inconsistent and confusing — they make maintenance/error tracing harder.
- Suggestion: standardize propagation formulas (e.g., each column refers to the prior column or to a single input cell) and simplify SUM ranges to the intended rows.

9) Cashflow sheet — SUM ranges include empty rows (LOW)
