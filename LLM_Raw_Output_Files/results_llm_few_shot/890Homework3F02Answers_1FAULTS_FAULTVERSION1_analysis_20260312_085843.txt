2) Q2 (both "WITHOUT DEPRECIATION" and "WITH DEPRECIATION") — circular references in tax-loss carryforward logic
- Fault: e.g. D46 = IF(D47=0,SUM(D45:D47),0) and D47 = IF(D45<0,0,D45 + D46). D46 depends on D47 and D47 depends on D46 → circular reference.
- Why it breaks: Excel either returns error or unpredictable results; carryforward logic must refer to prior-period values (left column), not the same-period cells.
- Simple, robust approach (no circulars):
  - Reserve row (say row46) for "Carryforward at start of period".
  - Set initial carry (period 0): C46 = 0.
  - For period D (first actual year) set D46 = C46 (or explicit 0).
  - For each later period (E, F, ...) compute carryforward start from the previous period’s numbers (always refer left):
    Example formulas (for columns D then E):
      - D46 (start carry) = C46   ← initial carry (0)
      - D47 (Taxable income) = MAX(0, D45 + D46)
      - D48 (Taxes) = -D47 * $C$37   (taxrate)
      - E46 (start carry) = IF(D45 < 0, D46 + -D45, MAX(0, D46 - D45))
      - E47 = MAX(0, E45 + E46), etc.
  - Explanation: E46 uses previous EBT (D45) and previous carry (D46) only — no circularity. Taxes use taxable income row (row47).
- Implementation tip: add a helper “carry-end” or compute start-of-period carry in each column from the prior column; do not refer sideways to same column.

3) Q1 / Q1 c-d — IRR ambiguity and Excel IRR behavior (explanatory / warning, not a formula bug)
- Observation: cashflows change sign more than once (future negative CFs), so multiple IRRs exist. Excel's IRR function returns one root (depends on guess or algorithm).
- Recommendation: prefer NPV profile to decide (you already compute NPV over rates). If using IRR in Excel use the guess argument to try different roots or use XIRR with dated cashflows, but the safe rule: use NPV rule when cashflow signs flip.

4) Q3 (Ford) — using raw 3-year average with a negative year for perpetuity and P/E
- Fault: C34 = AVERAGE(C28:E28) includes a large negative year → perpetuity value C37 = C34/0.09 and P/E = C41/C34 become meaningless or distorted.
- Why it’s risky: negative/unusual year skews the average and leads to nonsense perpetuity and P/E.
- Fix / recommendation: use normalized/adjusted earnings (exclude one-off or use median or last-2-year average) before dividing by a small perpetuity rate, or explicitly check C34>0 before using it in ratios. Example guard:
  =IF(C34<=0,"invalid",C34/0.09)
