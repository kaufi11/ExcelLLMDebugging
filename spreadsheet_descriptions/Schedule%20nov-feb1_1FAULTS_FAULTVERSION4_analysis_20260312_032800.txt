Sheet summary
- Single sheet: Sheet1 — a training catalog with course codes, titles, durations, computed prices and scheduled dates across four monthly blocks (NOV-02, DEC-02, JAN-03, FEB-03).

Layout and key columns
- Column A: course code or section headers (e.g., "Introductory Courses (Languages):").
- Column B: course title / description.
- Column C: number of days (duration).
- Column D: price in USD (calculated by formula referencing column C).
- Columns E–T: schedule grid (row 2 lists day numbers for each column). Months are grouped with top headers in:
  - F1 = NOV-02, J1 = DEC-02, N1 = JAN-03, R1 = FEB-03.
  - Row 2 under these columns lists the specific day numbers (E2..T2 = 4,11,18,25,2,9,... up to 24).

Course rows and groupings
- Courses are grouped by category with section header rows: Introductory Courses (Languages), Administrators (Oracle Server), Oracle9i/Oracle8i, Oracle Developer, Oracle Designer, DSS (Express/Financial/Sales Analyzer), Datawarehousing, Web, Oracle Applications (Financials, Technical Foundation, Human Resources), Java, etc.
- Each course row typically contains: A = code, B = title, C = # days, D = formula for USD, and one or more schedule cells (E–T) with date ranges (e.g., "18-20", "21-22").

Formulas and pricing rules
- Most price formulas in column D multiply the per-day rate by the number of days in column C:
  - Common rates seen: 250 (e.g., D4 =250*C4), 200 (e.g., D17 =200*C17), 300 for Oracle Applications (e.g., D85 =300*C85).
- Examples:
  - D4 (DMRDD) = 250 * C4
  - D17 (O8iDBA) = 200 * C17
  - D87 (GL) = 300 * C87
- Notable exceptions / possible errors to review:
  - D6 (SQL1) is listed as =200/C6 (division) — likely intended to be =200*C6.
  - Some rows (e.g., D82) reference C82 but no C82 value is shown in the data provided, so those formulas may evaluate to zero or error depending on the workbook.
- Rates appear category-dependent: many developer/designer/administrator courses use 200–250 per day; Oracle Applications use 300 per day.

Scheduling / calendar
- Schedule cells contain date ranges placed under the month-day columns (E–T). For each course a date range appears in one or more of these calendar columns indicating when the course is run (e.g., G6 = "18-20" under NOV block).
- The row 2 day-number row provides the mapping of week start dates across the four-month span.

Other content / notes
- Footer text lines present VAT and legal notes (rows ~105–110) in column A/B (e.g., "VAT is charged according to local regulations.", reservation-of-rights note in English and Russian, and a Russian note about custom course offerings).

Actionable suggestions (if you want changes)
- Fix formula D6 to =200*C6 (if intended).
- Check any Dxx formulas that reference missing C values (e.g., C82) and standardize per-day rates if desired.
- Consider using a clear date header structure (e.g., real dates in row1/row2) or a separate calendar mapping for easier automation.
