Overview
- Workbook has 2 sheets: "Profs" (main contact list) and "URLs" (list of web links).
- "Profs" is a tabular contact database of academics / researchers. Rows ~2–75 contain individual records; row 1 is the header.

Sheet: Profs — layout & contents
- Header (A1:P1): # | sent | Name | Position | University | Email address | Suggested by | Personal or Related Web Page | Address | (J header not present consistently) | City | S/P | CNTY | Zip | Phone | FAX
- Key columns:
  - A: record number. A2 is the literal 1; most subsequent cells use incremental formulas (e.g. A3 = A2+1, A4 = A3+1, etc.). A few anomalies: A52 = A49+1 (skips rows), A63 = A62-1 (decrements), so numbering is mostly sequential but not strictly uniform.
  - B: "sent" status (values like x, u).
  - C: Name (includes Dr./Prof. prefixes).
  - D: Position (Professor, Assistant Professor, Graduate Student, Director, etc.).
  - E: University / Affiliation.
  - F: Email address (many filled; some entries are “?”).
  - G: "Suggested by" (names of introducers/referrers).
  - H: Personal/related web page (present for many rows).
  - I, J, K: address lines (I and J used variably; K typically City).
  - L: State/Province (S/P).
  - M: Country (CNTY).
  - N: Zip/postal code.
  - O / P: Phone and FAX (populated for some rows).
- Data notes:
  - ~74 records (rows up to 75). Some duplicate names (e.g., Joe Paradiso appears more than once in different roles).
  - Some fields are blank or contain “?” (missing email or other data).
  - Mixed formatting of addresses — some rows use J for a second address line; most contact details are in O/P.
  - Column A uses simple arithmetic formulas to generate sequential IDs (works until the few anomalies noted).

Sheet: URLs — layout & contents
- Single-column list (A). A1 = "Random URLs".
- Rows alternate between a descriptive label and the corresponding URL (examples):
  - Worcester Polytechnic Institute — http://www.wpi.edu/
  - MIT Laboratory for Computer Science — http://www.lcs.mit.edu/
  - MIT Media Lab — http://www.media.mit.edu/
  - WIND, Congestion Manager, CAG, CellularIP, IETF Mobile IP, CMU Monarch, MosquitoNet, MEDIA (KTH), INRIA, NUS Mobile IP, etc.
- Purpose: quick reference of project / institutional web links related to the contacts.

Formulas & quirks to be aware of
- Primary formula pattern: column A uses =previous_row + 1. A few exceptions (A52, A63) break strict sequencing.
- Some rows reference earlier cells across gaps (e.g., A52 = A49+1), so inserting/deleting rows can affect numbering consistency.
- Missing emails (F) and incomplete phone/fax entries (O/P) exist; some contact pages are provided in H instead.

Summary
- "Profs" is a contact roster with structured columns for status, identity, affiliation, email, referrer and full contact details. Column A auto-numbers most rows via simple +1 formulas but contains a few anomalies. "URLs" is a single-column list of related web links (description + URL).
