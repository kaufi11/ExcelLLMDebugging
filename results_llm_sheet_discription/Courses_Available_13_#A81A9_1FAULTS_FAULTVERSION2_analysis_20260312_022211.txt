Better, robust fix (recommended)
Use a formula that always finds the last numeric ID above and adds 1. This ignores gaps in column A (header rows left blank) and prevents accidental big jumps.

- Put this in the first ID cell of a block (A4) if empty:
  A4 = 1
- In the next ID cell (A5) and copy down only into the rows that should have IDs (do NOT paste into header rows):
  A5 = LOOKUP(9.99999999999999E+307,$A$4:A4)+1

Explanation
- LOOKUP(9.999E+307,$A$4:A4) finds the last numeric value in A above the current row (it ignores blanks or text). Adding 1 gives the next ID. This prevents a single bad cell (like +808) from propagating wrong values and is robust if some rows have blank A (section headers).
- Don’t paste the formula into header rows (like “Desktop Courses”, “SAP Courses”); leave those A cells blank. Paste only into the course rows.

Notes
- The =+ prefix (e.g. =+A4+1) is harmless — it works the same as =A4+1 — but it’s stylistically odd.
