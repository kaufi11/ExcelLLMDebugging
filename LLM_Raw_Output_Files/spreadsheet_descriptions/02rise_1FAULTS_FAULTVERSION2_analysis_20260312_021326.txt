High-level overview
- Workbook purpose: class roster / financial tracking for RISE training in 2002 — counts of participants by company for each class, calculation of student-days, public cost, RISE tuition (RISE cost), and savings; per-class and aggregated summaries; a company-level summary; and year-over-year financial metrics.

Sheets
1) Sheet4 (titled "RISE CLASSES (JAN.-DEC. 2002)")
- Layout: column A = Class Title; B = Class Date; C = Number of Class Days; D = Public Cost (per class); E = RISE Cost (tuition per class).
- Columns F through BG: one column per participating organization (e.g., F = Capital One, G = Philip Morris, H = Isle of Wight Co. Schools, … up to BG). Each class row lists how many participants that company sent to that class.
- BK (per class): "Number of Participants Per Class" — BKn = SUM(Fn:BGn).
- BL (per class): "Total Number of Student Days Trained" — BLn = Cn * BKn.
- For each class there is a following "Total" row pattern that:
  - multiplies each company’s headcount by the RISE Cost ($E$) to compute that company's RISE tuition for the class (formulas like =G3*$E$3, etc).
  - BO (per class) = SUM of those company-dollar amounts across F:BG (total actual RISE tuition collected for the class).
  - BM (per class) = public cost * participants (D*BK) — the hypothetical public-sector cost.
  - BN (per class) = BM − BO (the RISE savings for that class).
- There is also a "Total Savings" row for each class computing per-company savings (company-dollar total − D*company_count) and BO totals for savings.
- At the bottom of the sheet are aggregate summary formulas:
  - C268 = total number of classes (SUM of class-count rows).
  - BK268 = SUM(BK3:BK264) — total participants across all classes.
  - BI268, BI269, BI270 etc compute grand totals (examples: BI268 sums public-cost totals across columns F:BG; BI269 and BI270 are grand totals for RISE cost and RISE savings respectively).
  - BO268/BO269/BO270 are large SUMs aggregating BO (class-level totals) across the whole year (many BOxx are summed).
- Pattern: each class appears as three-row group (class row, "Total" dollarization row, "Total Savings" row). The sheet automates participant sums, student-days, class-level public cost vs RISE cost comparisons, and per-company dollar shares.

2) Companies (titled "COMPANIES PARTICIPATING IN RISE CLASSES - 2002")
- Layout: column A = Company Name; B = Class Participants (total headcount for 2002 by company); C = Total Public Cost; D = Total RISE Cost; E = Total RISE Savings.
- Each company row shows totals (B..D) and column E = C − D (FORMULA =SUM(Cx − Dx) used to show savings).
- Bottom row provides grand totals:
  - B58 = SUM(B3:B56) (total participants across companies)
  - C58, D58, E58 = SUM of C3:C56, D3:D56, E3:E56 respectively.
- This sheet is the company-level roll-up sourced from the per-class data on Sheet4.

3) Financial Info (titled "2002 RISE Financial Information")
- Purpose: year-over-year comparison (1997–2002).
- Columns B–G are years 1997..2002.
- Rows include:
  - # classes held, canceled, rescheduled
  - # students trained, # student days
  - average class length (student days / students)
  - average cost per day (total RISE tuition / student days)
  - average cost per class (= avg class length * avg cost per day)
  - total RISE tuition dollars, total RISE savings, total public cost (note: total public cost = total RISE tuition + total RISE savings; a comment indicates assumption of $200 per day per diem).
- Also contains explanatory notes for the calculations (how averages and totals are derived).
- Key 2002 numbers (from this sheet):
  - Total RISE tuition dollars (2002): 525,562
  - Total RISE savings (2002): 838,743
  - Total Public cost (2002): 1,364,305
  - Number of students trained (2002): 1,023
  - Number of student days (2002): 2,406
  - Average cost per day (2002): 218.44

Key formulas and flows
- Company participant totals (per class) -> BK = SUM(F:BG).
- Student-days per class = C * BK (BL).
- Company dollar contribution per class = company_count * RISE cost (company cell in "Total" row = company_count * $E$row).
- Per-class RISE tuition total (BO) = SUM(per-company dollar totals across F:BG).
- Per-class public cost (BM) = public cost (D) * participants (BK).
- Per-class RISE savings (BN) = BM − BO.
- Company sheet entries are roll-ups (sums) of per-class contributions and public-cost equivalents; savings per company are C − D.
- Financial Info summarizes totals and computes averages using the year’s aggregated tuition, student-days and savings.

Overall layout & intent
- Wide ("cross-tab") class-by-company table where each class (rows) records participant counts by company (columns F:BG).
- Automatic per-class calculations (participants, student-days, public cost estimate, actual RISE tuition collected, savings).
- Company-level summary sheet that aggregates results for each company and computes per-company savings.
- Financial Info sheet provides multi-year summary metrics and documents how averages/totals are calculated.
