Findings (short)
- Error: H19 uses =COUNT(H7:H17) but should be summing the "4 – Very Good" counts. COUNT returns 9 (the number of filled cells) instead of the total 313.
- Consequence: the row-of-column totals is wrong and the per-rating column totals no longer match the sum of the per-item totals.

Fix (exact)
1. Select H19 and replace the formula with:
   =SUM(H7:H17)
2. Press Enter.

Result after fix
- H19 will show 313.
- The five column totals (G19..K19) will then sum to 1289, which equals SUM(L7:L17) — so everything will balance.

Why this fixes it (simple)
- COUNT counts how many numeric cells exist; SUM adds the numeric values. You need SUM to add the survey counts, not COUNT.

Optional helpful suggestions
- Add a grand total for the per-item totals in L19: =SUM(L7:L17). This provides an easy check that row totals equal column totals.
- If you only want the date in A3, format the cell as Date (no time) to clean the header.
