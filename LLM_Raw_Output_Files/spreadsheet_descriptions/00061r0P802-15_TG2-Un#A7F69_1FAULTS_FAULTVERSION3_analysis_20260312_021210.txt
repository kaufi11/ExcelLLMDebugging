Summary of workbook

Sheets:
- Profs (primary contact list)
  - Header row (row 1) defines columns:
    - A: Number (auto-increment), B: Name, C: Position, D: University, E: Email address, F: Suggested by, G: Personal or Related Web Page, H: Address, I: (no header in your extract — used for secondary address lines in some rows), J: City, K: S/P (state/province), L: CNTY (country), M: Zip, N: Phone, O: FAX.
  - Row count / range shown: header in row 1; data rows A2 through A63 (62 contacts listed).
  - Column A uses a simple incrementing formula pattern: A2 = 1, A3 = A2 + 1, A4 = A3 + 1, etc., producing sequential IDs. Exception/anomaly: A54 = A53 - 1 (this will break the sequence and create a duplicate/step-back in numbering).
  - Typical row contents: name, title, affiliation, email, who suggested the contact; optional columns G–O contain personal webpages, street address (H/I), city/state/country, zip, phone and fax. Example: row 2 contains Roy D. Yates (Director, Rutgers) with email ryates@winlab.rutgers.edu and Suggested by Kevin Marquess; row 28 (Andrew T. Campbell) includes full web page, address, city/state/country, zip, phone and fax.
  - Data consistency/notes:
    - Many rows leave columns G–O blank (partial records).
    - Several email fields contain "?" (missing email).
    - Column I lacks a header but is used for additional address info in multiple rows.
    - Some names/entries appear more than once (e.g., Joe Paradiso appears as Principal Research Scientist and also as Graduate Student).
    - Phone/fax formatting varies (with/without country code, parentheses, spaces).
- URLs (reference list)
  - Single-column layout (column A) with a title in A1 ("Random URLs").
  - Rows alternate between a description/project name and the corresponding URL (e.g., A3: "Worcester Polytechnic Institute, Center for Wireless Information Studies", A4: "http://www.wpi.edu/"; A10/A11 for MIT LCS, etc.).
  - Contains links to multiple labs/projects (WIND, CM, Media Lab, LCS, CellularIP, IETF Mobile IP, CMU Monarch, MosquitoNet, etc.).

Key formulas and values
- Column A (Profs): incremental numbering via A(n) = A(n-1) + 1 starting at A2 = 1. One intentional/unintentional deviation at A54 = A53 - 1.
- No complex formulas elsewhere shown; mostly static text values and URLs.
- Notable populated cells: G28/G29/G30 etc. contain personal web pages; N and O hold contact numbers for selected rows (e.g., N28 (212) 854-3109, O28 (212) 932 9421).

Overall layout and usage
- Profs is a simple contact database with one contact per row and fixed columns for contact metadata; intended to be read horizontally per contact.
- URLs is a flat list of project/organization names paired with their web addresses in adjacent rows.
- Minor structural inconsistencies to consider cleaning: fill/make explicit header for column I, correct the A54 numbering formula if undesired, standardize phone/fax formats, and replace "?" placeholders with missing data or blanks.
