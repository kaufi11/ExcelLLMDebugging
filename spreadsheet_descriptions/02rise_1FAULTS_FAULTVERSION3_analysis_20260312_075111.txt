Summary of the workbook (concise)

Sheets
- Jan.-Dec. 2002 (large detail sheet, shown as "Sheet4 / Jan.-Dec. 2002")
  - Row layout: repeating blocks for each class:
    - Data row (e.g., row 3): A = class title, B = class date, C = number of class days, D = public cost (per class or per student-day depending on context), E = RISE cost (per student), columns F through BG = participant counts by company (one company per column).
    - Immediately below each class row there are three summary rows labelled "Total", "Total Savings" (and sometimes intermediate totals). These compute per-company totals, totals per class and savings.
  - Key per-row formulas:
    - BK (per class, column BK) = =SUM(F#:BG#) → total participants in that class.
    - BL (column BL) = =C#*BK# → total student-days for that class.
    - Per-company "Total" cells (e.g., F4, G4, ...) = company_count * $E$row (e.g., =F3*$E$3) — cost attribution using the RISE cost for that class.
    - BM (per-class total public cost) = =D#*BK# (e.g., BM4 = D3*BK3).
    - BO (per-class total actual RISE cost) = =SUM(F4:BG4) (sum of all company totals for the class).
    - BN (per-class savings) = =BM# - BO# (public cost minus RISE cost total).
    - Total Savings "Total" row for companies: each cell = company_total - (public_cost * company_count) or similar pattern (the sheet uses many SUM(...) patterns to compute savings by company across the block).
  - Workbook-level aggregates (bottom of the sheet):
    - Row 268–271 supply grand totals and aggregates:
      - C268 (C total) = SUM of class days count (C3:C264).
      - BI268 = grand total public cost (sum across F268:BG268).
      - BI269 = grand total RISE cost (sum across F269:BG269).
      - BI270 = grand total RISE savings (sum across F270:BG270).
      - BK268 = total participants across all classes (=SUM of BK per-class).
      - BL268 = total student-days (=SUM of BL per-class).
      - BO268 and BO269/BO270 are large SUM(...) formulas adding up all per-class BO/BM/BN results.
  - Layout note: columns F..BG are one column per participating company (column headers list roughly 70+ companies); the sheet repeats the same calculation block for each class.

General structure & formula patterns
- The detailed class sheet uses a consistent block pattern: class row → "Total" row(s) → next class. All per-company calculations are done by multiplying participant counts by the class's RISE cost (cell E#) and by aggregating with SUM across columns.
- Company participation columns (F:BG) are summed horizontally per class (BK) and vertically across classes (Companies sheet and totals rows).
- Savings are computed as Public Cost totals minus RISE Cost totals (BM - BO or equivalent).
- The workbook relies heavily on SUM across large cell ranges and on a few absolute references (e.g., $E$#) to compute per-company cost allocations.

Key takeaways
- Primary purpose: track each RISE class in 2002, capture per-company participant counts and compute costs (public cost vs. RISE tuition), aggregate total student-days, RISE revenue, and estimated public-sector savings.
- Two summary views: (1) Companies sheet — totals per organization (participants, costs, savings); (2) Financial Info — year-over-year metrics and 2002 summary values (tuition, savings, student-days, averages).
- Main calculation fields to inspect when auditing: BK (participants per class), BL (student-days per class), BM/BO/BN (per-class public cost / RISE cost / savings), and the grand total rows (row ~268–271) that combine all per-class results.
