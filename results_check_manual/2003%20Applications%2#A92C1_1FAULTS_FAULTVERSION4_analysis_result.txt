Other issues / inconsistencies to review
1) AHTF!F14 vs AHTF!E14 (range inconsistency)
   - E14: =SUM(E2:E13) (TOTAL AHTF FUNDING REQUESTED)
   - F14: =SUM(F3:F13)
   - Problem: The E formula sums E2:E13 but the F formula starts at F3, omitting F2. That is inconsistent (though it may be intentional if F2 has no recommended amount). If you intend the Recommend total to sum the same row-range, F14 should be =SUM(F2:F13).
   - Suggested action: If F2 intentionally has no numeric recommend, leaving F14 as-is will not change its value; otherwise change F14 to =SUM(F2:F13) for parallelism and clarity.

2) Use of SUM() on a single cell (cosmetic/inconsistency)
   - Non Profit!E28 =SUM(E27) and Non Profit!F28 =SUM(F27)
   - County!E28/E? (similar pattern)
   - These work correctly but are stylistically inconsistent — a plain =E27 (or =F27) is simpler. No functional error, just an inconsistency.

3) Text in numeric columns
   - AHTF!F10 contains text "Pending Feasibility Study". SUM ignores text, so totals that sum recommended amounts (F-column) are fine, but presence of text in a Recommend column can be confusing. Consider using a separate notes column or ensure the cell is excluded from numeric totals intentionally.

4) Hard-coded constants in formulas
   - AHTF!E16 and AHTF!F16: =2432890*0.15
   - This works, but uses a hard-coded base (2432890). If that base should come from a cell (for traceability/maintenance), consider referencing the source cell instead of embedding the number.

5) Several Recommend cells are manually entered rather than formula-driven
   - On the Municipal, Rehab, AHTF and other sheets many F cells are left blank or contain manual numbers while others are set to =E(row). This may be intentional (only some requests are recommended) but is a potential source of accidental mismatch/copy-paste errors (example: the F5 error above). Review whether you want recommend column to be formulaic (=E#) for rows that are fully recommended.
