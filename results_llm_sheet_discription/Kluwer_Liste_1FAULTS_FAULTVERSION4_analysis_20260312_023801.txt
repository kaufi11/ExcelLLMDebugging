Summary — the two problems you need to fix
1) Row 2, column I (I2) contains an openpyxl ArrayFormula object instead of a numeric value or a normal Excel formula. That cell will not behave like a number (breaks exports, sums, comparisons, etc.).  
2) Row 41, column J (J41) uses the wrong operation: J41 = H41 / 1.5 instead of J41 = H41 * 1.5. That produces a wrong institutional e‑book price for that record.

Why these matter (brief)
- I2 being an object prevents correct numeric processing for the whole column and can break automated workflows.  
- J41 being a division gives a lower value where every other J is 150% of H; it’s an obvious outlier and will create incorrect totals or reports.
