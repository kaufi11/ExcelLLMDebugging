Sheet: Q1
- Purpose: DCF sensitivity table for cash flows [-85, 120, 100, -140] at periods 0–3; computes discounted cash flows for many discount rates and sums them.
- Layout: Row1 = Period headers (0..3). Row2 = nominal cash flows. Column B labels (DCF:, C (discounted) etc.). Column J lists discount rates for each sensitivity row (J4..J44 from -0.10 to 0.30).
- Key formulas:
  - Per-period discounted CFs: e.g. D4 = $D$2/(1+$J4)^$D$1 (E4,F4,G4 analogous).
  - Row sums: B4 = SUM(D4:G4) (repeated down rows B4:B44).
- Key values/notes:
  - Input cash flows: D2=-85, E2=120, F2=100, G2=-140.
  - J4 = -0.10 initial rate; rates increment to 0.30.
  - A46 states: "The NPV is positive for discount rates which are >7% and <43%."
  - IRR: sheet reports IRR formula E48 = IRR(D2:G2). Commentary rows note multiple IRR solutions in presence of non‑conventional cash flows.
- Minor inconsistency: D5..G5 formulas refer to $J5 except G5 uses $K5 in your listing — likely a typo (should be $J5).

Sheet: Q1 a-b, graph

- Key formulas: B7..B47 = NPV(rate, future CF range)+initial CF.
- Key values: rates sweep from -0.10 to 0.30; used for plotting NPV curve to locate sign changes.

Sheet: Q1 c-d, additional info

Sheet: Q2

  - Discount/WACC = 15% (C4, C40, C62).

- Layout:

- Key formulas:
  - Taxes: e.g. D8 = -D7*$C$3 (tax on income proxy).

Sheet: Q4

Sheet: Q5
- Purpose: Simple valuation formula demonstration using Gordon growth model.
- Key formulas and examples:
  - P/E = 1/(r - g). Example: company with r=10%, g=5% gives P/E = 1/(0.10-0.05).
  - E/P = r - g (the inverse).
- Layout: Two example rows for Company 1 (g=5%) and Company 2 (g=2%).

General observations / best-practice notes (concise)
- NPV vs Excel NPV: the workbook correctly uses NPV(rate, future CFs) + initial CF at t=0. Remember Excel NPV assumes first value is at t=1.
- Multiple IRRs: non‑conventional sign patterns (negative cash flow after t=0) can produce multiple IRRs — workbook demonstrates this and uses different guesses in IRR() to find alternate roots.
- Reusable template pattern: many blocks are copies of the same structure (period headers, cashflows, taxes, discounting) with parameters centralized (tax rate, discount, loan, depreciation).
- Watch for small errors: e.g. one G5 reference to $K5 (likely should be $J5); verify consistent references (some debt/depr blocks use different cell references for the same input).
