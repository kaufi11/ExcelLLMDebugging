Summary of the problems
- Many cells in rows 9–39 (columns E:M and some others) contain wrong/chained SUM formulas like F9 = SUM(E9), G9 = SUM(F9), E9 = SUM(E42) etc. These reference a single cell or the previous column, so they don't show per-column or per-row service counts — they create a cascading/misplaced calculation.
- Totals row (row 40) mostly references the wrong cells: F40 = SUM(E40), G40 = SUM(F40) etc. That sums totals of other columns instead of summing the column's data.
- One obvious typo: K19 = SUM(235) — SUM of a literal number is meaningless; probably intended to be a number or proper reference.
- Likely missing per-row totals: Column C ("Total #") should be the sum across the service columns (D:M) for each day, but that formula isn't shown.

Why this is wrong (short)
- SUM(single cell) or SUM(previous column total) does not compute the intended column totals or per-row totals. The sheet is summing the wrong ranges, so bottom totals and per-row totals are incorrect.
