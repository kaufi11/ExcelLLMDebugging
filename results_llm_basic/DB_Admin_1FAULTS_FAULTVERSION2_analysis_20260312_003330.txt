Why this fixes it (plain):
- Each L-cell multiplies the question’s Rating (Rate!E11:E20) by a factor based on the user’s selection (K-cell). Those L formulas were pointing to the wrong Rate!E row (an offset/copy-paste error), or in Ap3 L20 used <> so it never matched correctly. Correcting the Rate!E reference and using equality tests makes each L-cell pick the correct rating and compute the right score. Once corrected, the Summary sheet that sums these L-values will be accurate.

Prevention tip (short):
- Don’t hard-code matching E-cells by hand. Instead use a lookup so the score looks up the rating for the question text. Example pattern:
  =IF(K16=2, INDEX(Rate!E:E, MATCH(B16, Rate!G:G, 0))*0.33, IF(K16=3, INDEX(Rate!E:E, MATCH(B16, Rate!G:G, 0))*0.66, IF(K16=4, INDEX(Rate!E:E, MATCH(B16, Rate!G:G, 0))*1, 0))))
  This automatically finds the right Rate row and prevents offset/copy-paste errors.
