Summary — main error
- Sheet "In-kind contribution template": cell F6 has the wrong operator. It uses =D6/E6 (division) but should be =D6*E6 (multiplication).

Why this is a problem (simple)
- D6 = 30 and E6 = 150. =D6/E6 returns 0.2, but the correct contribution is 30 * 150 = 4,500.
- That single wrong row understates the labor subtotal and every downstream total (F16, G16, G29, G41, F43) by 4,499.8.

How to fix (exact steps)
1. Select cell F6 on the "In-kind contribution template".
2. Edit the formula and replace =D6/E6 with =D6*E6, then press Enter.
   - Or: copy F5 (which is =D5*E5) and paste/fill down to F6 (and any other labor rows) so formulas stay consistent.

Verify
- After the fix, F6 should show 4,500.
- F16 (labor total) should increase by 4,499.8 compared with the wrong value.
- Check G41 or the grand total cell to confirm the final total now matches expected sums.

Optional improvement (best practice)
- Use consistent formulas by filling down the column instead of typing each formula manually to avoid operator typos.
- Add a quick sanity check: conditional formatting that flags any currency cell < 1 (or unusually small) so accidental division mistakes are easier to spot.
