1) Clear formula fault (probably the intentionally broken cell)

- Cell: E23 (floor UA)
- Problem: E23 is hard-coded as =6 instead of being calculated from area and R-value. Other sheets use E23 = C23/D23 (which yields 78.9473684210526). Because E23 is 6 the TOTAL (UA) in E26 = 2026 (sum(E19:E25)). In the other/expected case E26 = 2098.94736842105.

4) Input-format dependency in conditional formulas (potential user error)
- Sheets: all
- Cells/logic: D38 = IF(C13<1, D37/100000, 0) and F38 = IF(C13=G34, D37/F9, 0)
- Problem: these IFs assume furnace efficiency C13 is entered as a fractional value (<1 for combustion systems, =1 for electric). If a user instead enters "60" (i.e., percent rather than 0.6) the IF tests fail: D38 would return 0 and D37 would have divided by 60 (leading to a wildly incorrect result), or worse D37 = D36/C13 would divide by 60 instead of 0.6 and produce a far-too-small fuel requirement.
- Impact: brittle to different input conventions; easy to get wrong results.
- Suggested fixes:
  - Enforce/clarify that efficiencies must be entered as decimals (0.6).
  - Or change formulas to tolerate either format (e.g., normalize inputs: =IF(C13>1, C13/100, C13) and use that normalized value in computations).
  - Add input validation (e.g., IF(C13=0,NA()) or an error message) to avoid divide-by-zero.

5) No guard against zero/blank furnace efficiency (divide-by-zero risk)
