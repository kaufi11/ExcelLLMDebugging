Summary — main error found
- On the Formulas sheet cell C18 is not a formula referencing the COGS schedule; it contains 207 (a hard-coded value). It should be linked to the COGS result (C34). This breaks the income statement: Gross margin and downstream rows on that sheet will be wrong or inconsistent.

Why this is a problem
- C18 should equal the COGS schedule result (C34). A hard-coded 207 disconnects the income statement from the COGS schedule and gives incorrect totals whenever sales/inputs change.

Fix (quick)
1. Select Formulas!C18.
2. Replace the value with the formula: =C34
   (or, if you prefer the file’s style: =+C34)
3. Recalculate (F9) and verify C18 now = 105000 (with the sheet’s inputs/rounding).

Alternative fix (if you want to copy from the working sheet)
- Copy Example!C18 and Paste → Formulas!C18 (Paste Formulas).

Verify
- After fixing, check consistency: C18 should equal C34; E18 should equal E34. Use Excel’s formula view (Ctrl+`) or Formula Auditing to confirm.

Prevention / quick checks
- To find other accidental hard-coded numbers where formulas expected: Home → Find & Select → Go To Special → Constants (Numbers) while on the Formulas sheet. Any cells found in areas that should contain formulas likely need review.
- Consider naming key cells (e.g., Sales_Aug, COGS_Aug) or protecting the sheet to avoid accidental overwrites.
