2) Inconsistent handling of "no value" in percentage IF() formulas
- Examples: Sheet "1" row 7 (Gross profit %), formulas like =IF(C$4=0,"-  ",C6/C$4)
- In other sheets similar IF() expressions use =IF(...,"",...) (empty string) rather than "-  " (two-character string).
- Issue: returning a text string ("-  ") where a numeric is expected produces mixed types in the column. Although SUM/AVERAGE typically ignore text, this can break charts, further numeric calculations, or type-sensitive operations.
- Recommendation: pick a consistent placeholder for “no value” (prefer "" or NA() if you want an error) and apply consistently.

3) Use of text "-" in numeric cells
- Several cells contain the text "-" (not a numeric zero). Examples: many segment/regional cells (e.g. sheet "4" C5/C6/C7 are "-"), sheet "2" treasury-stock cells C36:D36 etc.
- Issue: text "-" will be ignored by SUM and other numeric aggregations (which may be what you want), but it is risky because it’s not a numeric zero and will break functions that expect numeric types.
- Recommendation: If you intend “no data / not applicable” use blank/"" or NA(); if you intend zero use 0. Be consistent.

- Sheet "1" row 21 (Cumulative effect of accounting change) has a value only in L21. The net income rows correctly sum C20:C21 etc, which is fine (blank cells are ignored). This is not an error but worth confirming that the lone entry in L21 is intentional.
