High-level structure (all sheets)
- Common layout: header text in rows 2–5, column headers in row 6, data rows start at row 7. Key columns (B:I):
  - B: Program/Region tag or Region #
  - C: RFP# (proposal ID)
  - D: Applicant
  - E: Project Title
  - F: Results (Fund / Not Fund / Non-Competitive)
  - G: Project Funds Requested
  - H: Approved / Awarded Funds (0 if not funded)
  - I: Cumulative total of Awarded Funds (running sum) — used to track against the sheet funding limit shown in the header

- Column I uses formulas that cumulatively add the Awarded Funds in H down the list (typically I7 = H7 then In = I(n-1) + Hn). Some sheets use G instead of H for the initial cell.

General observations / potential issues
- Consistent structure across sheets simplifies aggregation by program, region, or applicant.
- Column I is intended as a running total of awarded funds; most sheets implement this via I7 = H7 and In = I(n-1) + Hn. However there are inconsistencies:
  - Watershed Protection I7 = G7 (uses requested instead of awarded).
  - NPS Pollution I32 formula =47+H32 likely a mistake.
- Several projects received partial awards (H < G); many “Not Fund” rows have H = 0.
- Header rows include each sheet’s funding limit — these are the control totals to compare against the cumulative column I.
- A few numeric rounding/format differences (e.g., 311,280.25 requested but 311,280 awarded).
