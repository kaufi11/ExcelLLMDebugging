Summary — what’s wrong
- Many formulas show #REF! because they reference cells that were deleted or moved. Examples: O39:O40, O39:P39:Q39:R39, S44, P45, P50/P51/P53, etc. Any formula containing #REF! will return an error and break downstream calculations.
- The contract/phase dollar totals are in the wrong column: the subtotal formula for the phase amounts is in E16 (=SUM(D11:D15)) instead of in D16. Same pattern likely for the additional-basic-services subtotal (E29). That inconsistency causes later formulas to reference the wrong cells (or a deleted cell) and produces errors.
- Some allocation formulas use percent/decimal inconsistently (for example dividing by 100 in one place and not in another) — this will give wrong dollar allocations if not standardized.
- Some SUMs are unnecessarily odd (e.g., SUM(S44:S44)) — not an error by itself but indicates copy/paste problems.

Why this happened
- Someone moved or deleted the original source cells (or shifted amounts from column D to column E) after other formulas were built, so dependent formulas now point to missing cells (#REF!).
- Copy/paste across rows without using absolute references ($) created broken links.
