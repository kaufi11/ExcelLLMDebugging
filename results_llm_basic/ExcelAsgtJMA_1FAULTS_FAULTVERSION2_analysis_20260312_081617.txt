Summary of errors
- Column-range typo in every Points cell (C9:C24): formulas use Dn:CCn (e.g. =SUM(D9:CC9)) instead of the intended Dn:In (D through I). This is a typo (CC vs I).
- One row (C19) uses COUNT(D19:CC19) instead of SUM(...), so it returns a count of numeric cells rather than the points total.
- Grade % formulas (B9:B24) use =SUM(Cn/$C$6). SUM(...) around a single value is unnecessary (though not harmful). The result should be formatted as a percentage.

Why these are wrong
- D:CC is a very wide range; it usually includes blank cells and is inefficient and error-prone. If extra numbers exist outside I, they would be incorrectly included. COUNT returns the number of numeric cells, not the sum of their values, so C19 is wrong (it will return 6, not the total points).
- Wrapping the division in SUM is unnecessary and confusing.

1. Points column (C9:C24): replace each formula with:

- Select B9:B24, type =C9/$C$6 and press Ctrl+Enter, then format as Percentage.

Why this fixes it
- Using D:I limits the range to only the six score columns (D through I), giving the true points total.
- SUM aggregates the numeric values; COUNT would only count how many numeric entries exist.
- Using Cn/$C$6 gives the fraction of the total points; formatting as % shows it as a percent.
