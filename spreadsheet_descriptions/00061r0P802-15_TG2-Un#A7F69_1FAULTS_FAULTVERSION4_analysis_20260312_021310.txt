Overview
- Workbook has 2 sheets: "Profs" (main contacts table) and "URLs" (list of web links).
- "Profs" is a header + record table spanning columns A:O. Header row (row 1) maps:
  - A: Number, B: Name, C: Position, D: University, E: Email address, F: Suggested by,
  - G: Personal or Related Web Page, H: Address, I: (occasionally used), J: City,
  - K: S/P (State/Province), L: CNTY (Country), M: Zip, N: Phone, O: FAX.
- Records occupy rows 2–63 (about 62 people).

Key formulas and patterns
- Column A is auto-numbered: A2 is the hard value 1; most subsequent rows use A(n)=A(n-1)+1 (e.g. A3 = A2+1, A4 = A3+1, …).
- One anomaly: A19 contains =A18-1 (breaks the sequential numbering and can produce a duplicate/misaligned index).
- Many web addresses are stored as plain text in column G; URLs sheet duplicates/extends some of these links.
- No other calculated columns/formulas are present; most fields are literal text.

Notable data/content points
- Some emails are missing or replaced by "?".
- Duplicate/near-duplicate person entries: e.g., Joseph A. Paradiso (row 46) and Joe Paradiso (row 49).
- Address fields are inconsistently split across H/I/J (some rows use I, some put full address in H).
- Country is stored in column L but sometimes state/country placements vary (inconsistent data entry).
- Phone/fax formatting varies (+1, parentheses, spaces, international formats).

"URLs" sheet
- Single-column list (col A) with alternating labels and URLs (titles on one row, the URL on the next).
- Contains links for WPI, MIT LCS, MIT Media Lab, WIND, CM, CAG, CellularIP pages, IETF mobileip, CMU Monarch, MosquitoNet, etc.
- Row spacing is irregular (some label rows are empty or skipped).

Summary / suggestions (brief)
- Fix A19 to A19 = A18+1 to restore numbering continuity.
- Normalize address/email/phone columns (consistent splitting and formats) and remove duplicate person records.
- Convert web-address text to hyperlinks if desired and consider moving URLs to two-column layout (Label / URL) for clarity.
