- Problem: The existing formula uses the pattern =Applied*(new_NUE) - current_removed (for example =E3*(0.426)-E27). That computes the additional N removed if you keep applied fertilizer constant and NUE goes up — NOT the fertilizer savings required to keep the same yield when NUE increases. The worksheet label asks for "savings ... for each 1% increase in NUE (same yield)" — that is: if NUE increases by 1 percentage point, how much less fertilizer would be needed to obtain the same crop N removal (i.e. same yield). The current formula is the wrong quantity.
- Correct formula (recommended)
  If:
    A = cereal fertilizer applied (e.g. E3 or E4)
    R = current N removed from fertilizer (e.g. E27 or E31)
    r = current NUE = R / A (cells E29, E30, D33, etc)
  Then fertilizer required at the improved NUE (r + 0.01) to get same removal R is:
    new_applied = R / (r + 0.01)
  Savings (applied - new_applied) = A - R / (r + 0.01)
  Equivalent algebraic form (sometimes simpler):
    Savings = A * (0.01 / (r + 0.01))
- Example replacements:
  - Developed 96 E31 (replace =(E3*(0.426)-E27) ):
      =E3 - E27/(E29 + 0.01)

      =E3 - E27/(E29 + 0.01)
  - World 1996 E32 (replace =(E4*(0.343)-E28) ):

  - World 2002 D44 (replace =(D6*(0.343)-D31) ):
      =D6 - D31/(D33 + 0.01)
- Also update the dependent 20% values (e.g. *20) to use the corrected 1% savings cell. E.g. E37 = E32 * 20 (but only after E32 is corrected).

Why it matters: with the corrected formula, the magnitude of the "savings" changes significantly (often larger than your current numbers). The current workbook is reporting "extra removal if applied fertilizer remains constant", not "fertilizer savings to maintain same yield".
