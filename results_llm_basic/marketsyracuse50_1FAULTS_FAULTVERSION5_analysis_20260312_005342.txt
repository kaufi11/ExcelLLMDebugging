2) Possible percent/scale inconsistency in column G / H (common source of wrong HHI)
- Symptom: G formulas use =F10/$F$36*100 (so G returns percent points like 30), but the header says (%) and you may have formatted G as Percentage. If you both multiply by 100 and format as % you get values 100× too large; likewise H (squares of G) will be off by 10,000× if scales mix.
- Two clean, consistent approaches (pick one):

  Option A — keep G as decimal percent (recommended)
  - G10 = F10 / $F$36
  - Format column G as Percentage (e.g. 2 decimals). G36 should show 100%.
  - H10 = (G10^2) * 10000  — this produces the standard HHI on 0–10,000 scale (e.g. 30% -> 900).
    (Equivalent: H10 = (G10*100)^2)

  Option B — keep G as percent points (numbers 0–100)
  - G10 = F10 / $F$36 * 100
  - Format column G as Number (not %). G36 should be 100.
  - H10 = G10^2

2. Replace the G formulas with one consistent formula (pick Option A or B above) and apply it to all rows.
