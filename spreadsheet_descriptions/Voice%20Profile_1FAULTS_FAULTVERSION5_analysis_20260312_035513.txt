Summary of workbook structure and contents

Overall
- Purpose: capture voice telecom usage/costs from multiple sources (bills/contracts), apply consistent pricing and produce aggregated totals and unit metrics for NOIE analysis.
- Sheets: Introduction (instructions/contact), Example (filled sample with formulas), FINANCIAL DATA & ANALYSIS (blank template for data entry).
- Data model: per-service rows (local, long distance, long‑distance-to-mobiles, international, mobile, inbound 1800/1300, etc.). Multiple source columns are summed into a TOTAL column. Unit metrics (cost per call, average duration) are calculated, often as weighted averages.
- Comments: many cells have inline comments; enable comment indicators to view guidance (Tools | Options | View | Comment Indicator Only).

Sheet: Introduction
- Layout: instruction text in column A (rows ~15–38) explaining how to use the workbook; contact details for Lynton Bond.
- Row 35 contains repeated mailto: links across many columns (likely hyperlink placeholders).
- Copyright/disclaimer text in A36–A38.

Sheet: Example
- Purpose: worked example showing sample data and the formulas to use.
- Column layout: columns B/D/F hold data from different sources (bills/contracts); column C/E/G often list the source text; column H is TOTAL (aggregates B + D + F).
- Key formula patterns:
  - Totals: H7 = B7 + D7 + F7 (and similarly for other sections).
  - Sub‑totals: B7 = B8 + B9 + B10 (split of cost categories).
  - Average cost per call: e.g., B22 = B20 / B21; H16 = IF(H15<>0, H14 / H15, "").
  - Weighted average durations: H17 = IF((B15 + D15)<>0, ((B15*B17)+(D15*D17))/(B15 + D15), "").
- Representative sample values (Example sheet):
  - Number of handsets: B11 = 752; D11 = 300.
  - Planned handsets: B12 = 820.
  - Local calls cost: B14 = 15000; D14 = 7324.
  - Local calls volume: B15 = 107500.
  - Long distance (not to mobiles) cost: B20 = 7300; D20 = 6500; B21 = 10152 calls; B22 = B20 / B21 (avg cost per call).
  - Long distance to mobiles: B32 = 1250; B33 = 1114 calls; B34 = B32 / B33.
  - Mobile phones cost: B50 = 9000; B51 = 163 mobiles; B52 = 13750 calls; B53 = avg duration 105s.
  - Inbound 1800: B59 = 2500; B60 = 2568 calls; B61 = B59 / B60.
  - Inbound 1300: B65 = 1600; B66 = 5512 calls; B67 = B65 / B66; B69 = 425 calls > duration threshold.
  - Mix / ratios: B28 = "5/25/40/30" (zones), B29 = "80%/20%" (intercap), B30 = "75%/25%" (peak/off‑peak), etc.
- Many cells in H mirror the aggregation pattern (Hx = Bx + Dx + Fx).

Sheet: FINANCIAL DATA & ANALYSIS
- Template for real data entry; same structure as Example but inputs arranged in C/E/G with TOTAL in column I.
- Column headings: B = labels (service rows), C/E/G = DATA and SOURCE columns, I = TOTAL, J = COMMENTS.
- Key formula patterns mirror Example, replacing H with I and summing C + E + G:
  - I7 = C7 + E7 + G7; I15 = IF(I14<>0, I13 / I14, "").
  - Weighted averages use the two source columns (e.g., I16 formula computes weighted duration from C and E).
  - Many Ixx formulas simply reference the corresponding Cxx (I23 = C23, etc.) where single‑source values expected.
- Intended workflow: enter source data into C/E/G (or fill one column), review totals/metrics in column I.

Notes / recommendations
- Formula guards: IF(...<>0, ...) used to avoid divide‑by‑zero and return blank otherwise.
- Aggregation pattern is consistent—when filling the template, ensure costs and volumes are entered in matching rows/columns so totals and averages compute correctly.
- Comments on cells provide task‑specific guidance—enable comment indicators and hover to read them.
