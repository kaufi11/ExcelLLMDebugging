Error found
- Cell D60 (Spring 1998, "All Grades" Tested) uses =SUM(D53:D58) but cell E60 (Passing) uses =SUM(E53:E59). The tested total excludes the Grade 10 Tested value (D59) while the passing total includes Grade 10 (E59). F60 =E60/D60 therefore divides a larger numerator by a too-small denominator → incorrect percent.

Fix
- Change D60 to =SUM(D53:D59)

Why this fixes it
- Both the Tested and Passing totals will then sum the same set of grade rows (Grades 3–10). The percent F60 =E60/D60 will then be numerator and denominator from the same complete population.

Quick extra checks to prevent similar mistakes
- For every "All Grades" row make sure the SUM ranges in the Tested and Passing columns match exactly (e.g., D17 & E17 use D10:D16 and E10:E16).  
- Use the same row endpoints (first grade row to last grade row) or use a single formula for percent like =SUM(E53:E59)/SUM(D53:D59) so the two ranges are obviously paired.  
- Use Formula Auditing (Trace Precedents) to spot mismatched ranges quickly.
