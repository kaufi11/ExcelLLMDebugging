Summary of errors
- Root cause: many formulas on Ap1 / Ap2 / Ap3 reference the wrong Rating cells on the Rate sheet (wrong E##), and one formula on Ap3 uses <> instead of = (logical error). Because Summary pulls the L‑column scores, totals are wrong.

Why these fixes
- Each B-row pulls a text question from Rate!G11:G20; the corresponding numeric score is in Rate!E11:E20. The L formulas must use the same E cell index as the B row’s G reference. Several L formulas pointed to the wrong E index, producing incorrect partial scores. Ap3 L20 used <> which produces incorrect branch testing — change to =.

Quick suggestion to prevent this in future
- Use a single lookup to pair G→E (e.g. match the question text in column B to lookup the correct E value) or use INDEX/MATCH instead of hardcoding E11/E12/... so shifting rows won’t break formulas.
