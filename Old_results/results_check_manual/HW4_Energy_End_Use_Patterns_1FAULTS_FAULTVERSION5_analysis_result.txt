Details and recommended fixes

1) Johnson - Base: gas-range monthly cost
- Problem cell: C47
- Current formula: =C46/C10
- Why it's wrong: C46 is therms/month for the range (10.2). C10 is the price of natural gas ($/therm, 0.7). The cost should be therms * $/therm, not therms ÷ $/therm.
- Correct formula: =C46*C10
- Effect: current value 14.5714 (10.2 / 0.7) is incorrect; corrected value should be 7.14 (10.2 * 0.7).

2) Missing walls AREA (causes E23 = 0)
- Problem cells: C23 (walls area) is blank in multiple sheets while E23 contains =C23/D23.
- Effect: UA contribution for walls is calculated as 0 because area is blank. If walls area was intended to be non-zero, add the area value into the appropriate C23 cell for each sheet.
- Recommendation: supply the walls area in C23 (for each sheet) or change the formula to reference the correct cell if the area was placed elsewhere. If there truly are no walls (unlikely), add a comment/explanation.

3) Inconsistent input cell positions across sheets (maintenance risk)
- Examples:
  - Johnson - Base length/width/height are in C6/C7/C8 while Johnson - More Efficient uses C5/C6/C7.
  - Price-of-gas is in C10 in some sheets, in C9 in others; electricity price sometimes in C10 or C11.
- Effect: formulas within each sheet appear to be adjusted to the sheet’s layout, so current calculations are internally consistent, but it’s easy to introduce errors when copying or comparing sheets.
- Recommendation: normalize layout across sheets or convert frequently used values (length, width, height, price-of-gas, price-of-electricity, btu/kWh) to named ranges (e.g., PriceGas, PriceElec, BtuPerKwh). Using named ranges would remove fragile column/row dependence and avoid mistakes like the one above.
