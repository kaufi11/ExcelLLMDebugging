- Sheet "B Direct Payments", cell I11

Error type
- Hard-coded / inconsistency error

Explanation (short)
- I11 contains the value 3 instead of totaling the row’s payment values (F11:H11). That makes row and grand totals incorrect/inconsistent with the underlying value columns.

Repair formula

Why this fixes it (short)
- It restores the same pattern used on other rows (I10,I12–I14). Totals (I15 and any roll-ups) will now update automatically when F11–H11 change and remain consistent with the rest of the sheet.
