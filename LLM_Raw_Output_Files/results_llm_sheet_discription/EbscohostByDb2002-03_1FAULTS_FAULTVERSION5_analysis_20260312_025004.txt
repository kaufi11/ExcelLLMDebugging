Findings — error
- On the KauCC sheet cell E20 (Image Collection) the formula is =SUM(D20:D20) — it only sums the PDF column. It should include both Full‑text (C20) and PDF (D20).
- That makes the per‑database Total (E20) show 0 instead of 112. If anyone sums the per‑database totals (E10:E28) the sheet total will be understated by 112.

Quick fix (one step)
- Edit KauCC!E20 and replace =SUM(D20:D20) with =SUM(C20:D20) or simply =C20+D20.

Why this fixes it
- Per‑database totals are defined as Full‑text + PDF. The current formula excludes the Full‑text value (C20) so the per‑database total is wrong. Changing the formula restores E20 to C20 + D20 (112 + 0 = 112).

Recommended consistency / prevention
- Standardize formulas:
  - For each database row put E# = C# + D# (or =SUM(C#:D#)) and copy down for all rows.
  - Compute column totals with:
    - B29 = SUM(B10:B28)
    - C29 = SUM(C10:C28)
    - D29 = SUM(D10:D28)
    - E29 = SUM(E10:E28) (or =C29+D29)
- To find similar errors fast: on each sheet press Ctrl+` to show formulas, or use Home → Find & Select → Go To Special → Formulas and inspect column E for inconsistent formulas.
