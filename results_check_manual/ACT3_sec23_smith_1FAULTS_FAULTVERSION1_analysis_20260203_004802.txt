I looked through Sheet1 and found a few consistent mistakes. Short summary, root cause, and exact fixes below.

Problems
1) Column E (Week 4) is wrong
   - E3:E8 contain =AVERAGE(Bx:Dx). Header says "week 4" so these should be the week‑4 hours (numbers), not averages. Because of that F (totals), E10 (weekly total), and overtime are all wrong.

2) Overtime formulas (column G) use the wrong row references
   - Example: G3 ends with MAX(E4-40,0) — that should be E3. Many G formulas are off by one (and G8 even references E10). This makes overtime values incorrect.

3) Gross pay formulas double‑count overtime and some reference the wrong rows
   - They use (F + G) * rate. Since G is overtime hours already included in F (total hours), (F+G) counts overtime twice.
   - Some C formulas use wrong F/G rows (e.g. C16 uses F5+G4).

Quick fixes (exact formulas to put into the sheet)

B) Correct overtime formulas (G3:G8)

C) Correct gross pay formulas (C15:C20)
- Assuming overtime is paid at 1.5× (common rule): gross pay = total_hours*rate + overtime_hours*rate*0.5

Why this fixes it (brief)
- Week‑4 must be real hours to compute accurate weekly totals.
- Overtime must be calculated using the four week values from the same row (no off‑by‑one references).
- Gross pay formula uses total hours once and adds only the overtime premium (0.5× rate for overtime) so overtime is not paid twice.
