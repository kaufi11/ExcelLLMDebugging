Findings
- Error: Cell I11 contains a hard-coded value 3 instead of a formula. All other rows in column I use SUM(Fx:Hx).
- Effect: The total I15 (=SUM(I10:I14)) can be wrong if the payment values in F11:H11 do not sum to 3. This breaks automatic recalculation and can produce incorrect totals.

Why it's wrong (short)
- Totals must be calculated from the payment columns. A fixed value in I11 will not update if F11, G11 or H11 change and will give a misleading grand total.

Fix (simple)
1. Select cell I11.
2. Enter the formula: =SUM(F11:H11)
3. Press Enter.

Optional cleanup and checks
- Copy the same formula down I10:I14 (if any of those were accidentally hard-coded).
- Verify I15 updates correctly (should equal SUM(I10:I14)).
- To find other accidental hard-coded totals: Home → Find & Select → Go To Special → Constants (uncheck Text/Logicals/Errors if you only want numbers).
- Consider protecting formula cells (Review → Protect Sheet) to prevent accidental overwrites.

Why this fixes it
- Replaces a manual number with a formula so the row total always equals the sum of its component payment cells and the grand total stays accurate and automatic.
