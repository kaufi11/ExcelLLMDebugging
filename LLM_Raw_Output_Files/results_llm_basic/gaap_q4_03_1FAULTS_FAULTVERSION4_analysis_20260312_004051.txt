- Many H-column “Change” cells (especially H11) are hard-coded or blank/0 instead of using a percent-change formula.

- General: several percent-change formulas do not guard against division-by-zero and will show #DIV/0! (or produce misleading results when prior-period value is zero or negative).

What to change (exact fixes and why):

1) Fix D14 on Q4 2003
- Replace: =COUNT(D9:D12)
- With: =SUM(D9:D12)
Why: COUNT returns the count of numeric cells (4), not the sum of amounts. SUM gives the intended Free cash flow total.

2) Fix any H-column percent-change cells (use one of these patterns)
- If you want a blank when prior = 0:
  =IF(F9=0,"",(D9-F9)/F9*100)
- If you prefer N/A instead of blank:
  =IF(F9=0,NA(),(D9-F9)/F9*100)
- If you want percent of magnitude (use ABS of denominator):
  =IF(F9=0,"",(D9-F9)/ABS(F9)*100)
Apply the appropriate formula to H9, H11, H12, H14 etc on each sheet (change the row reference to the row in that sheet).
Why: Protects against divide-by-zero and makes the intent clear. It also ensures H11 (accounts receivable securitization) calculates the % change instead of showing 0.

3) Fix the hard-coded arithmetic in Q3 2003-4 Qtr L11
- Replace the text/formula "= -80+40" with the correct historical value (or link it to the proper source cell).
- If the intended result is -40 and that is the correct historical number, put -40; if the correct historical number is -130 (or something else), use that.
Why: Hard-coded math strings cause confusion and may be wrong for audit trail. Use real values or links.
