Summary — main problems I found
1) Inconsistent / wrong ranges in the summary rows (rows 69–72).  
   - Examples: B69 =MAX(B33:B68) but B70/B71/B72 use B2:B68. C69 and D69 have the same problem. N69 and Q69 use N17:N68 / Q17:Q68 while the AVERAGE/STDEV rows use N2:N68 / Q2:Q68.  
   - Some MIN formulas include the summary row (e.g. N70 =MIN(N17:N69) includes N69), which is wrong and can create misleading results or circular references.

2) Operator‑precedence mistakes in some N cells (HW4).  
   - Formulas like =17+5/2 are evaluated by Excel as 17 + (5/2) = 19.5. If you intended the average (17+5)/2 you must add parentheses.

3) Possible “formula as text” entries.  
   - Several cells are shown as “FORMULA =20+4”. If the cell literally contains the text "FORMULA =20+4" it will not calculate; the real formula must start with = (e.g. =20+4).

What this causes
- Wrong maxima/minima/averages/stdev values because some input rows are left out or summary rows are accidentally included.
- Wrong numeric results where parentheses are missing.
- Some cells not calculating at all if formulas are stored as text.
