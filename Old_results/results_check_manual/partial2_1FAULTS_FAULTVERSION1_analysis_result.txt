2) Averaging inconsistency (logical/design issue)
- Column I uses formulas like =(F# + G# + H#) / 3 for every row. Excel treats blank cells as zero in that sum, so rows with only 1 or 2 homework entries are still divided by 3. That produces artificially low averages when some homework cells are blank. Examples:
  - Row 3: F3 is blank, G3=100, H3=100 → I3 = (0+100+100)/3 = 66.6667. If the intent is to average only the existing homework scores, the correct average would be 100.
  - Row 12: only G12 = 100 (F12 and H12 blank) → I12 = 100/3 = 33.3333 (probably not intended).
- Recommendation (choose one depending on intended policy):
  - If you want to average only non-empty homework scores: replace =(F# + G# + H#) / 3 with =AVERAGE(F# : H#). This ignores empty cells.
  - If blanks should count as zeros (explicitly), keep current formula or use =SUM(F# : H#)/3 (explicit).
  - If you want to divide by the number of non-empty homework cells but treat missing as absence differently, use: =IF(COUNT(F# : H#)>0, SUM(F# : H#)/COUNT(F# : H#), 0)
- Action: update the I-column formulas consistently across all rows to the appropriate formula (I recommend =AVERAGE(F# : H#) if blanks mean "not submitted").
