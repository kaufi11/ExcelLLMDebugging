Summary — main errors found
1) Sheet1!T19 — Wrong formula
- Fault: T19 = COUNT(D19:S19). COUNT returns the number of numeric cells, not their sum.
- Effect: T19 is a small integer (count) instead of the total points for that student. That wrong value feeds CB19 (CB19 = T19) and all downstream %/overall calculations (U19, CA19, CE19, CF19, etc.), so the student's totals and averages are wrong.
- Fix: Replace with a sum:
  =SUM(D19:S19)
- Why: SUM adds the assignment points; COUNT does not.

2) Data-entry / text placeholders inside numeric ranges
- Fault: Several numeric ranges that are summed include text like "x", "X", "+", "#" or "." (examples: BG7:BL7, BR7, BR9, BL15, AL7, AZ13, etc.). SUM ignores text so totals may silently omit intended values; if those placeholders were meant to represent 0 or missing points, it's better to use numeric zeros or a standard marker.
- Effect: Totals may be ambiguous or misleading; other formulas that expect numbers may show unexpected results.
- Fixes (choose one):
  - Replace text placeholders with 0 (recommended if they mean zero).
  - Or use a consistent numeric code (e.g., -1) and document it, or a comment explaining meaning.
  - If you must keep a text marker, make totals explicit with error-trapping, e.g.:
    =SUMIF(range,"<>x",range)  (not ideal)  
  - Best: clean the data so SUM ranges contain only numbers.
