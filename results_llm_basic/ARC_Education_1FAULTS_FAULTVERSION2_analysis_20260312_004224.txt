1) Ratios entered as text instead of calculated numbers (Student - Teacher sheet)
- Problem: Ratio cells contain text like "15.5-1" or "16-1" (or "**") instead of being calculated. That prevents correct sorting/filtering/aggregations and is error-prone.
- Fix: Use a formula to compute attendance ÷ teachers, then display as a numeric ratio. Example (for Clayton 1990-91): in D7 use:
  - =B7/C7
  Then format to one decimal (or use text display):
  - If you want "15.5-1" text: =ROUND(B7/C7,1)&"-1"
  - Prefer numeric for calculations: =ROUND(B7/C7,1)
- Why: Keeps values numeric so you can compute averages and totals correctly.

2) Mis-typed attendance values with misplaced decimal (e.g. BUFORD E15 = 1.548)

- Fix: Replace the incorrect value with the correct integer (likely 1548). If you don’t know the correct value, check the source. You can also inspect adjacent years (B34/K34 etc) to infer the intended value.

4) Percent values stored inconsistently (Teachers With Advanced Degrees sheet)
- Problems:
  - Early-year percent columns show values like 50.9 (meaning 50.9%), but in later years percent cells show decimals like 0.4690 (46.90%)—these are the same data expressed differently. Also some total counts (Total No.) have decimals (e.g. 1995.8) because percent × staff produced fractions.
- Fix:
  - Decide one representation: either percentages (0.469) formatted as % or percentage numbers (46.9). Recommended: store true fraction (0.469) and format as Percentage with 1 decimal.
  - To apply: select the percent cells (e.g. B21:B31, D21:D31, etc.) and apply Percentage format (1 decimal) OR multiply the decimal values by 100 if you want the 46.9 stored directly.
  - For counts showing decimals (Total No.), convert to integers with ROUND, e.g. replace formula with =ROUND(percent_cell * staff_total_cell,0).
- Why: Consistent percent formatting avoids confusion and fractional headcounts.

5) Missing or placeholder data (“**”) — behavior and handling
- Problem: Many cells use "**" for unavailable data. SUM ignores text, but formulas that expect numeric input (ratios, averages) will error if they reference "**".
- Fix:
  - Replace "**" with blank cells (delete) or use NA() or an explicit error-handling formula.
  - If you must keep placeholders, wrap formulas in IFERROR/IF to skip them, e.g.: =IF(OR(B7="**",C7="**"),"",B7/C7)
- Why: Keeps calculations robust and avoids wrong results.

6) Misaligned or inconsistent columns (headers vs. data)
- Problem: In some blocks headers and data appear shifted (e.g. the 1999-2000 block header columns differ), causing numbers to be in the wrong column relative to the header.
- Fix: Visually inspect each multi-year block to ensure header labels are above the correct data columns. If columns are shifted, cut/paste to realign or move header row so it sits above the intended data columns.
- Why: Misalignment causes wrong interpretation of which year a number belongs to.
