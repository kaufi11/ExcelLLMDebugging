Thanks — I reviewed the workbook description and found several clear errors/bugs and what to change. Short, clear list below with why and the fix.

1) D6 (Expense yearly growth)
- Error: D6 contains =(C6-728)/728 — uses the constant 728 instead of prior-year expense.
- Why it’s wrong: growth should compare C6 to C5 (previous year), not to 728.
- Fix: change to =(C6-C5)/C5

2) E7 (fee revenue for 2001) — forward reference
- Error: E7 = E8 - 2258280 (it references the next row). That’s a forward reference / brittle.
- Why it’s wrong: historical values should not be derived from future rows; this is confusing and may break if E8 changes.
- Fix / options:
  - If you have the actual 2001 fee revenue, enter that value directly in E7.
  - If you want to derive 2001 from 2002 using your assumed growth rate (A3: 4%), use =E8/1.04
  - If you want it from an explicit growth cell, use =E8/(1+<growth_cell>)
- Explanation: using prior-year or an explicit growth divisor removes the forward reference and is logically correct.

3) E16, E17, E18 (dollar amount of premiums for groups that hit the cap)
- Errors:
  - E16 =(0.46*C7) and E17 =(0.43*C8) and E18 =(0.46*C9) — these multiply a percent by a cell that is not the total premium base (and E18 uses the wrong percent constant, 0.46 instead of 0.35).
  - They appear to multiply percent × group count or percent × expense, which is meaningless.
- Why it’s wrong: the header for E-column is “$Amt (% of total prem)”. That means you must multiply the percent (D16..D18) by the total premium dollar amount for that year — not by a count or by an expense cell.
- Fix: Replace those formulas to multiply D16..D18 by the correct total-premiums cell for each year. Example patterns:
  - If total premiums for 2001 are in cell X7, then E16 = D16 * $X$7
  - If total premiums for 2002 are in X8, then E17 = D17 * $X$8
  - If total premiums for 2003 are in X9, then E18 = D18 * $X$9
- If you don’t have a total-premiums column, create one and populate it, then reference it in E16:E18.
- Also correct E18 to use D18 (0.35) rather than hard-coded 0.46.
