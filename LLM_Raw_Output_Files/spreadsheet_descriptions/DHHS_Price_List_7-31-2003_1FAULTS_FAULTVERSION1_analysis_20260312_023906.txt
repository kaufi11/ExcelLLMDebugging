Overview
- Workbook contains 3 sheets: "Exhibit A Updated 7_31_2003" (primary), "Sheet2" and "Sheet3" (no details provided).
- Main sheet is a vendor price/support schedule for the Department of Health and Human Services (Oracle products), organized as groups (Databases, Application Servers, Tools, Integration, Data Warehousing, Applications by functional area, etc.), followed by "GENERAL LICENSING RULES" and notes.

Layout / key columns (left → right)
- Column A: Program / Product name (many rows grouped by category).
- Column B: Qty (mostly 1).
- Column C: Term (mostly "Perpetual").
- Column D: Licensing Type / Unit (e.g., Named User, Processor, Universal Power Unit, Application User, Computer, Workstation, etc.).
- Column E: List Price (unit list price / base price).
- Column F: Support 1 year (typically calculated as =E * 22% in most rows; some rows have hard-coded values).
- Column G: Licensing fee or other computed fee (most common formula =E * 0.2 for NAMED/UPU, =E * 0.5 for PROCESSOR; some product-specific multipliers like =E * 0.3 appear).
- Column H: Updates (typically =G * 15%).
- Column I: Product (additional support?) (typically =G * 7%).
- Column J: Total Updates / Total 1-year (typically =H + I).
- Column K: Notes / Minimums (text notes, minimums and special instructions).

Common formula patterns
- F (support 1 year) ≈ =E * 22% (applies to the vast majority of rows).
- G (fee used to compute updates/support breakdown) is usually:
  - =E * 0.2 for Named User / Universal Power Unit / Application User / Computer, etc.
  - =E * 0.5 for Processor-type licensing.
  - occasional exceptions (e.g., a processor row using 0.3).
- H = G * 15% (updates).
- I = G * 7% (product/support component).
- J = H + I (sum of the two components).

Notable values / high-cost items

- Many standard application items have moderate list prices (e.g., typical Application User = 3,995; various options in low hundreds).

Grouping and notes
- Products are grouped by functional headings (e.g., "Oracle Database", "Application Servers", "Tools", "Integration Products", "Data Warehousing Products", "Applications" with subgroups like Procurement, Financial, HR, etc.).
- Rows A129–A146 and following list General Licensing Rules and explanatory notes (11 numbered items) that explain UPU minimums, option matching rules, gateway inclusions and other licensing constraints.
