Short answer — main error found
- On sheet "A-122" cell C17 has the wrong formula: it’s =C16-1 instead of =C16+1.
- That one “-1” breaks the whole sequence in column C (creates duplicate numbers and an off‑by‑one shift for subsequent entries).

Why it’s a problem
- Column C is the A‑122 paragraph numbering used elsewhere for cost‑principle mapping. A wrong number will mislabel cost principles and can produce incorrect mappings/reports.

Fix (quick)
1. Go to sheet A-122.
2. Select cell C17.
3. Change the formula to =C16+1.
4. Fill/drag C17 down through the rest of the C column sequence so the +1 pattern continues.
5. Recalculate (F9) and check that the column now reads a clean increasing sequence.

Better (robust) alternative — prevent recurrence
- Replace the incremental formulas with a stable formula using ROW or a fixed start value. Example (if C5 = 29 and C5 is in row 5):
  - In C5 put 29 (as now), and in C6 use: =C$5 + ROW() - ROW(C$5)
  - Or in any cell use: =29 + (ROW()-5)
- This avoids manual +/- mistakes when inserting rows.

Extra check to run
- Use Find (Ctrl+F) to search “-1” in formulas or use Formula Auditing → Show Formulas to scan for other accidental minus signs in sequences.
