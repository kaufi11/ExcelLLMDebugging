Short summary of the errors, why they matter, and exact fixes.

Errors found
1) Typo / wrong divisor in M3
   - M3 = L3/944 is clearly wrong (should be dividing by number of homeworks). This distorts M3 and the column-average M47.

2) Inconsistent averaging method (all row averages use a fixed divisor 10)
   - Most Mn use =Ln/10 even when some homework cells are blank. That under‑reports averages for rows with missing scores (SUM counts blanks as 0 but you probably want average of the present scores).

3) Missing / possibly mis‑aligned score cells in many rows
   - Several rows have blank cells in B:K (e.g., row4 missing E4 and J4, row25 missing B25, etc.). That creates ambiguous totals/averages (is blank = 0, or is it "not entered"?). If blanks were unintended (data shift), totals/averages are wrong.

4) Column averages (row 47) are affected by the above
   - M47 = AVERAGE(M3:M45) uses the bad M3 and the fixed-divisor Mn values, so it is not a reliable class average.

Why these fixes work (short)
- AVERAGE(Bn:Kn) ignores blanks — so it divides by the number of submitted scores only.
- COUNT(Bn:Kn) lets you divide by actual number of numeric entries when you need the total divided by entries (L n / COUNT(...)).
- Fixing the M3 typo removes the single outlier that was skewing column M's average.
