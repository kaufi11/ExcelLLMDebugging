3) Potential inconsistency in how "TOTAL CHARGE" is calculated
- Cells: C37 and F37
- Formulas: C37 = SUM(C12:C35)*0.9 ; F37 = C37 + SUMPRODUCT(C12:C35,F12:F35)
- Problem: C37 multiplies the total costs by 0.9 (presumably a 90% factor), but the SUMPRODUCT used to add mark-ups uses the full C12:C35 values (it does not apply the *0.9). - Impact: If the 0.9 was intended to scale all base costs used to compute revenues, adding SUMPRODUCT without the 0.9 factor will produce a mismatch between base and added markup; revenue may be overstated/understated depending on intent.
- Fix: Clarify intent. If the markup should apply to the 90%-adjusted base, change the SUMPRODUCT to SUMPRODUCT(C12:C35*0.9,F12:F35) (or wrap C12:C35 with the 0.9 factor). If the 0.9 only applies to one part of the calculation, leave as-is but document rationale.

4) NPV calculation is verbose / brittle — use PV or compact annuity formula
- Cell: F85
- Formula found (condensed): =NPV($C$69,IF($C$65<1,0,F$80),IF($C$65<2,0,F$80),...,IF($C$65<11,0,$F$80)) - $F$76
- Problems & observations:
  - The formula repeats the same cashflow (F80) for each possible year up to 11 arguments — this is hard to maintain and error-prone.
  - It includes an 11th-year IF even though the sheet indicates a maximum lease length = 10; the last IF uses $F$80 with an absolute reference (harmless but inconsistent).
- Impact: It works for the current input but is fragile if the model horizon changes. Also it's unnecessarily long and can lead to mistake if someone increases lease length beyond the hard-coded IFs.
- Better approach:
  - Because the yearly cash flows are level (F80 each year), compute present value of an annuity: =PV($C$69,$C$65,-F$80,0,0) - F$76
    or
  - Compute NPV as F$80 * (1 - (1+$C$69)^(-$C$65)) / $C$69 - F$76
- Fix: Replace the long IF list with the PV-based formula above for clarity and robustness.
