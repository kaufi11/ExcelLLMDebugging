Summary of the problem
- Many L-cells on Ap1 / Ap2 / Ap3 reference the wrong Rating-cell on the Rate sheet. These are copy/paste offset errors so the numbers that feed the Summary are wrong.
- One hard-coded value: Ap3!L20 = 5 (should be a formula like the other proficiency cells).

Why that breaks things
- Each Bxx on the Ap sheets is a description taken from Rate!G11:G20. The matching score must come from the same row in column E (Rate!E11:E20). Wrong E-row references mean wrong weighted scores, so Summary totals are incorrect.

How to verify
- Change one Rate!E cell (for example Rate!E16 from 100 to 200) and confirm the corresponding Ap L-cell and Summary number update accordingly.
- Scan Ap sheets for any other cells where the Rate!E index does not match the Rate!G index on the same row — fix with the same pattern.

Optional improvement (prevents this class of errors)
- Instead of hard-coded Rate!E11/E12... use a single lookup: place the key label (e.g. the text from Rate!Gxx) in Ap, then use INDEX/MATCH or VLOOKUP to fetch the matching Rate!E value. That makes the formula resilient to moved rows and copy/paste mistakes.
