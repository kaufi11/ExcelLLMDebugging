High-level summary of the workbook (concise):

Overall
- Purpose: Toxic Release Inventory (TRI) emissions and offsite transfer data for Silicon Valley (annual rows by year). Values are mostly in millions of pounds (converted from raw pounds by dividing by 1,000,000).
- Common formula patterns: conversions (cell = raw_pounds/1000000), running year increments (A[n] = A[n-1] + 1), SUM for totals, simple differences and percentages (e.g., N = L - M, O = N / L).
- Layout style: each sheet uses repeated row-per-year blocks with consistent columns for Air / Land / Water / Offsite / Total Releases and separate columns for Transfers (Recycled, Energy recovery, Other treatment, Incineration, total).

Sheet: 2003 Report
- Title cell A3 indicates "TRI - In Millions of Pounds".
- Main columns (example row headings): A = Year, B = Air, C = Land, D = Water, E = Offsite Release or Disposal, F = Total Releases; separate transfer section J: Year, K = Recycled, L = Energy Recovery, M = Other Treatment, N = Incineration, O = total transfers.
- Data rows shown: 1987–2000. Example key values:
  - 1987: Air 9.468937, Land 0.0024, Water 0.168488, Offsite 3.60004, Total Releases F7 = 13.239865. Transfers O7 = 14.847612 (K7–N7 shown).
  - 1999: Total Releases F19 = 4.255724; Transfers O19 = 26.282264 (large transfer amount).
  - 2000: Total Releases F20 = 4.790791; Transfers O20 = 23.790164. (Row 20 also contains extra columns G/H with values 8.449074 and 0.638154… — outlier cells.)
- No explicit formulas listed in this snippet for 2003 sheet cells, but follows the same division/SUM pattern elsewhere.

Sheet: 2002 Report
- Header: "Toxic Release Inventory Emissions" and "Millions of Pounds" notes; File: GTRI2002.xls.
- Structure: two parallel blocks — TRI releases (columns B–E, F–H raw pounds) and disposed/recycled/incinerated offsite (B/C/D in second block).
- Repeated formula pattern: B[n] = F[n]/1,000,000 (convert raw pounds in F column to millions in B), C[n] = G[n]/1,000,000, D[n] = H[n]/1,000,000, E[n] = SUM(C[n]:D[n]), years increment with A[n] = A[n-1] + 1.
- Additional summary columns: R computes totals (R11 = B11+C11+D11), R24 = (R11-R23)/R11 (a percent change).
- Zip-code JV data block (A43 down): columns B–K = pounds per zip; L = TOTAL POUNDS (SUM(B:K)/1,000,000), M = SVEP (some subset SUM(B:D)), N = L - M, O = N / L (fraction), P/Q = # companies, R sometimes converts N back to pounds (N*1,000,000).
- Many rows include NA entries where data missing.

Sheet: 1999 Report
- Very similar structure and formulas to 2002 Report (same conversion pattern: B = F/1,000,000, etc.).
- Contains the disposed / recycled offsite block (years 1987–mid‑1990s) with F/G as raw pounds and B/C = converted millions.
- Repeats the same layout for offsite recycled/incinerated summary.

Sheet: data
- Compact summary of incinerated (col B) and recycled (col C) by year with:
  - A column = year, B = incinerated (raw pounds), E = total recycled & incinerated (raw pounds), C = formula E - B (recycled = total − incinerated).
- Years present: 1991–1999 (E values are total combined, B given, C computed).

Sheet: notes
- Duplicate/notes summary of incinerated and recycled for 1997–1999 (same values as data sheet); likely for easy reference.

Key formulas/patterns to note
- Conversion from raw pounds to millions: =raw_pounds/1000000 (frequent: B = F/1000000, etc.).
- Totals: =SUM(range) used to aggregate zip-code columns and air/land totals.
- Year incrementing: A[n] = A[n-1] + 1 for sequential years.
- Differences/percent: N = L - M (difference between total pounds and subset), O = N / L (share).
- Some ratio/change calculations: R24 = (R11 - R23) / R11.

Data quality/observations (brief)
- Units are millions of pounds on most summary columns; raw pounds retained in adjacent columns on some sheets.
- Some NA entries and occasional extra values (e.g., G20/H20 on 2003 Report) that look like intermediate calculations or misplaced cells.
- Repeated structure across "1999 Report" and "2002 Report" indicates consistent conversion/aggregation methodology.
