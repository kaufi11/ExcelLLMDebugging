Sheet: Masterlist — overview
- Structure: single sheet named "Masterlist" with two main columns:
  - Column A: numeric ID/index for each row (mostly auto-incremented).
  - Column B: course titles and occasional category headers.

- Layout and sections (examples):
  - Top banner/header: B3 = "Skillbuilder DX I.T. Courses".
  - Main IT course list runs from row 4 downward (B4…B162) with corresponding numeric IDs in A4…A162.
  - B164 = "Desktop Courses" — desktop course block runs B165…B188 with IDs in A165…A188.
  - B190 = "SAP Courses" — SAP block runs B191…B244 (many titles prefixed with numeric SAP codes).
  - B246 = "Business and Professional Development Library" and B247 = "(Soft-Skill) Courses" — soft-skill block runs B248…B307 with IDs in A248…A307.
  - A313 contains the text "ORDER 3 List"; an additional block begins at A314/B314 and continues B314…B327 (financial/technical courses).

- Formula pattern:
  - Most cells in column A use a simple increment formula of the form =+A(previous)+1 (e.g., A5 =+A4+1), producing sequential integers.
  - Some formulas omit the leading plus sign (e.g., A165 = A162+1) — functionally equivalent.
  - A4 and A314 are hard-coded starting values (A4 = 1, A314 = 1).

- Notable anomalies / key values to check:
  - A63 is hard-coded as =5 (explicitly set), which breaks the sequential numbering (subsequent A64 uses =+A63+1 so continues from 5).
  - A313 contains the text "ORDER 3 List" (non-numeric), which will break numeric continuity if upstream formulas reference it.
  - Column B contains mixed content: category headers, course titles, and some course codes embedded in the titles (especially in the SAP section).
  - There are occasional typos in titles (e.g., "Introdoctory", "Developement", "Eror").

- Overall impression:
  - A two-column master catalogue where column A is intended as an automatic ID counter and column B stores hierarchical course/categorization text. The numbering approach is consistent and simple, but manual entries and resets (A63, A313, A314) create discontinuities that should be reviewed if strict sequential IDs are required.
