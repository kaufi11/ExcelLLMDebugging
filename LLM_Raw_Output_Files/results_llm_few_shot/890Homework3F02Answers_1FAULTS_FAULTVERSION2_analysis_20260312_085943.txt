2) NPV use / t=0 handling (consistency check)
- Location: Q1 a-b graph and many NPV rows (e.g. Sheet "Q1 a-b, graph" B7:B27 etc)
- Issue: NPV(rate, range) in Excel discounts only t=1...n. You must add the t=0 cash flow separately if present.
- Status in file: Most places do this correctly (NPV(...) + $B$3 or +$B$2). Just be careful — if you ever include t=0 in the NPV(...) range you will double-discount it or produce wrong sign.
- Recommendation/fix (if needed): Always use NPV(rate, cashflows_from_period1_onwards) + cashflow_at_t0.

3) Sign convention for depreciation / confusing sums (risk)
- Location: Q2 and WITH DEPRECIATION blocks (e.g. C39 = C49/C38, D44 = $C$39 and then SUM used)
- Issue: C49 is stored as -200 (project cost), dividing gives C39 = -100. Depreciation is negative in the model, so formulas use SUM(EBITDA, depreciation) instead of EBITDA - depreciation. That works numerically but is confusing and error-prone.
- Recommendation/fix: Use a consistent convention:
  - Store depreciation as a positive expense (e.g. C39 = -C49 / C38 → OR better: change C49 to positive 200 and set C39 = C49/C38).
  - Then compute EBT explicitly: EBT = EBITDA - Depreciation (not SUM).
- Why: Clear sign convention avoids accidental double negatives and makes formulas easier to audit.

4) IRR multiple-solution risk (conceptual)
- Location: Q1 and Q1 c-d blocks where cashflow signs change
- Note: When cashflows change sign more than once IRR may have multiple roots or none. Excel's IRR returns one root (depends on guess). This is not a spreadsheet error but a mathematical property — use NPV profile or choose NPV rule.
- Recommendation: Use NPV curve (NPV at different discount rates) or XIRR with a carefully selected guess, and report multiple possible IRRs if they exist.
