I examined all formulas in the workbook you provided and found a few real problems and a handful of minor inconsistencies (range off-by-one, missing data used by formulas, inconsistent rounding). Summary below with exact cells and recommended fixes.

1) Primary (real) error — mismatched SUM vs COUNT ranges in "900 Class"
- Problem:
  - L81 = =SUM(L4:L39)+SUM(L44:L79)
  - L82 = =COUNT(L4:L40)+COUNT(L44:L79)
  - These ranges do not match: the SUM excludes L40 while the COUNT includes L40.
- Effect:
  - L83 (=L81/L82) uses a sum that misses an item while the denominator counts it → wrong average. That propagates to L85 (adjustment) and therefore to every M# formula that uses L$85 (all adjusted exam scores in this sheet).
- Recommendation:
  - Make the SUM and COUNT cover the same cells. If L40 is a data row that should be included, change L81 to include L40 (e.g. SUM(L4:L40)+SUM(L44:L79)). If L40 is a header/should be excluded, change L82 to COUNT(L4:L39)+COUNT(L44:L79).

Those blanks reduce the COUNT or are treated as 0 depending on formula — verify whether blank should be excluded (COUNT) or treated as zero (SUM).
