Summary of the actual error I found
- Cell BH43 contains the constant 7 instead of the pattern used everywhere else: IF(Q$2="x",Q43,0). That is the single clear wrong entry in the sheet you posted.

Why this is wrong / the impact
- All the CL/CM totals on each row are built by summing those IF(...) cells (AU..CI). Putting a constant 7 in BH43 forces that row (and every aggregation that includes it: CL43, B43, B46, the I52/J52 totals and the final comparisons) to be too large. In short: totals and comparisons that include row 43 are inflated and therefore incorrect.

Exact fix
- Replace BH43 (currently =7) with the consistent formula:
  =IF(Q$2="x",Q43,0)

Why that fix works
- The sheet’s pattern is: for each course-column the cell either includes the course’s rating (if that course is selected with an "x" in row 2) or 0. Using IF(Q$2="x",Q43,0) restores the intended conditional inclusion of Q43 only when that course is active. Then row-sums and global totals will calculate correctly.
