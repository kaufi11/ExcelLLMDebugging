Spreadsheet: "Exhibit A Updated 7_31_2003" (plus Sheet2 and Sheet3 — no detail provided for those)

Overview
- Single large price/fees table organized by product groups (Database, Application Servers, Tools, Integration, Data Warehousing, Applications, etc.). Product rows run A8:J126+ with intermittent section headers in column A (e.g., "Application Servers", "Tools").
- Final section (rows ~129–146) contains "GENERAL LICENSING RULES" and numbered notes referenced from column K.

Column layout (A:K)
- A: Program / Product name (section headings appear in A36, A44, A54, A60, etc.)
- B: Qty (mostly 1)
- C: Term (mostly "Perpetual")
- D: Type (license metric: Named User, Processor, UPU, Application User, Computer, etc.)
- E: List Price (base unit price or metric price)
- F: Formula/value (generally =E * 22% but some rows are hard-coded numbers)
- G: Formula/value (generally =E * 0.2, but several rows use other multipliers (0.3, 0.5) or contain precomputed values)
- H: Formula = G * 15% (labelled as "Only" or "Updates" in header area)
- I: Formula = G * 7% (labelled as "Support" or "1 Year")
- J: Formula = H + I (labelled "Total Updates and Product Support")
- K: Notes / Minimums (references to Note 1..11 or minimums such as "200 min.", "1 min.")

Common formula patterns
- F: =E * 22% for most rows (service/fee column). Some rows have numeric overrides (e.g., F66=1100, F75=7.7, F98=1538.9, F118=6599).
- G: =E * 0.2 for most rows (base used to calculate H and I). Exceptions:
  - Processor-license lines often use a higher factor (e.g., G10 = E10 * 0.5; G17, G19, G24, G29, G32, G35, etc. use *0.5).
  - A few rows use other factors: G39 = E39 * 0.3; G21 is recorded as =E21/0.5 (likely an error — should be *0.5).
  - Some G cells are precomputed numbers (not formula).
- H: =G * 15% (updates-only portion)
- I: =G * 7% (support 1-year portion)
- J: =H + I (total of updates+1yr support)

Notable values / examples
- Large list prices: E51 (120,000), E50 (95,000), E118 (29,995), E39 (20,000), E19 (20,000), E24 (10,000), E17 (10,000), E10 (40,000).
- Example calculation (row 10): E10=40,000 → F10 = 8,800 (E*22%), G10 = 20,000 (E*0.5), H10 = 3,000 (G*15%), I10 = 1,400 (G*7%), J10 = 4,400 (H+I).
- Some rows show F or G already multiplied (e.g., F98=1538.9 while E98=6995; F98 ≈ E98*0.22).

Inconsistencies / potential errors to review
- G21 uses =E21/0.5 (division) instead of expected multiplication by 0.5.
- Mixed use of multipliers in G: 0.2 (most), 0.3, 0.5; confirm these are intentional for each license metric (Processor vs Named User vs Computer vs UPU).
- Several F and G cells are hard-coded numbers instead of formula — check for manual overrides and consistency (rows: 66, 75, 84, 85, 86, 87, 88, 89, 91, 92, 93, 95–97, 98, 100–107, 111–114, 116–118, etc.).
- Some header labels are ambiguous (top row shows multiple "DHHS" headers); verify intended column headings.

Notes and references
- Column K references Note 1..11; the "GENERAL LICENSING RULES" section (rows ~129–146) expands on those notes (minimums, UPU rules, matching options to DB licenses, approvals required, etc.).
- Many products have minimum-licensing rules listed in K (e.g., minimum UPUs, minimum user counts, minutes, etc.).
