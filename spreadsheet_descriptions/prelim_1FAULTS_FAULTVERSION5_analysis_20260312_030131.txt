Summary of workbook structure and contents

Sheet: program
- Purpose: master conference program (chronological event list).
- Header/meta (top rows): A1 = conference title; co-chairs and organizers in A3–A7 and F7–F8; venue/date: A7 = "MIT - 10, 11 and 12 November 2003".
- Main table headers (row 11): A = EVENT, B = DAY, C = START (time), D = END (time), E = TYPE, F = TOPIC / SPEAKERS, G = Length (minutes), H = ROOM.
- Typical rows: events listed sequentially (registration, sessions, breaks, lunch, keynote, dinner, guided tour). Times are stored as Excel times (e.g., 08:30:00).
- Formulas: Column A mostly autogenerated with =Aprevious+1 (e.g., A13 = A12+1, A14 = A13+1, etc.) to number events sequentially.
- Notable anomalies:
  - A36 uses =A35+676 (large unexpected jump).
  - A41 is entered manually as 26 (break in the formula-driven sequence).
- Key values: keynote row (A31) lists Joan Blades and Jane Fountain; conference dinner row (A33) 20:00–22:30; MIT guided tour at A41 / A119 (14:45, Lobby 7).
- Rooms: Mostly "66-110"; posters use "56-191"; some AV room "7-431 (AVT)".

Sheet: schedule
- Purpose: detailed paper/session schedule, authors and paper titles.
- Column mapping:
  - A: day/session headings and time ranges (text rows like "MONDAY 10", "Monday 10, 08.30 - 09.00AM").
  - B: PAPERID
  - C: SESSIONID (e.g., "5-poster", "1-1")
  - E: AUTHOR(S)
  - F: INSTITUTION
  - G: AFFILIATION (sometimes)
  - H: COUNTRY
  - I: TOPIC (session theme)
  - J: PAPER TITLE
- Content layout: blocks separated by session headings (Session I, II, III, etc.), with each paper row listing PAPERID, SESSIONID, author, affiliation, country, session topic and paper title.
- Cross-reference: SESSIONID and TOPIC values correspond to program session types/topics (can be cross-checked against program sheet).
- Key entries: conference opening and welcome (Opening row shows ICPPIT chairs); keynote details on Tuesday (Joan Blades, Jane Fountain) and conference dinner (Tuesday 20:00, restaurant in Cambridge); Wednesday sessions in ROOM 7-431.

General notes
- Time formatting is mixed in descriptive text (AM/PM strings) but the program sheet uses Excel time values in columns C/D.
- The workbook is organized for human-readable program printing and for cross-referencing papers to sessions via SESSIONID; the main structural issue to watch is the broken event-number sequence in program column A (A36 and A41).
