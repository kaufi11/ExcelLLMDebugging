Spreadsheet overview (concise)

Sheets
- Intro
  - Title and project description (DUCK94 nearshore experiment; sponsors, date range).
  - Master experiment index table (headers in A6:D6): Experiment #/PI Code, Experiment Title, Investigators, Time period.
  - Lists experiments (rows A7:D11) linking to individual experiment sheets.

- RHBJune
  - Experiment metadata in row 1 (No. 2, PI code RHB, Month: June) and PI/co-PI names.
  - Instrument table (starts row 8) with repeated headers: Gauge ID, From/To dates, Description, Sampling Rate, Record Length, Interval, FRF Coordinates (Cross Shore, Longshore), Water Depth, Height Above Bottom, Gauge Depth, Data Type, Format, Additional Info.
  - Entries: piezometers PA0, PA1 and PD0–PD5; sampling 10 Hz, record length 4–6 hrs, location SIS / Pier, data digital. Some fields unknown (?) or “Varies”.
  - Notes: a question text in D19–D20 about number of sensors.

- RHBSept
  - Same structure as RHBJune but Month = September and date ranges differ (From/To 12–23).
  - Same piezometer IDs and 10 Hz sampling; M-type labeled “Time Series” and format “Digital”.
  - Same question about number of sensors in D18–D19.

- FRF
  - Experiment No. 21; title "FRF Measurements Program".
  - Instrument table (headers row 3–4): ID, Description, Rate, Record length, Interval, FRF Coordinates (Cross Shore F, Longshore G), Water Depth (H), Bottom (J), a computed column (I) using formulas, Type, Format, Additional Info.
  - Repeated instrument rows 111, 121, …, 511, 519, 529, 601, etc. Typical values: Rate 2 Hz, Record = 10240 sec, Interval every 3 hrs.
  - Key formulas: column I uses expressions like =SUM(H5-J5) (i.e., H - J) to compute the difference between Water Depth (H) and Bottom (J). One outlier uses =SUM(H20-100) instead of H20-J20.
  - Notable values / anomalies: negative coordinate in F23 (-927.3), several N/A entries (sensors without depth), inconsistent use of SUM() around simple subtractions.

- DOZRJune
  - Experiment No. 29, Month = June. PI: Brad Werner, Co-PI: Steve Elgar.
  - Instrument table similar to others. Pressure gauges p01–p08 (Setra capacitance) listed: sampling 2 Hz, continuous, FRF coordinates (cross-shore values like 100–150, longshore ~880–900), water depths and "Above Bottom" values (J / K / L).
  - Additional video and camera records listed (30 Hz for video), formats SVHS/Super 8/digital.
  - Some start/stop dates shown (C column), some fields “?” or n/a.

- DOZRSept
  - Same structure as DOZRJune but Month = September and date ranges differ (From/To 5–19). Same p01–p08 entries and video equipment listing.

Key constants and recurring values
- Sampling rates: piezometers 10 Hz (RHB sheets); many pressure gauges and FRF instruments 2 Hz.
- Record lengths: piezometers ~4–6 hrs; FRF instruments = 10,240 sec (≈2.84 hr).
- Common columns across sheets: ID, date range (From/To), Description, Sampling Rate, Record Length, Interval, FRF Coordinates (Cross Shore, Longshore), Depth, Height/Above Bottom, Data Type, Format, Additional Info.

Formulas and calculation notes
- Column I on FRF sheet: uses =SUM(H#-J#) which behaves as H# - J#; likely intended to calculate vertical offset (depth minus bottom). One cell uses =SUM(H20-100) — inconsistent and likely an error.
- No other explicit Excel formulas were provided in other sheets (mostly static values or “?” placeholders).

Data quality / anomalies to watch
- Several “?” and blank/Varies/N/A fields (intervals, record lengths, coordinates) indicating missing metadata.
- Inconsistent formula usage (SUM around simple subtraction, one hard-coded subtraction by 100).
- Some coordinate/depth values appear negative or unusual (e.g., F23 = -927.3).
- Repeated header rows and slightly different header placements between sheets (rows 6–9 vary).

Overall layout
- Intro is the index and descriptive front page.
- Each experiment has its own tab with metadata at the top and an instrument inventory table beneath; columns are consistent across sheets with minor variations.
- Emphasis is on instrument ID, sampling specs, FRF coordinates, depths, and data type/format.
