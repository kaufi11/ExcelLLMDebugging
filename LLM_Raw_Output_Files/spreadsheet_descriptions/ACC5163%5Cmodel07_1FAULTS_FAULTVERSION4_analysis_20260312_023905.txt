Worksheet name: MODEL07 — "PUSH‑DOWN WORKSHEET & CONSOLIDATION WORKSHEET (for Chapter 7)"

High-level overview
- Single worksheet that contains two linked working areas:
  - PUSH‑DOWN worksheet (starts around row 49) — used to reflect push‑down basis adjustments (old basis, adjustments, new basis).
  - CONSOLIDATION worksheet (starts at row 100) — used to prepare consolidation eliminations and to hold stored problem data (K105:U132).
- Extensive on‑sheet instructions and macro keys (rows 1–50 and 136–186) describe loading problem data, posting entries, and erasing columns.

Major blocks and layout (row references approximate)
1. Header & instructions (A1–A16; A138–A186)
   - File title, general instructions, macro key summary (macros A–G load problems; P posts basic elimination; Q posts push‑down entries; U/W/X erase data).

2. Analysis of investment account — Excess cost elements (rows ~15–23)
   - Area for assigning excess cost amounts by asset class (Inventory, Land, Bldg & Eq, Accum Depr, Patent, Goodwill, etc.).
   - Totals: B22 has FORMULA =SUM(C22:I22).

3. Entries to implement push‑down accounting (rows ~24–46)
   - Debit/credit list for push‑down journal entries; many rows flagged "(macro Q will post)" (macro Q posts to push‑down worksheet starting row 49).
   - Dr/Cr totals use SUM formulas: C39 =SUM(C27:C38), D39 =SUM(D27:D38).

4. PUSH‑DOWN worksheet (starts row 49)
   - Column layout: C = Old Basis, D = Adjusting Entries (Dr), E = Adjusting Entries (Cr), F = New Basis.
   - Repeating pattern for balance sheet lines (rows ~54–64): Fxx = (Cxx + Dxx − Exx). Total assets: F65 = SUM(F54:F64).
   - Proof: D76 = (D65 + D74) and E76 = (E65 + E74) to check debit/credit postings.

5. Analysis of investment account — Book value element & Basic elimination (rows ~78–96)
   - Book value totals: B85 = SUM(C85:F85).
   - Basic elimination entry lines (Common Stock, APIC, Retained Earnings, Revaluation Capital, Investment in Sub) are formulas referencing the analysis totals (e.g., C90 = $C$85, C91 = $D$85, C92 = $E$85, C93 = $F$85, D94 = $B$85).
   - Elimination totals: C95 = SUM(C90:C94), D95 = SUM(D90:D94).
   - Macro P posts the basic elimination to the CONSOLIDATION worksheet.

6. CONSOLIDATION worksheet and Problem Data (row 100 onward)
   - Header: "CONSOLIDATION WORKSHEET AS OF ..." and multi‑column keys for different problem columns (K:U).
   - Columns K:U (rows 105–132) store UNADJUSTED problem data for problems 7‑1 through 7‑7 (noted at A136: "PROBLEM DATA is stored to right ... K105 to U132").
   - The consolidation balance sheet area (rows ~104–132) inputs subsidiary/parent amounts keyed into K:U for each problem column. Many cells show sample numeric values (examples: K105=55000, K106=180000, K108=300000, K111=500000, K112=723000, K117=120000).
   - Consolidation row formulas mirror push‑down convention: for the consolidation output columns, Fxx = (Bxx + Cxx + Dxx − Exx), and totals for assets and liabilities use SUM across rows. Proof of posting: D134 = (D118 + D132), E134 = (E118 + E132).

Key formulas and rules (recurring patterns)
- New basis (push‑down): F = C + D − E (applies to most balance sheet lines).
- Consolidation computed columns: F (or analogous) = B + C + D − E (used to compute consolidated balances).
- Totals computed by SUM over account rows (e.g., C65 = SUM(C54:C64), C118 = SUM(C105:C117), etc.).
- Basic elimination references analysis totals with absolute references (e.g., $C$85, $D$85).
- One special formula: F70 uses a literal 295: F70 = (295 − D70 + E70) — a constant is included for Common Stock (verify intended value).

Macros (usage and effect)
- A–G: Load unadjusted problem data into the PUSH‑DOWN (and some also into CONSOLIDATION) from stored problem sets (A→7‑1, B→7‑2, ... G→7‑7).
- P: Post the basic elimination entry to the CONSOLIDATION worksheet (uses formulas in rows ~89–95).
- Q: Post unamortized excess cost reclassification entries to the PUSH‑DOWN worksheet (posts the entries listed in the push‑down entries block).
- U: Erase columns C, D & E of the PUSH‑DOWN worksheet.
- W: Erase column B of the CONSOLIDATION worksheet.
- X: Erase postings in columns D & E of the CONSOLIDATION worksheet.

Problem data region (K105:U132)
- Holds the numeric starting balances for multiple problems (7‑1 through 7‑7). These cells feed the consolidation columns and are referenced by the consolidation calculations.
- Examples of stored values (row: value): K105=55,000; K106=180,000; K108=300,000; K111=500,000; K112=723,000; K113=−123,000; K117=120,000. (Many more values across K:U are present.)

Important user actions and flow
- Load a problem with the appropriate macro A–G (Ctrl+letter) to populate K:U (and push‑down).
- Fill the analysis areas (excess cost and book value elements).
- Use macro Q to post push‑down adjustments into the PUSH‑DOWN worksheet (row 49 area).
- Use macro P to post the basic elimination entry into the CONSOLIDATION worksheet (row 100 area).
- Verify debit/credit proof rows (D76/E76 and D134/E134).

Notes / cautions
- Many rows are placeholders for manual posts (some lines are marked "post manually").
- Check the special constant in F70 (295) — ensure it matches your problem assumptions.
- Macros overwrite or erase columns — use erase macros (U/W/X) carefully.
