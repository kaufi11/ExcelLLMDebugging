Summary — main error found
- Faulty cell: O19
- Problem: Wrong function used. O19 =COUNT(D19-M19) — COUNT returns how many numeric values are in its argument (here it returns 1), not the numeric difference you want.
- Resulting symptom: O19 shows 1 instead of the expected underspend 0 (D19 1 minus M19 1 = 0).

Fix
- Replace O19 with:
  =D19-M19

Why this fixes it (short)
- D19-M19 performs the intended subtraction (allocation minus period total).
- COUNT(D19-M19) only counts that single numeric result (always 1 when numeric), so it never gives the difference.

Other notes / clean-up suggestions (optional but recommended)
- Many “difference” cells use SUM(Dxx-Mxx). SUM(Dxx-Mxx) works (it evaluates the subtraction), but it’s clearer to use =Dxx-Mxx.
- Many totals use SUM(A+B+C…) — this works but is clearer and more robust to use either SUM(A,C,B,...) or SUM(A:Z) where appropriate.
- Replace COUNT or other aggregate functions only when they’re actually intended for counting.
