Findings (short)
- Error: Row 2000 (A20) in sheet "2003 Report" has two stray values in G20 = 8.449074 and H20 = 0.6381540899397388. All other rows have G/H blank on that sheet.
- Why it's a problem: G/H are outside the defined data blocks (releases in B–F and transfers in J–O). Those stray values can confuse reviewers, get accidentally included in SUMs/charts if ranges are broad (e.g., SUM(B:K)), or indicate a mis‑paste / misaligned data entry.

Quick fix (do this)
1. Clear G20 and H20 (select the two cells → Delete).
2. Search workbook for any formulas referencing G20 or H20 (Home → Find → =G20 or use Trace Dependents). If any formula uses them, correct the formula to point to the intended cells (likely in the J–O transfer block).
3. If those numbers belong to another column, cut them and paste into the correct column; otherwise leave them cleared.

Why this fixes it (simple)
- Clearing/removing stray values restores consistent layout so totals and charts only use intended columns. If they were mis‑placed data, moving them to the correct column preserves the value without corrupting summaries.

Extra suggestion (prevention)
- Convert the data area to an Excel Table or tighten SUM ranges (explicit ranges like SUM(B7:F20) and SUM(K7:O20)) so accidental entries outside the table won’t be included.
