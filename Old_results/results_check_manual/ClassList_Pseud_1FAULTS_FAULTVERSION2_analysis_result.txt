Summary of concrete formula errors / inconsistencies
1) Homework sheet — L5 excludes HW10
- Problem: Homework!L5 contains =SUM(B5:J5). All other student Total cells use SUM(Bx:Kx).
- Effect: HW10 (column K) is omitted from Apple/aya/etc. Example: aya (row 5) K5=10 is excluded; L5 value is 141 but B5:K5 sums to 151.
- Fix: Change Homework!L5 to =SUM(B5:K5).

2) Mid-term sheet — column B is being excluded from totals
- Problem: Mid-term totals (column L) use =SUM(Cx:Kx) (and the "Total possible points" in L5 is =SUM(C5:K5)), so column B is not included in any total. However column B holds numeric values in some rows (e.g. B11 = 3 for CAGIRL).
- Effect: Any numeric values in column B are ignored in totals. If column B is intended to be part of the scored items, all L formulas should include column B.
- Examples: Mid-term!L11 =SUM(C11:K11) = 83 but if B11=3 is meant to be counted the correct total would be 86.
- Fix: Decide whether column B is a real question/score column:
  - If yes: change the L formulas to =SUM(Bx:Kx) and change L5 to =SUM(B5:K5).
  - If no (B is some extraneous data column), keep as-is but document/label B as not being part of the totals.

3) Mid-term sheet — mislabeled / text cell instead of formula
- Problem: Mid-term!B46 contains the string "average(C4:C42)" rather than an actual formula. It looks like a leftover label/text while the actual averages are in C46:O46.
- Effect: confusing and inconsistent presentation.

4) Mid-term sheet — header duplication and potential ambiguity

5) Final sheet — inconsistent SUM ranges (extra H column included in row 7)
- Problem: Final!I7 uses =SUM(B7:H7) while nearly all other row totals use =SUM(Bx:Gx) (B:G are Q1–Q6).
- Effect: including column H (which appears blank) is inconsistent with other rows and may be an accidental typo.
- Fix: Change Final!I7 to =SUM(B7:G7) for consistency.

6) Homework and Mid-term sheet — AVERAGE over ranges that contain text flags
- Problem: Homework!N4:N42 (and possibly other columns) contain text flags ("HC", "P", "x"). Homework!N44 has =AVERAGE(N4:N42). AVERAGE ignores text, so the mean reflects only numeric members of the range.
- Effect: the computed average may be misinterpreted if the user expects all rows to be counted; also AVERAGE will silently ignore rows with text.
- Fix: Either ensure the averaged range contains only numeric data or compute average with a formula that explicitly counts numeric entries (e.g. =AVERAGEIF(N4:N42,">=0") or =AVERAGE(IF(ISNUMBER(range),range)) as appropriate) and document behavior.

7) Final Grades sheet — mixing numeric totals and manual letter entries
- Problem: Most Final Grades!F cells are numeric totals computed with =SUM(Bx:E x), but some rows contain manual text values (for example F4 = "F" rather than a numeric total).
- Effect: If you try to compute statistics (averages, sorts) on column F these text entries will break numeric calculations or be ignored.
- Fix: If you need numeric totals for all students, replace manual entries with the SUM formula and put the letter grade in a separate column. If the letter indicates special handling (HC/P), keep separate flags and leave F numeric only where appropriate.
