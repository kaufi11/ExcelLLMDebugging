Overview
- Workbook contains 3 sheets: "Portals comparison" (main), "Sheet2" (blank), "Sheet3" (blank).
- Purpose: vendor evaluation matrix based on San Diego State University rubric (note/credit in A3–A4). Compares Oracle Portal, uPortal, CleverPath and SharePoint across criteria grouped I–VII.

Layout (sheet "Portals comparison")
- Header and notes: A1 title; A3–A4 rubric attribution; G4 explains S = score, WS = weighted score.
- Criteria rows: rows 8–43 list individual criteria, grouped by section (I. Look and feel, II. Security, III. Personalisation, IV. Support, V. Standards, VI. Administration, VII. General).
  - Column A: section and criterion name.
  - Column B: short criterion heading.
  - Column C: weight (numeric).
  - Columns D–F: descriptive levels (Insufficient / Adequate / Excellent).
- Vendor columns: four vendor blocks starting at G, J, M, P respectively:
  - Oracle Portal block: G = Comments, H = Score (S), I = Weighted Score (WS).
  - uPortal block: J = Comments, K = S, L = WS.
  - CleverPath block: M = Comments, N = S, O = WS.
  - SharePoint block: P = Comments, Q = S, R = WS.
- Weighted score logic: each WS cell multiplies the corresponding Score by the criterion weight in column C (e.g., I8 = H8*C8). Totals for each vendor appear in row 44:
  - I44 = SUM(I8:I43) (Oracle total)
  - L44 = SUM(L8:L43) (uPortal total)
  - O44 = SUM(O8:O43) (CleverPath total)
  - R44 = SUM(R8:R43) (SharePoint total)

Key formulas & cells
- Typical weighted formula pattern: [WS cell] = [S cell] * [C (weight) cell], e.g., I8 = H8*C8; L9 = K9*C9; O16 = N16*C16; R41 = Q41*C41.
- Totals: I44, L44, O44, R44 sum the weighted scores for each vendor.
- Summary table at A45 onward contains per-section maxima and an overall maximum and vendors' overall totals:
  - B55 = Overall maximum (186)
  - C55 = OP (142) and D55 = uPortal (146.5) as provided in the summary (note: summary shows only two vendor columns in the posted excerpt).
