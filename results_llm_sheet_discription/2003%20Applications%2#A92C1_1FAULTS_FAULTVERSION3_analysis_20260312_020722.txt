Summary of main errors (short) and how to fix them

1) Wrong cell reference in Municipal row 5 (Hatboro)
- Problem: F5 contains =E6 (points to Hatfield's Request) instead of =E5. That makes the Recommend for Hatboro equal 26,000 (Hatfield) instead of 147,970 and produces a wrong Municipal recommended total (F22).
- Effect: Municipal F22 is understated by 147,970 − 26,000 = 121,970.
- Fix: Edit F5 and change the formula to =E5.
- Result after fix: Municipal F22 increases by 121,970 (from 1,952,645 → 2,074,615).

2) AHTF recommended subtotal excludes row 2 and has a non-numeric cell
- Problem A: F14 is SUM(F3:F13) which omits F2 (Canus). If you intend to sum all Recommend values in the AHTF table use F2:F13.
- Problem B: F10 contains the text "Pending Feasibility Study". SUM ignores text, so the current formula will treat that as 0 — that may be intended but can be confusing.
- Fix:
  - Change F14 to =SUM(F2:F13) to include the entire recommend column.
  - Replace the text in F10 with either a numeric placeholder (0) or leave it blank and document it, or enter a numeric recommended amount when decision is made. Optionally add a comment to explain “pending” rather than putting text in the cell used for calculations.

3) Hard-coded constant in AHTF Continuum of Care match
- Problem: E16 (and F16) use =2432890*0.15. That hard-codes 2,432,890 rather than referencing a source cell. If the base amount changes you must update it manually.
- Fix: Replace the hard-coded number with a cell reference that contains the base amount (for example =<cellWithBaseAmount>*0.15). If there is no single cell with that base currently, create one (label it) and reference it.
