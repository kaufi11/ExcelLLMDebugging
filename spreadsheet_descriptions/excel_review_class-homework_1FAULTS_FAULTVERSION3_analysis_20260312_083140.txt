Summary of workbook (3 sheets)

1) Sheet: Q'back — "Quarterback Efficiency" (B2)
- Purpose: stores QB passing stats and derived efficiency metrics.
- Layout: header row in row 4 (A4:L4) — Quarterback, School, Attempts, Completions, Interceptions, Yards, TD, Completion %, Touchdown %, Interception %, Yards/Attempt, Points.
- Data rows: A5:A11 (7 QBs); numeric inputs in C5:G11 (Attempts, Completions, Interceptions, Yards, TD).
- Key formulas (per row 5–11):
  - Completion % (H): =D#/C#*100
  - Touchdown % (I): normally =G#/C#*100 (note: Bryant row I5 shown as =G5/C5/100 — likely a typo that divides twice)
  - Interception % (J): =E#/C#*100
  - Yards/Attempt (K): =F#/C#
  - Points (L): =K#*8.4 + H# + I#*3.3 - J#*2
- Notes/issues:
  - Completion, TD and INT columns are percentages (0–100). The Points formula mixes these percent values directly (no unit conversion), so check whether I (TD%) should be in decimal form when multiplied by 3.3 (current layout multiplies a percent value).
  - Bryant (row5) has an inconsistent TD% formula that will produce an incorrect (very small) value.

2) Sheet: drug bust
- Purpose: time series of arrests for drug abuse violations by age group (1970–2000) with metadata.
- Layout: Years in A3:A33 (1970–2000), Adult arrests in B3:B33, Juvenile arrests in C3:C33; column E contains source/metadata notes (Bureau of Justice Statistics, filename, authors, contact, date, description).
- Key values: adult arrests rise through the 1980s/1990s, peaking around 1.37M in 1997–2000 (B30:B33 ~1,370,400–1,375,600). Juvenile arrests peak mid‑1990s (~211,100 in 1996).
- Notes: Column E is free text metadata; numeric time series is in A–C.

3) Sheet: lookup
- Purpose: student scores by course, with total/average and a grade lookup table.
- Layout:
  - Header row (A3:H3): Student, four course columns (B–E), F = Total /400, G = Average /100, H = Letter Grade.
  - Student rows A4:A9 with course scores in B4:E9.
  - Grade lookup table in F11:G18 (Points -> Letter): mapping entries (e.g. 45 → F, 50 → F, 55 → D, 60 → C, 65 → B, 70 → A, 75 → A).
- Notes/expected formulas (not present but implied):
  - Total (F#) should be =SUM(B#:E#)
  - Average (G#) could be =F#/4 or =F#/400*100 depending on intent (header "Average / 100" implies =F#/4)
  - Letter (H#) normally via VLOOKUP/LOOKUP against F11:G18 (ensure sorting or use exact match rules).

Overall layout notes
- Three distinct data areas: player efficiency (with many derived %/ratios), a long time series of arrest counts with metadata, and a student-score table with a grade lookup table.
- Watch units and consistency: percent vs decimal in Q'back formulas and Bryant's TD% formula which appears to be erroneous.
