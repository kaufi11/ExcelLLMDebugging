Summary — Sheet: Reporting

Layout
- Row 1 (B1:M1): Week labels (Week 4 → Week 15).  
- Row 2 (B2:M2): Corresponding date ranges for each week.  
- Column A: Major categories (Management, Models, Requirements, Elective, Studio I) and their sub-activities (Reading/Preparing, Homework, Class, plus meeting/role lines inside Studio I).  
- Columns B:M: Hours per activity by week.  
- Column N: Total hours per activity across all weeks.

Formula patterns
- Category subtotal rows (e.g., Management B3, Models B7, Requirements B11, Elective B15, Studio I B19) use vertical SUM formulas per week, e.g., B3 = SUM(B4:B6), B7 = SUM(B8:B10), etc., repeated across columns C–M.  
- Activity total (column N) mainly uses explicit addition across weeks, e.g., N4 = B4+C4+...+M4 (rather than SUM(B4:M4)). This pattern repeats for most N cells.  
- Grand weekly totals (row 35) sum the category subtotals: B35 = B3+B7+B11+B15+B19 (same for C35:M35). N35 = N3+N7+N11+N15+N19.

Key values / notable entries
- Large single-week entries: C9 = 22; D9 = 15.5; F9 = 18; K9 = 20.5; M5 = 24.5 — these are high homework/elective values.  
- Many rows are zero (roles like Team Lead, Client Liaison, QA, Support, etc.).

Anomalies / potential errors
- Row 24 (Client Meetings) has N24 = B24 - C24 - D24 - ... - M24 (subtractions) — likely a mistake; expected a sum across weeks.  
- Inconsistent use of SUM ranges vs. manual addition in column N (functional but less maintainable).

Overall
- This is an hours-tracking/reporting sheet organized by category → activity → week, with weekly subtotals per category and per-activity totals in column N, and grand totals per week in row 35.
