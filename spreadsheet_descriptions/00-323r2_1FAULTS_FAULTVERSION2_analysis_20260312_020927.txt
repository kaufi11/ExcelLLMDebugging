High-level overview
- Workbook: 3 sheets. Sheet1 contains the complete timing budget; Sheet2 and Sheet3 are empty.
- Purpose: compute setup/hold budgets and receive/transmit margins for two timing modes (columns B = Fast-80, C = Fast-160) and per-path setup/hold contributions (columns J/K/L and P/Q/N/R groups).

Layout and sections (Sheet1)
- Row 1: headers — A: Item, B: Fast-80, C: Fast-160, D: Comments; J/K/L: Setup / Setup / Hold; additional calculation columns appear in N–R and P–Q for device-specific derived values.
- Rows 4–6: Nominal timing (REQ/ACK period, data transfer period, ideal setup/hold).
  - Key values: REQ(ACK) period: Fast-80 = 25 ns, Fast-160 = 12.5 ns. Data transfer period: 12.5 ns / 6.25 ns. Ideal setup/hold: 6.25 / 3.125 ns (Fast-80) and 3.125 / 1.5625 ns (Fast-160 shown as 3.125 in C6).
- Rows 9–24: Non-compensatable error/input terms (tolerance, driver/receiver asymmetry, noise, crosstalk, chip noise, jitter, strobe offset, residual skew, data delay tolerance). Each term has values for B (Fast-80) and C (Fast-160), and half-cycle splits in J/K/L (many formulas = $Cxx/2).
  - Example entries: data driver asymmetry C11=0.5, crosstalk C17=0.7, cable skew C29=2.5, ISI of data C31=4 (worse-case).
- Row 25: Non-compensatable total = SUM(rows 9:24) for B and C (and J/K/L sums).
- Rows 27–35: Compensatable skew items (Tx/Rx chip skew, cable skew, two-trace skew, ISI of REQ(ACK), detection ambiguity) and their sums (row 35).
- Rows 36–38: Compensation (ISI compensation C36=2, skew compensation C37=4.4) and compensation total (C38 = SUM(C36:C37)).
- Rows 39–40: Total error inputs (B39 = B25 + B35; C39 = C25 + C35) and post-compensation error (B40 = B39 - B38; C40 = C39 - C38).
- Row 41: Minimum data valid window: B41 = B5 - B40 (similarly for C41). J41/K41/L41 compute per-path minimum setup/hold leftover using J6/K6/L6 minus corresponding errors.

Calculated device-specific columns
- Columns N, P, Q, R contain derived timing metrics and device-specific setup/hold calculations:
  - P12 = C5 - C9 - C12 - C15 - C21 (Transmit assertion period)
  - P13 / P14 compute Transmit Hold/Setup times subtracting a series of L- and K-column components (complex chained formulas)
  - P15/P16/P17/P18 compute receive hold/setup and net margins using P13/P14 and many J/K references and compensation terms.
  - Q variants adjust P values by ±3.125 or other offsets; R21 and R22 include extra constants (+5.4 -0.4).
  - N10/P10 and N23/P23 compute receive assertion periods and other pass-throughs.

Key formulas and patterns
- Sums: B25 = SUM(B9:B24), C25 = SUM(C9:C24), B35/C35 = SUM(compensatable range), B39/C39 combine non-compensatable + compensatable.
- Post-compensation: C38 = SUM(C36:C37); C40 = C39 - C38.
- Minimum window: B41 = B5 - B40 (and equivalent for C41).
- Many J/K/L cells are half-values of the C column terms (e.g., J11 = $C11/2).
- Several device formulas use absolute references ($P$13, $L$9, etc.) and some chain many terms (e.g., P17 includes many J- and J$ references and a -(J30/2)).
