Found errors (short list) and fixes:

1) Municipal sheet — cell F5
- Fault: F5 contains =E6 (points to Hatfield's request).  
- Why it’s wrong: It gives Hatboro the wrong recommended amount (shows 26,000) and duplicates Hatfield’s value in the Recommend total.  
- Fix formula: =E5  
- Explanation: F5 should reference its own row's Request (E5). If you want to enforce the 190,000 cap used elsewhere, use =MIN(E5,190000).

2) AHTF sheet — cell F14
- Fault: F14 contains =SUM(F3:F13) (omits F2).  
- Why it’s wrong: The AHTF recommended total excludes row 2 (Canus) if a Recommend value is ever entered there. This is an inconsistent range/omission.  
- Fix formula: =SUM(F2:F13)  
- Explanation: Totals should include the full block of Recommend cells (F2:F13) so newly entered recommends aren’t left out.

Recommendation
- After fixing, check any totals that use F22 (Municipal recommended total) or the AHTF totals in other sheets — they will update.  
- Quick audit tip: use Trace Precedents/Dependents or search for formulas that reference a cell one row off (common slip) to find similar errors.
