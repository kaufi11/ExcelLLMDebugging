Summary — MODEL07 (single worksheet containing both PUSH-DOWN and CONSOLIDATION sections)

Purpose
- Template for push-down accounting and consolidation exercises (Chapter 7). Includes areas to load problem data, analyze excess-cost and book-value elements, prepare push‑down adjusting entries, and post the basic elimination entry for consolidation. Macros automate data loading, postings, and clearing.

Layout (key areas / ranges)
- Header & instructions: rows 1–37 (file name, general instructions, macro usage notes).
- Analysis of investment—excess cost elements: rows ~15–23 (assigned amounts, SUM formula B22).
- Entries to implement push-down accounting (dr/kr list; macro Q posts): rows ~24–39. Dr/Cr totals formula: C39 = SUM(C27:C38), D39 = SUM(D27:D38).
- PUSH-DOWN worksheet: begins row 49. Columns show Old Basis (C), Adjusting Entries Dr (D) / Cr (E), and New Basis (F).
  - Balance sheet lines A54:A65 (Cash through Goodwill). Pattern: Fxx = (Cxx + Dxx - Exx) for asset lines (e.g., F54 = C54 + D54 - E54).
  - Totals: C65 = SUM(C54:C64), D65 = SUM(D54:D64), E65 = SUM(E54:E64), F65 = SUM(F54:F64).
  - Proof of postings: D76 = D65 + D74 and E76 = E65 + E74.
- Analysis of investment—BOOK VALUE element: rows ~78–85 (summation B85 = SUM(C85:F85)).
- Basic elimination entry area (to be posted to Consolidation; macro P available): rows ~88–96.
  - Elimination formulas use absolute refs to the book-value analysis (e.g., C90 = $C$85, C91 = $D$85, C92 = $E$85, C93 = $F$85, D94 = $B$85). Totals C95 and D95 are SUMs of those lines.
- CONSOLIDATION WORKSHEET: begins row 100.
  - Instruction / entry header rows 100–103.
  - Consolidation balance sheet layout: rows 104–132. Columns K:U hold stored problem data for multiple problem keys (Problems 7-1 through 7-7); these input values appear in K105:U132 per A136/A138 notes.
  - Consolidation line formulas replicate the push-down pattern:
    - For per-entity asset detail F105 etc: Fxx = Bxx + Cxx + Dxx - Exx (where applicable).
    - Totals per column: e.g., K118 = SUM(K105:K117), similarly for L118:M118 ... U118.
    - Liabilities & equity totals: C132 = SUM(C120:C131) etc; consolidated totals F118 and F132 sum component columns.
  - Proof of debit/credit for consolidation: row 134 (D134 = D118 + D132, E134 = E118 + E132).
- Problem data storage: explicit note A136 — problem data stored in K105:U132 (these are the values loaded by macros).
- Macro list & instructions: rows 138–184. Key macros:
  - A–G: load UNADJUSTED problem data into PUSH‑DOWN (and for some, CONSOLIDATION) for Problems 7-1 … 7-7.
  - P: post basic elimination entry to CONSOLIDATION.
  - Q: post unamortized excess-cost reclassification entries to PUSH‑DOWN.
  - U: erase columns C, D & E in PUSH‑DOWN; W: erase column B in CONSOLIDATION; X: erase postings in columns D & E in CONSOLIDATION.

Key formulas and patterns to note
- New basis (assets): F = C + D - E (push-down worksheet).
- Liabilities/equity rows sometimes use F = B + C - D + E (reflecting different sign conventions shown).
- Totals: column sums across rows for assets (e.g., SUM of 105:117 range) and for liabilities/equity (SUM of 120:131).
- Basic elimination links to book-value analysis with absolute references ($C$85, $D$85, $E$85, $F$85, $B$85).
- Proofs: verify postings by adding balance-sheet totals and liabilities/equity totals (rows 76 and 134).

Other notes
- Push-down worksheet entries begin at row 49; consolidation starts at row 100.
- Macros automate loading and posting; macro instructions and key-letter mappings are documented in rows 138–184.
- Many input cells in the consolidation area (K105:U132) are prefilled numeric examples for different problems; these drive the consolidated totals and proofs.
