- A few nonstandard/hard-coded formulas and references exist (examples: B26 =602+1 yields 603; B40 =+B53+1 and B47 =+B54+1 create forward/back references that may produce unexpected numbering or circular references).
- D1 is a volatile TODAY() function that updates on open.
- Many rows include simple status/role codes (H, C, D, M) entered as plain text.

Notable observations / potential issues
- The sheet mixes content types (departments, services, projects, potential projects) in one long table separated by section headings (e.g., "Desktop & Server Tools", "Registries & Directories", "eCommerce", "Email", "Web", "Authentication, Access and Roles").
- Column usage is consistent (roles in E–H, customers in I, dependencies K, notes L) but some header positions and repeated headings appear in multiple rows.
- Several B-column formulas refer to non-adjacent or later rows which could break numbering or cause circular references if rows are reordered or deleted.
- Some email addresses and names use inconsistent casing and domains (e.g., cotton.Stanford.EDU); entries include typos and freeform notes that may need cleanup.
