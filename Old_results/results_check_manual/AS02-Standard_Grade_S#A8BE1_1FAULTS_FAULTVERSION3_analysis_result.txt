- Examples: SG1!F44 (value is "-"), SG2 and SG1 have several "-" cells in numeric columns indicating “not applicable”.
- Problem/Note: Using the text hyphen prevents these cells from being included in SUM and COUNT as numbers. That may be intentional (to indicate N/A), but it can cause unexpected subtotal behaviour if a SUM was expected to include a zero. In the current workbook this likely contributed to the author using COUNT in F53 to count non-empty numeric cells (but the intended operation was a SUM).
- Recommendation: If a numeric zero is more appropriate use 0; if you want to show N/A but still sum, consider blank (empty) vs zero depending on the intended aggregation. Document which approach is intended.

- Problem/Note: These do not produce incorrect results here (they divide a cell by itself to return 1), but the inconsistent use of absolute addressing is a maintainability issue and can lead to errors if rows are moved/copied.

4) Percent-change formulas assume numeric denominators (watch for division-by-zero)
- Pattern: Many cells use =(F#-E#)/E#. If E# can be 0 or non-numeric ("-"), this will produce #DIV/0 or invalid results.
- Recommendation: If there's any chance E# = 0 (or text), wrap the formula in an IFERROR or test E#>0 before dividing, e.g. =IF(E#=0,NA(),(F#-E#)/E#) or =IFERROR((F#-E#)/E#,"").
