Summary of workbook structure and key contents

Overall layout (all main sheets)
- Common column headers (row 1 on each sheet): A: Applicant, B: Project, C: Code, D: Description, E: Request (numeric), F: Recommend (numeric or formula).
- Sheets: Municipal, Non Profit, County, Rehab, HOME, AHTF.
- Many Recommend (F) cells are formulas that copy Request (E) (e.g., F2 =E2) or contain numeric recommendations; several F cells are blank or contain anomalies.

General observations / potential data issues
- Cross-sheet linking is used for top-level aggregation (County sheet pulls Municipal!, Non Profit!, Rehab! totals; AHTF pulls HOME totals).
- Multiple duplicates: Rehab items also appear on Non Profit; ensure intentional duplication or separate accounting.
- Inconsistent Recommend column: many blank recommends, some set equal to requests via formula, some hard-coded amounts, one anomalous formula (F5 Municipal =690).
- Some totals reference different ranges (e.g., AHTF F14 =SUM(F3:F13) excludes F2).
- Text entries in recommend cells (e.g., "Pending Feasibility Study") will break numeric summation if not handled consistently.
