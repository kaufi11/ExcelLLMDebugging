Summary of workbook structure and contents

Sheets
- Profs (primary data table)
  - Header row (row 1) defines columns:
    - A: Number, B: Name, C: Position, D: University, E: Email address, F: Suggested by, G: Personal or Related Web Page, H: Address, (I: no header present), J: City, K: S/P (state/province), L: CNTY (country), M: Zip, N: Phone, O: FAX
  - Rows 2–63 contain individual contacts (academics, students, researchers).
  - Column A uses simple sequential formulas to generate record numbers (e.g., A3 = A2 + 1, A4 = A3 + 1, …). Most rows increment by 1; one anomaly: A29 = A28 - 1 which breaks the sequence.
  - Typical populated fields: Name (B), Position (C), University (D), Email (E — several entries use “?” or are blank), Suggested by (F), Web page (G) for many entries, address fields in H/I/J/K/L/M, phones/faxes in N/O (format inconsistent).
  - Notable complete record example: row 28 (Andrew T. Campbell) includes web page (G28), full street (H28), city/state/country/zip (J/K/L/M) and phone/fax (N/O).
  - Data issues / inconsistencies (observed): missing header for column I; uneven splitting of address components (some use I, others only H); inconsistent phone/fax formats; some emails marked “?”; occasional duplicated or out‑of‑order numbering due to A29.

- URLs (simple link list)
  - Single‑column (A) layout with alternating descriptive labels and URLs.
  - Examples: A4 = http://www.wpi.edu/, A11 = http://www.lcs.mit.edu/, A14 = http://www.media.mit.edu/, A18 = http://wind.lcs.mit.edu/, etc.
  - Not structured into separate description/URL columns (descriptions and URLs are on separate rows).

Key values / counts
- Profs contains entries from row 2 through row 63 (sequential numbering intended from 1 upward; final Number value is 62 under the standard A# = previous + 1 scheme).
- Many institutions repeat (Columbia, MIT, Rutgers, Univ. of California variants, Carnegie Mellon, etc.).
- Several entries have full contact details (address, phone, fax, web page); others have partial data or placeholders (“?”).

Formulas & layout notes
- Primary formula pattern: column A auto-increments by referencing the cell above (A[n] = A[n-1] + 1) with at least one exception (A29).
- No other formulas appear in the excerpt; most cells are plain text.
- The worksheet is a flat table intended for contact directory use; the URLs sheet is an unstructured list of link references.
