High-level summary
- Workbook contains 2 sheets: "Profs" (main contact directory) and "URLs" (list of project/site links).
- "Profs" is a row-based contact list (header row at row 1, data rows from 2 → 75) with one record per row. Column A holds a sequential ID generated by simple formulas; other columns hold status, identity, affiliation and contact details.
- "URLs" is a simple list of site names and their URLs (entries are entered on alternating rows: description then URL).

Profs sheet — structure and key columns
- Headers (row 1): 
  A: #, B: sent, C: Name, D: Position, E: University, F: Email address, G: Suggested by, H: Personal or Related Web Page, I: Address, J: (sometimes street/line 2), K: City, L: S/P (State/Prov), M: CNTY (used for country in many rows), N: Zip, O: Phone, P: FAX
- Row count: data through row 75 (many populated rows; some gaps around rows 50–51).
- Column B ("sent"): contains markers like "x" or "u" (likely mail/response status).
- Column A (record number): mostly generated by incremental formulas like A3 = A2+1, A4 = A3+1, etc.; exceptions:
  - A35 uses =A34+8 (creates a jump),
  - A52 uses =A49+1 (there are apparent blank rows 50–51),
  - Remaining rows follow the increment pattern to A75.
- Common contents:
  - C/D/E/F: person name, academic/research position, institution, email address (some email cells contain "?" where missing).
  - G: who suggested the contact.
  - H: personal/related web pages for select contacts (entered as full URLs).
  - I–P: address, city, state, country/region, zip, phone and fax where provided. Several rows include full postal and phone/fax details (examples: row 17 Kaveh Pahlavan; row 38 Andrew T. Campbell; row 35 Kevin Karcz; rows 61–62 Columbia student addresses).
- Data quality/notes:
  - Inconsistent use of columns J and M (J used only occasionally for a second address line; M header is "CNTY" but frequently contains country names).
  - Some emails and fields are unknown ("?").
  - A few rows include full international phone/fax numbers; formatting varies.
  - Several contact rows include personal web page hyperlinks in column H.

URLs sheet — structure and contents
- Single-column style list under A1 ("Random URLs"): descriptions and URLs alternate by row (e.g., A3 description, A4 URL; A6 description, A7 URL).
- Contains links to university labs, projects and working groups (WPI, MIT LCS, Media Lab, WIND, CM, CAG, CellularIP, IETF Mobile IP, CMU Monarch, MosquitoNet, etc.).

Overall layout and formulas
- Main structural formula pattern: column A uses simple arithmetic to produce sequential record numbers (A(n) = A(n-1)+1), with a couple of deliberate deviations creating numbering gaps.
- Data is primarily text entries (names, positions, emails, URLs, addresses); only column A contains formulas.
- Layout is flat/relational style (one contact per row, fields in fixed columns), suitable for filtering/sorting but has some inconsistent field use and missing data to clean up.
