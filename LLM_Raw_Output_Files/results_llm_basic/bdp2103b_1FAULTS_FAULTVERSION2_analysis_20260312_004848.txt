Summary — main error found
- Cell O95 (Set up costs, "underspend" column) contains a hard-coded value 7. It should be a formula (D95 − M95) like the other "underspend" cells. Because O95 is wrong, any subtotal/total that includes O95 (e.g. O99 and higher totals) will be incorrect by that amount.

Why this is an error
- D95 = 3 and M95 (SUM of months) = 3, so the correct underspend is 0. O95 = 7 is inconsistent with the row's data and the sheet's pattern of formula-driven underspends.

Fix (quick)
1. Select O95.
2. Replace the 7 with the formula:
   =D95-M95
3. Press Enter. Totals that reference O95 will update automatically.

Notes and small improvements (optional)
- Many cells use SUM(Dxx-Mxx) (e.g. O12 =SUM(D12-M12)). Excel computes that correctly, but it's confusing. Prefer plain subtraction (=D12-M12) for clarity and to avoid accidental mistakes.
- Also some SUM formulas use addition inside SUM like SUM(C12+C26+...). That works but is unconventional; better to use SUM(C12,C26,...) or simply add them.
