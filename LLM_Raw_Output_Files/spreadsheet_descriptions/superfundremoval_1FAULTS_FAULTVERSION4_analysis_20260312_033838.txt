Issues & anomalies to note (may affect calculations)
- Weight-sum checks compare SUM(range) to the literal "100%" text; but weightings are entered as decimals (e.g., 0.2, 0.1428). Because 1 ≠ "100%", the IF tests will return "ERROR" even when weights sum to 1.0. Either convert weight cells to percentages (100%) or change the check to compare to 1 (e.g., <>1).
- In Section II, cell G21 uses IF(C21="yes",(704*F21),...) — the multiplier 704 appears to be a typo (expected 1). This will produce an incorrect weighted score if C21="yes".
- Inconsistent use of rounding/precision for multi‑choice factors: Section IV uses 0.33 and 0.67 in IFs, whereas weights are 0.333/0.334 — minor rounding inconsistency may slightly affect totals.
- Some F cells are zero (F54, F55) and corresponding G cells explicitly set to 0; these are marked N/A.
- Several formula cells return empty string "" for unanswered/other options (consistent pattern).
