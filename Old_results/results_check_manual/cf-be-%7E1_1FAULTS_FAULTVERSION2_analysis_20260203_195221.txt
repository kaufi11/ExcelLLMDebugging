4) Inconsistent / unnecessary use of SUM() and + prefixes (clean-up, not fatal)
- What’s wrong: Many formulas use SUM() around single-cell arithmetic (e.g. =SUM(B6+B14)) or + at the start (e.g. =+O23/O$57). This isn’t an error but is confusing and can hide mistakes.
- Fix: Use clean formulas like =B6+B14, =O23/O$57, etc.
- Why: Cleaner formulas are easier to audit and less likely to hide copy/paste errors.

5) Variance % formulas risk #DIV/0
- What’s wrong: Variance formulas like =(D8-C8)/C8 will return #DIV/0 if the base (C8) is zero or blank.
- Fix: Wrap with a safe test, e.g. =IF(C8=0,"", (D8-C8)/C8 ) or =IFERROR((D8-C8)/C8,"").
- Why: Prevents error values showing up and makes reports more robust.

6) Check TOTAL % columns that divide totals by themselves
- Note: P57 in Year1 is =O57/O$57 (total sales divided by total sales) — it will always equal 1 (100%). That’s not an error if you intend 100% in the % of Sales column for total, but it is redundant. Confirm intent.
