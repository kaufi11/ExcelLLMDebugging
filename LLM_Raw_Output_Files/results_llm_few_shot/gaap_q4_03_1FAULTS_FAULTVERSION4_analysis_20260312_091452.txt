Fixes (short, exact formulas) and short explanations

1) Q4 sheet — D14
- Fault: D14 =COUNT(D9:D12) returns 4 (count of numeric cells) instead of summing cash flows.
- Fix: replace with =SUM(D9:D12)
- Why: COUNT returns the number of numeric cells; you need SUM to get total free cash flow.

2) All sheets — percent change cells (example pattern H9, H11, H12, H14)
- Fault: current formula =((D9-F9)/F9)*100 will produce incorrect results when F9 is zero and can be left hard-coded (H11 = 0 in some sheets).
- Safe replacement (preferred): =IF(F9=0,NA(),(D9-F9)/F9*100)
- Alternative (show absolute change when prior = 0): =IF(F9=0,D9-F9,(D9-F9)/F9*100)
- Why: IF prevents #DIV/0 and avoids hard-coded zeros; NA() or a text string signals that % change is not defined when base = 0.

Apply the same pattern to H11, H12, H14, and any other percent-change cells (replace cell refs as needed, e.g. for H11 use F11/D11 references).
